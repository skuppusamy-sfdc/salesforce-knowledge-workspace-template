# Test Scenarios: PR1050558-6510 — Clinician Contract: Remove 'Refresh Client Rates' & Move functionality to the Clone process

## Source

### Acceptance Criteria (verbatim)

**AC1: Remove "Refresh Client Rates" from the 'Summary' tab**
Given I am a recruiter within the Corporate Support apps
When I am on a **draft** Clinician Contract
Then on the Summary tab, I shall no longer have button = "**Refresh Client Rates**"

**AC2: Update the cloning process in "Contract Request" guided process**
Given I am a recruiter within the Corporate Support apps
When I am on a Clinician record with an Executed Clinician Contract
And I select "Contract Request" guided experience
And select "Start from Previous Active Contract"
And upon completion of the guided experience, when the system is cloning the contract
Then when creating the cloned contract, the system shall run the 'Refresh Client Rate' and:
- Hard Delete rate(s) that are NOT in the PO Contract (see story for matching logic PR1050558-4630) [Existing Functionality]
- Update the 'Facility Rate' [lookup to the appropriate PO Contract Rate (PCA)] & 'Facility Level Rate' [Facility's Rate] on the Preventive Care Agreement (PCA) [Existing Functionality]
- Update 'Is Deleted' to Checked/True for incorrect Time Type — Part of story PR1050558-6492
- Do NOT update current rate based on Facility Rate — **REMOVE THIS LOGIC**, users will select 'Configure Rate' and decide how the clinician's rate should be updated based on updated Facility Rate and Previous Rate.

**NOTES (from AC):**
- If a Draft was created prior to moving the "Refresh Client Rates" to the Cloning process, the Recruiter will need to cancel and start a new contract.
- If there is a new PO Contract executed while the Clinician Contract is still being worked on (status = draft), then the recruiter will need to cancel and start a new contract.

### Solution (verbatim)

Remove the button and move the functionality to Contract Cloning Process as Asynchronous JOB at the time of Contract Clone Creation when the asynchronous job clones the CPA, PCA records. The logic already exists in APEX **ContractSyncRemoteAction**.

### Refinement Notes (active — non-struck-through items only)

- Rate values that will change
- Shifts that will be deleted (view only)
- Recruiter can choose: Refresh all locations OR Refresh only primary portfolio locations
- Summary shown before confirmation

### Design Notes

Two tables:
1. Rate values to change: Group by HPF (accordion display), sorted by Type of Rate (CPA type), show all current rates: Facility Rate (from current PO contract), Current Rate, Previous
2. Shifts to be deleted (not part of PO Contract) — show for ALL Facilities, cannot unselect. Same columns as #1.

Remove updating of Rates to match the Facility rate. User should select "Configure Rates" and update the Current Rate as needed.

## Objective

Validate that:
1. The "Refresh Client Rates" button is completely removed from the Clinician Contract Summary tab for all contract statuses.
2. The rate-sync logic (previously triggered by the button) now executes automatically as an asynchronous job during the Contract Clone process ("Start from Previous Active Contract").
3. The clone-time rate sync correctly: hard-deletes unmatched rates, updates Facility Rate lookups, marks incorrect Time Type records as deleted, and does NOT auto-update Current Rate based on Facility Rate.

## Prerequisites

- **Data**:
  - At least one Clinician record with an Executed Clinician Contract
  - A Practice Operations (PO) Contract in status "Executed" or "Superseded" with RecordType = "Client" linked to the same facility/account
  - PO Contract with ContractPaymentAgreement (CPA) records of types: Shift, Stipend, Variable, Target, Leadership Incentives
  - PO Contract with PreventiveCareAgreement (PCA) records under each CPA, including rates, shifts, and Facility Titles
  - Clinician Contract with CPA/PCA records that match AND do not match the PO Contract (for delete and update scenarios)
  - PCA records with varying Time Type values (for PR1050558-6492 logic)
  - Multiple ProviderNetworkContract (HPF/Facility) records on the Clinician Contract
  - Rate_Pay_Code__c records associated with PCA records (for cascade-delete validation)
- **Permissions**: Corporate Support profile or equivalent (recruiter role)
- **Environment state**: QA or Sandbox environment with the feature deployed; prior "Refresh Client Rates" button should have been present in the UI before this deployment
