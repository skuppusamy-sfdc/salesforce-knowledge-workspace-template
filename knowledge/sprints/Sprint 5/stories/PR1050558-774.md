## Description

As a SFO (Staffing Fulfillment Optimization) user, I want the ability to generate facility rates by designating the type of rate and specified rate for each Type & Facility Title combinations, so that we have clearly defined rates for each Service Account

## Acceptance Criteria

Given I am a SFO in the Corporate Support or Corporate Support Console app

When I navigate to a **Contract** record (record type = **Client**) 

And clicked the **Specify Rates** button 

Then I am presented with a Rate Type screen to capture:

- Type of Rate (Contract Payment Agreement, field = Type)

** Required to select one from the following list of values to proceed to the next screen:

*** Stipend

*** Shift

**** **Variable*

*** Target

*** Leadership Incentives

*** Bonus

- Variable Type

** Select one option between (required):

*** Encounter

*** Other Variable Types

Then when I select **Variable**, I see the Rate Details screen and I have the following option to:

- Go back to the previous screen (button = Previous)

** And if I selected go back to the previous screen (button = Previous), then user is returned to the Type of Rate screen with the previously selected value

- Populate the following to proceed:

** Variable

**** **Type*

***** If “Other Variable Types” was selected in the previous screen, then select Type from (required, multi-select): **Basic 1B, Basic 1B HM, Basic RVU, Fixed Pooled, Practice RVU, QPI, Legacy Encounter*

***** If Encounter was selected in the previous screen, only **Encounter* is displayed

***** In back-end but not available to select: **Basic, Basic 1A, Basic 1C, Basic RVU/Calculated, Modified, Modified 2.0, Variable Pooled*

*** +Encounter+ (When Type = Encounter is selected, then display the following fields:

**** Daily Encounter Threshold (required, text area (255))

**** Daily Encounter Pay Cap (required, text area (255))

*** +All Types+ display the following fields:

**** Target Plan Rate (optional, currency (16, 2))

**** # Months Provider Guarantee (optional,  number (18 15, 0))

**** # Months Startup Guarantee (optional,  number (18 15, 0))

**** RVU Metric Type (optional, picklist: RVU, wRVU)

*** Facility Title

**** select between (required, multi-select): all Facility Titles

*** Rate

**** Required, editable Currency field (16,0); user to capture whole number

**** Unit of Measure (required); per Metric, % Salary Reduction, per Monthly Tier, per Hour, per Shift, per Month

And upon clicking **Finish,** then the system shall create the appropriate Contract Payment Agreement, and **Preventive Care Agreement** records as specified in the solution.
