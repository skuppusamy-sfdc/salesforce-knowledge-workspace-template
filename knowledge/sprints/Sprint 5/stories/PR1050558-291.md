## Description

As a Data Quality team member I need to specify facility titles for clinicians at each facility so that I have accurate information to move forward with contracting and onboarding

## Acceptance Criteria

Given I am a Data Quality team member in the Corporate Support apps

When I select facilities in the Create HPF wizard

Then I am **able to select all Service Accounts** - including those where a clinician already has a HPF where Status = Active

Given I am a Data Quality team member in the Corporate Support apps

When I select facilities in the Create HPF wizard

Then I am able to **specify the following**:

- Facility Title, required (select element)

- Time Type, required (select element)

- Practice Type, required (select element)

- Effective From, default to Today but can update (date)

And the specified attributes apply to all selected facilities

## Solution

- Refer UX-291.jpeg file as Mock UI to introduce four new fields.

- When User click save - populate all four attributes on newly created HPF records.

- Updated error to cover both search scenarios: No Service Account records found, or the Service Account record you searched for is already selected.

** This story is allowing flexibility for the user, they could have one Active and one prospect for the **same* HPF. In the future if business needs to restrict then we will add a new story. 

- Update Security Matrix, see tab = Permission Set Group. 

*** Adding **Permission set =** Manage Clinician Identifiers, Credentials, Specialties to **Permission set group* = Data Quality.   

- NOTE: No validations on the Effective From date, allowing business the flexibility to select any date.
