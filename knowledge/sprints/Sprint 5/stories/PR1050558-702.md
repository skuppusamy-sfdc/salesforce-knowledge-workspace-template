## Description

As a Clinician Contracting Specialist, I need records in Salesforce to be automatically created or updated when a Clinician contract is signed, so that the appropriate team members can begin their necessary activities in a timely manner

## Acceptance Criteria

_Story focus: Just creating the new fields._

Given I am a Clinician Contracting Specialist

When a Contract record where Record Type = Clinician is updated so that Status = Executed _note: this is done via automation from CLM, but to test this story the update can be done manually_

Then the following **fields are visible** records are updated:

** Healthcare Practitioner Facility  **Contract Status field  Executed (NOTE all field mapping will be a new story 1071)*

** Latest Contract Signed Date 

** Effective From (only when HPF has status = Prospect) [existing field]

** Anticipated Termination Date (when applicable, only specific titles)

- Email notification to account team and others that contract has been signed to be handled with scheduled reports

- Integration from Salesforce to QGenda and Workday to create clinician in story 1071

## Solution

Create the fields “Lastest Contract Signed Date” and “Anticipated Termination Date” in Object “Healthcare Practioner Facility” as per the DD.

Build a record triggered flow which is executed when Contract status is set to Executed. Query all related HPFs (via Provider Network Contract records) and update mentioned fields as specified in the AC. Refer Data dictionary for field api names. 

As discussed integration is out of scope for this sprint.
