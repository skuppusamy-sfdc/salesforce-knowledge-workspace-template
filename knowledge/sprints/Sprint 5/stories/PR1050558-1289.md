## Description

Steps to Reproduce:

1-Login as SFO User into the Corp HC Org and navigate to the Contract Client Record.

2-Open a Contract Client record with Status New.

3-Navigate to Summary tab and start the Specify Rate Flow.

4Select Type of Rate  Leadership Incentive, Stipend etc.

5-Select Details as follows: (example)

Type of Rate - Leadership Incentive

Type - Lead NPPA Quarterly At Risk Stipend, Director Incentive

Fac title - Medical Director, Interim Medical Director, EE System Medical Officer

rate - 75

unit - per shift

6-Now submit the flow and navigate to the Summary tab.

Issue 1 - We can see an extra record is visible for the Type of Rate, Type, Rate, Facility TITLE, Unit of Measure Combination.

Expected Result - An extra record should not be created.

Actual Result - We can see an extra record is visible for the Type of Rate, Type, Rate, Facility TITLE, Unit of Measure Combination.

Issue 2 -  Any type of Type of Rate records are not getting created and are not visible under Summary tab.

Expected Result - Any type of Type of Rate records should be created and should be visible under Summary tab

Actual Result - Any type of Type of Rate records are not getting created and are not visible under Summary tab.

Issue 3 - If existing Shift record is present on the Summary tab and user creates another Shift record with different set of combination, then also on Summary tab previously created Shift record is being displayed, newly created records are not visible.

Expected Result - If new combination are getting created then it should be displayed correctly on the Summary tab.

Actual Result - If existing Shift record is present on the Summary tab and user creates another Shift record with different set of combination, then also on Summary tab previously created Shift record is being displayed, newly created records are not visible.

## Root Cause

Old IP version was activated previously, may be got activated by someone by mistake.

Also while testing multiple contract records with the new status were present which was stopping the records to from getting generated.

Deployment and Configuration: Environment Mismatch
