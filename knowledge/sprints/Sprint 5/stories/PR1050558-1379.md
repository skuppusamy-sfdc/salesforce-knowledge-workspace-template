## Description

Steps to Reproduce:

1-Login as SFO in the Core HC Org and Navigate to the Corporate Support apps.

2Navigate to the Contract object and select All Client list view to  open a Contract Record of record type  Client.

3-On Contract (Client) and verify its Status is New and Navigate to Summary tab to confirm Specify Rates button is displayed.

4-Click on Specify Rates button and select Shift and Employed NP/PA value.

5-Click next and verify the Type field.

Issue 1 - When user selects Shift and Employed NP/PA value then under Type field has:

A The Columns name should contain  Shift, As of now it is - Full Name, Start Time, End Time, Day of the Week. 

Expected Result  The columns name should be  Shift Full Name, Shift Day of the Week, Shift Start Time, Shift End Time.

Actual Result  As of now it is  Full Name, Start Time, End Time, Day of the Week. 

B The Columns should be displayed as  Shift Full Name, Shift Day of the Week, Shift Start Time, Shift End Time. As of now it is Full Name, Start Time, End Time, Day of the Week. 

Expected Result  The Columns should be displayed as  Shift Full Name, Shift Day of the Week, Shift Start Time, Shift End Time. 

Actual Result - As of now it is Full Name, Start Time, End Time, Day of the Week.

## Root Cause

'Shift' hardcoded word was missing from column name. Raised by SF QA team as part of the related story testing during the sprint itself  [+PR1050558486+|https://salesforce.atlassian.net/browse/PR1050558-486]
