## Description

As a SFO, I need shifts to be filtered, so that I can configure rates appropriately

## Acceptance Criteria

Given I am an SFO in the Corporate Support apps

When I navigate to a Service Account record

And I open a Contract (Client) record where Status = Draft

Then I am able to Specify Rates from the Summary tab

- When I select Type Of Rate = Shift

- Then I have to select a single Shift Type (required)

** NP/PA, MD/DO, RN/LPN

** If NP/PA is selected, display Employment Type field (required)

*** Employed, Other

- When I navigate to the next screen: 

** Shifts are filtered according to the selected Shift Type

*** And filtered by NP/PA selection if applicable. 

** Facility Titles are filtered according to the selected Shift Type

*** See attached PDF or [Health Cloud Records|https://docs.google.com/spreadsheets/d/1B6WddJnXHDZD5kx5F1mZDqEvcAwYn1wCPejZ0ZgJpP0/edit?gid=719228658#gid=719228658] sheet (Tab = Code Set (Facility Title))

** If NP/PA was selected and Employment Type = Employee, then Time Type is available

builds on functionality from [https://salesforce.atlassian.net/browse/PR10505581415|https://salesforce.atlassian.net/browse/PR10505581415|smart-link] 

originally built in [Facility Rate Creation  Type of Rate = Shift: Add Shift Type|https://salesforce.atlassian.net/browse/PR1050558486]
