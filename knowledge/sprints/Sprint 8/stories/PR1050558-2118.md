## Description

As a Clinician Contracting Specialist, I need the Contracting Corporate Entity field on Contract records to be populated, so that I can accurately generate clinician contracts with the appropriate entity without manual review

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I associate a HPF to a contract

Then the Contracting Corporate Entity field is automatically populated from the following logic: 

- If all HPF records related to the Contract have the same Contracting Corporate Entity

** Populate the Contract’s Contracting Corporate Entity field with the same Account record from the HPF record(s) based on Professional Title and Employement Type (from Facility Title record)

- If HPF records have different Contracting Corporate Entity values

** Manual review is needed  process described in [https://salesforce.atlassian.net/browse/PR1050558301|https://salesforce.atlassian.net/browse/PR1050558301|smartlink]

## Solution

Use the flow created in [https://salesforce.atlassian.net/browse/PR1050558301|https://salesforce.atlassian.net/browse/PR1050558301|smart-link] 

Use the Guided flow when creating a new HPF to add the Corporate entity value to Contract’s Corporate entity for the first record. For the other HPFs if the same value as CE, do nothing (as its already updated)

If the value is different, it will be handled by story 301. 

**IF Contract.RecordType = Clinician and Employment Type = Employee** THEN

1. Look for all records from HealthcarePractitionerFacility (Contract> PNC > HPF) HealthcarePractitionerFacility.Contracting_Corporate_Entity__c of all record

## IF HPF.Contracting_Corporate_Entity__c is  the same for all HPF records THEN 

#**Update** Contract.Contracting_Corporate_Entity__c to that same Account name value

Update the CE to blank out when HPF’s Corporate Entity is different from newly added HPF
