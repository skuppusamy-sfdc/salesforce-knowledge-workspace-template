## Description

As an Integration User, When HPF is Inactivated then I need a trigger from SF to update Qgenda

## Acceptance Criteria

Given I am an Integration User

When the HPF becomes inactive then I need to send a Platform Event to Qgenda with that ID

- Platform Event Fields, Name = HPFUpdates

** HPF Id

** Global Provider Id

** Global Facility Id

** Inactive Date

** Type = “Inactive Date”

Note this should be triggering when HPF undergoes Facility Tittle Change or Employment Type change. 

## Solution

Create a Platform Even Name = HPFUpdates

Fields

1. HPFId = [HealthcarePractitionerFacility.Id|http://HealthcarePractitionerFacility.Id]

1. GlobalProviderId = HealthcarePractitionerFacility.PractitionerId.Global_Provider_ID__pc

1. GlobalFacilityId = HealthcarePractitionerFacility.AccountId.Global_Facility_ID__c

1. InactiveDate = HealthcarePractitionerFacility.EffectiveTo

1. Type = “Inactive Date”

 

Create/Update a Record Triggered flow and when the HealthcarePractitionerFacility status becomes inactive and HPF.FacilityTitle changes OR Employment Type changes

Security Matrix permission: **Data Integration**, **System Administrator**

Need to update the flow which sends HPF inactive PE, to send PE only when all HPFs for the Practitioner are inactive

select id, Status__c from HealthcarePractitionerFacility where PractitionerId  = ‘003gP000006jWwLQAU' and Status__c != 'Inactive’
