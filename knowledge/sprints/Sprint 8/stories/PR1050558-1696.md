## Description

As a Recruiter, I need to set Start Dates for clinician contracts, so that I can specify when the requested contract is effective

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I navigate to a Clinician record and open the Contract Request experience

Then I am able to:

- specify Start Date

** date field, required

- specify End Date

** date field, not required

*** If End Date was not populated and facility title is Nurse Practitioner or Physician Assistant then End Date on **Contract and Provider Network Contract* is Start Date + 365 (able to handle leap year scenario)

** _Note: there is no business use case for an IC contract having NP/PA and other facility titles in the same contract_

Given I am a Clinician Contracting Specialist in the Corporate Support apps

When a new clinician contract is updated so that Status = Executed

And this clinician has a prior Contract in Status = Active where Employment Type = same value as the newly executed contract

Then following updates occur automatically: 

- Currently Active/Original Contract

** End Date = 1 day before Start Date of newly executed contract

** only update when End Date is blank, do not override 

** [https://salesforce.atlassian.net/browse/PR10505581718|https://salesforce.atlassian.net/browse/PR10505581718|smart-link] describes status update to Canceled, this update needs to be removed as status should not change

## Solution

Add two date fields in the Contract request process, in the Contract selection screen

Start Date (Required), End Date (optional)

In the OS, check if the facility title IS is Nurse Practitioner or Physician Assistant and if End date is not populated, then End Date is Start Date (that was entered) + 365. 

Map these to StartDate, EndDate

Use the existing flow on Contract (that is used to change the status of contract to Executed)

Query for existing contract for the clinician (using current contract record’s AccountId)

Check for contacts that match employment type and with active status and update the End date to start date of current (flow) contract record

Remove the older implementation of changing the status to Cancelled and Cancel Reason from PR1050558-1718
