## Description

As a Recruiter, I need to set Start Dates for clinician contracts, so that I can specify when the requested contract is effective

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I navigate to a Clinician record and open the Contract Request experience

And the Clinician **has an Executed Contract**

And selected: **Start From Previous Active Contract**

Then I am able to:

- specify Start Date

** date field, required

*** **pre-populated with Executed Contract’s information; user can update*

- specify End Date

** date field, not required

*** **pre-populated with Executed Contract’s information; user can update*

** If End Date was not populated and facility title is Nurse Practitioner or Physician Assistant then End Date on Contract and Provider Network Contract is Start Date + 365 (able to handle leap year scenario)

** _Note: there is no business use case for an IC contract having NP/PA and other facility titles in the same contract_

And if the user selects **“Start From Previous Active Contract”**, then

** The System will set the **Start Date & End Date** inputted on **all the cloned records*

-Given I am a Clinician Contracting Specialist in the Corporate Support apps-

-When a new clinician contract is updated so that Status = Executed-

-And this clinician has a prior Contract in Status = Active where Employment Type = same value as the newly executed contract-

-Then following updates occur automatically:- 

- Currently Active/Original Contract

** End Date = 1 day before Start Date of newly executed contract

** only update when End Date is blank, do not override 

** [https://salesforce.atlassian.net/browse/PR10505581718|https://salesforce.atlassian.net/browse/PR10505581718|smartlink] describes status update to Canceled, this update needs to be removed as status should not change-

^ duplicate of [Clinician Contract: Set Start Date|https://salesforce.atlassian.net/browse/PR1050558-1696]

## Solution

While cloning, for CPA - Convert Contract’s Date to Datetime field and use the generic time

For PCA, no need to create/populate start date and end date, as it does not exist on the object.
