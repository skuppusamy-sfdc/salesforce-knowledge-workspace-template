## Description

As a Data Quality team member, I need an automated task created when clinicians complete their intake form, so that I can have a reminder to review the information provided for accuracy and completeness

## Acceptance Criteria

Given I am a Data Quality team member in the Corporate Support apps

When an intake form is completed for a clinician where their Account.Name does not match the Legal Name provided on the intake form

Then I am able to see that a task has been automatically created with the following information:

- Subject = Review Clinician Legal Name

- Record Type = Default

- Assigned To = Data Quality Queue

- Status = Open

- Due Date = 1 business day from automation start

- Type = Other

- Priority = Normal

- Name = clinician record

- Related To = assessment record (intake form)

## Solution

As part of the Omniscript that (is used for Intake form as part of discovery framework) 

Compare the Legal name with the Clinician’s Name 

if there is a difference, 

Create a Task based on the above condition. (If feasible use the Decision matrix)
