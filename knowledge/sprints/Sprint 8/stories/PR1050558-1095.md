## Description

As a Clinician, I need to complete my Conflict of Interest Form, so that I can provide necessary information to move forward in the recruiting process and it is retained for downstream onboarding processes.

## Acceptance Criteria

Given I am a Clinician with an open Intake Form case 

When I log in to the Core Connect portal

Then I am able to navigate to the Home tab and see the To Do component [Assessment component to launch]

And I am able to populate the following:

- This form is only for clinicians where Employment Type is Employee (check HPF records where Status does not equal Inactive, Facility Title lookup has a Sub Type of Employee)

** **Conflict of Interest Form* questions (For questions see document = [Clinician Intake Form Case_Questions|https://docs.google.com/spreadsheets/d/1tU55QDz-GqwJRIo73ywH_nGGh1BAckZnEfNuNVK_AZU/edit?usp=sharing]), tab = Conflict of Interest screen)

## Solution

Create Question set for the mentioned type of document - and add the appropriate questions and response.

This intake form applies only to clinicians where Employment Type is Employee (check HPF records where Status does not equal Inactive, Facility Title lookup has a Sub Type of Employee)

Create discovery framework OS, and add the question set per requirement.

deploy the OS in exp cloud

Assign permission for Discovery framework objects to External users/clinicians as needed - in a new permissions set

populate Completed Date when all questions are answered 

populate Case  lookup on Assesment (Intake Form)

NOTE from sakthi : Need to ensure we update the security, currently Portal User permission only has read access to tasks/activities/case.
