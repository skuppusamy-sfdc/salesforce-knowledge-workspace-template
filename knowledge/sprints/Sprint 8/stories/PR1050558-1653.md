## Description

As a user, I need the ability to view a component that highlights the **Offboarding** status(es) of a clinician, so that I am able to understand where they are during their journey with SCP Health.

## Acceptance Criteria

_Story Focus: Update At a Glance to support_ **_Offboarding_**

Split from Story [https://salesforce.atlassian.net/browse/PR1050558889|https://salesforce.atlassian.net/browse/PR1050558889|smart-link] 

Given a user of the system,

when I navigate to the **Person Account Clinician record At a Glance tab**,

Then I see a component with the following details:

- Titled “At a Glance” (see wireframe)

- A key/legend directly below it (see wireframe)

** Grey Circle Icon “Not Started”

** Blue + White Circle Icon “In Progress”

** Green Circle with Check Icon “Completed”

- Sections displaying “Section Header” values from the reference doc (“Status Label + Logic” tab - link below)

** Multiple Section Headers can be visible at the same time

** Sections have accordion feature (collapse / expand)

** Status/Label values (internal/external) visible nested under Section Headers only when section header is available

** Icons from the legend are available below the Status/Label value (see wireframe in the solution)

*** In Onboarding and **Offboarding sections, Internal is visible above Facility*

Split from Story [https://salesforce.atlassian.net/browse/PR1050558888|https://salesforce.atlassian.net/browse/PR1050558888|smart-link] 

Given I a **SCP Corporate Support user** using the Corporate Support App,

When I navigate to a clinician’s Person Account record,

Then I am able to

- View “Status / Label (Internal)” under the Section Headers from the “Status Label + Logic” tab when a Section Header is visible

** Values in “Status / Label (External)” IS NOT VISIBLE TO INTERNAL USERS

Given I am **external Experience Cloud user**,

When I navigate to my home page,

Then **Offboarding** Section Header **& Status Labels are NOT displayed.**

Split from Story [https://salesforce.atlassian.net/browse/PR10505581334|https://salesforce.atlassian.net/browse/PR10505581334|smart-link] 

Given an **SCP Health user using the Corporate Support app**,

When I navigate to a clinician’s “At A Glance” / Pizza Tracker component on their Person Account page,

Then the data displayed is deduced / displayed leveraging the following logic:

- No actual records are displayed within the component

- Sections displaying “Section Header” values from the reference doc (“Status Label + Logic” tab - link below)

** Conditionally display and hide headers based on logic from reference doc (“Header Logic” tab - link below)

*** Section Header names will be concatenated to display what section is referring to (ex. Job Name or HPF Name) (“Header Logic” tab - link and example below)

** There will be a Section Header (“Applicant”) and subheaders that are collapsible with the section header concatenated nested under it (see example below - horizontal view)

** Multiple section headers can be visible at the same time (ex. Applicant, Active, Onboarding can all be visible simultaneously)

- If no logic is documented and Section Header is visible, default icon/value to “Not Started”

- Icon values (Not Started, In Progress, Completed) are dynamically displayed based on the Logic in linked Reference Doc (“Status Label + Logic” tab  columns FH)

** Only one status per Status/Label are visible at a given time.

** The most “progressed” status is what is to be displayed

*** If Completed Logic = TRUE, display Completed icon

*** If In Progress Logic = TRUE and Completed = NOT TRUE, display In Progress icon

*** If Not Started Logic = TRUE OR (if no logic is documented AND Section Header is visible AND In Progress or Completed Logic = NOT TRUE), display Not Started icon

Given an **external user using Experience Cloud**,

when I navigate to my “At A Glance” / Pizza Tracker component on my home page,

Then the **Offboarding Section Header & Status Labels are NOT displayed.**
