## Description

As a Payroll team member, I need a notification when a clinician can’t be created in Workday because they share a tax ID with an existing clinician, so that I can manually create the clinician

## Acceptance Criteria

Given I am a Payroll team member in the Corporate Support apps

When there is a failure in clinician creation in workday due to a duplicate tax ID

Then I navigate to the Reports tab

And see the following **report**:

- Clinician Creation in Workday Failure 

** Filters: Error Type = Integration, Error Message (CONTAINS)= Workday

** Columns to Display: Name, Error Timestamp

And I receive the following **email notification**:

- Recipient = [PHYSPR@scphealth.com|mailto:PHYSPR@scphealth.com] (actual email for IC Payroll Team)

- Subject = Clinician Creation in Workday Failure 

- Message = 

Hello, 

The following clinician has not been created in Workday due to a duplicate tax ID. Please review the following report and review.  

and see a record has been created with the following attributes: 

- Error Log

** Error Type = Integration

** Error Message = Workday  Tax ID Number Already Exists  confirm with Mulesoft team

## Solution

Dependency : Workday has to create error log record with the Type : Integration and error message containing the word “Workday” and the error message specified above.

Create a report on Error log object

Create a conditional subscription to the report - when the Error log record is created, with the contains “Workday” keyword.

Need a list of users to subscribe to the report

OR

Create a flow on Error log object : condition type : Integration and Error Message : Contains workday, to send a email to the “[PHYSPR@scphealth.com|mailto:PHYSPR@scphealth.com]“ . with the message specified above and a link to the report.
