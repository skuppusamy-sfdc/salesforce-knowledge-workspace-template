## Description

As a RMO, I need the ability to capture feedback from an exit interview initiated rom an offboarding case, so that when a clinician is offboarding, we’re able to gather feedback of the exit interview in a formal capacity.

## Acceptance Criteria

Given I am a RMO (Regional Medical Officer), DMO (Division Medical Officer), DOO (Division Operating Officer), CMO (Chief Medical Officer) in the Corporate Support apps

_Note: no validation needed on who can create an exit interview_

When I navigate to a clinician’s SCP Offboarding case

Then I can open the Exit Interview child task from the related tab (click path updated in [https://salesforce.atlassian.net/browse/PR10505582237|https://salesforce.atlassian.net/browse/PR10505582237|smart-link] ), I can click “New Exit Interview” button from the header (only available when Subject = Exit Interview)

Then I am brought to an edit screen with access to following fields:

- Contact Name - set to Person Account (Clinician) the initial Case was related to, read only

- Case Owner - set to [Current User], read only

- Parent Case - set to SCP Offboarding case, read only

- Status - Not Started, In Progress, Closed

- Description - long text area

Then I click Save and the Exit Interview Case record is created :

- Subject: Exit Interview Notes

- other fields populated from entry above

Create page layout for Exit Interview case

NOTE: Per story [+2081  Opportunity Team (Deal Team)+|https://salesforce.atlassian.net/browse/PR10505582081] - **Chief Medical Officer** (CMO) Account Team role is being replace with:
