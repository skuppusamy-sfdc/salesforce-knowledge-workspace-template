## Description

As a RMO, DMO, DOO, CMO; I need view and edit access to the Exit Interview object to records I have created, so that I can ensure sensitive data is securely viewed and edited

## Acceptance Criteria

Given that I am a RMO (Regional Medical Officer), DMO (Division Medical Officer), DOO (Division Operating Officer), CMO (Chief Medical Officer) associated with all HPFs for the clinician where status does not equal Inactive

When I navigate to a Clinicians Person Account

Then I can view / edit “Exit Interview” cases

Given that I am NOT a RMO, DMO, DOO, CMO specified above

When I navigate to a Clinicians Person Account,

Then I cannot view any exit interview cases

NOTE: Per story [+2081  Opportunity Team (Deal Team)+|https://salesforce.atlassian.net/browse/PR10505582081] - **Chief Medical Officer** (CMO) Account Team role is being replace with:

** **Chief Medical Officer - EM (NEW)*

** **Chief Medical Officer - HM (NEW)*

Persona : Users with Manage Clinician Recruiting permission set will have access view / edit “Exit Interview” cases

## Solution

- OWD for Case = Private

- Add case team for exit interivew case record and add the account team members in the roles specified above as case team members, with read/edit permission. ( navigate to Case > Clinician > All HPF> All SA  account team roles  This would have been retrieved as part of 884. ).
