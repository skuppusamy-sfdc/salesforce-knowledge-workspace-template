## Description

**Steps to reproduce:**

1. Login as Recruiter - Myrtle Will

1. Open a clinician contract whose related Service account has a Practice Operations contract. Practice Operations contract should have rate values configured for Bonus

1. From Summary tab of clinician contract, click ‘Configure Rates’ button

1. Select the Type of Rate as ‘Bonus’ and click ‘Next’ button

1. Select a Type from the multi-picklist field

1. Enter values for all required fields and more than 3 digits for below fields and click ‘Finish’ button

#** **Payroll Cycle* (number, 2 digits 0 decimals)

#** **Duration Months* (number, 2 digits 0 decimals)

#** **Work Commitment per Month* (number, 3 digits 0 decimals)

**Expected Result:**

User should not be allowed to enter more than the mentioned digits or get an error message when ‘Finish’ button is clicked

**Actual Result:**

User is allowed to enter more than the mentioned digits and click ‘Finish’ button. But the additional rate is not getting saved. Ideally they should get an error when entering more than the allowed number of digits

## Root Cause

The field length was correctly configured on object level, since story did not specify the validation on front end hence it was not added considering it as user training part.
