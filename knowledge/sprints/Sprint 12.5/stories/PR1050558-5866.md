## Description

When a PO contract is executed under a Service Account with matching rates and shift types, Given that there are multiple clinician with same & mixed rate clinician contract, then after batch runs, system incorrectly pushes rate updates to **Mixed Rate Clinician contracts**  even when those clinicians should be excluded.

**Steps to Reproduce:**

1. **Setup:** Ensure a Facility/Service Account has multiple clinicians assigned.

1. **Configuration:** Set one clinician to a **Mixed Rate contract** and others to Same rates across shift type.

1. **Action:** Create  & Execute a PO contract where the **Rate** and **Shift Type** match the existing Service Account criteria.

1. **Trigger:** Generate CCM records so that both **Process Type = Automation** and **Process Type = Manual** records exist for the same period.

1. Run the batch Clinician coverage batch from Dev console

1. **Observation:** Inspect the contract rates for the Clinician on the "Mixed Rate" plan. New Facility Rate is updated on the contract.

**Test evidence:**

## Root Cause

We needed to update Condition as per new requirement that instead of running queueable job for not cancelled clinician contract we need to run for **not cancelled and not supplement** Clinician Contract.
