**Role:** Data Quality

**Test Case ID:** Gold Set final tab 25- terminate EE all site (involuntary)

**Description:** 

We do not want the SCP Offboarding Case to be eligible to be closed until all Facility Offboarding Cases are completed/closed for a provider, as this will cause the HPFs to not be inactivated. The validation rule should stop the user from marking the parent SCP Offboarding Case with a status = closed, when any associated facility offboarding boarding case(s) is not equal to closed; otherwise, the HPF will remain in its current state and not terminated/inactivated.

Robert Dobson in UAT2 was our example.

## Acceptance Criteria

Given I am a MSL or data quality team member in the Corporate Support apps

When the Clinician offboarding process is initiated as part of the guided experience [PR1050558306: Clinician Offboarding  Termination Details|https://salesforce.atlassian.net/browse/PR1050558-306]

And both SCP Offboarding & Facility Offboarding case(s) are created

Then the system shall prevent the user from updating the SCP Offboarding (Type = SCP) case Status = ‘Closed’ until all of the Clinician’s Facility Offboarding (Type = Facility) case(s) are in Status = ‘Closed’.

And display error message: Please close all Facility Offboarding case(s) by ensuring child tasks are completed OR updating the ‘Offboarding Decision’ prior to closing the SCP Offboarding case.

And upon creation of the Facility Offboarding case(s), the system shall auto populate the “Parent Case” field with the SCP Case.

## Solution

When creating the Child (Type = Facility, RT = Clinician_Offboarding) case Use the SCP case Case.Id field to populate the Parent Case on the Facilty case Case.ParentId

Existing Logic: All Child Task(s) for Case (Type = SCP, RT = Clinician_Offboarding) is Closed via Record Triggered Flow in Task the Case gets Closed

Now Logic: All Child Task(s) for Case (Type = SCP, RT = Clinician_Offboarding) is Closed via Record Triggered Flow in Task the Case gets Closed only when the All the Child Task(s) are closed and also all the Child Cases is also closed

Also when the Child Case is Closed check if all the Child Case is closed and also all child task for the parent Case is closed then Close the Parent Case
