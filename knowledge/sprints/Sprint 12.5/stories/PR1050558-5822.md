## Description

As a System Administrator I want to add a new Account/Deal Team Role = ‘Risk Management',  so that Risk management team can be added to service accounts with Account Access = read/write access and make the edits needed in the 'Insurance Information’ section.

NOTE: Account Access = Read/Write will give edit access on that specific Service Account fields that the user has FLS permissions on; user may not be limited to just specific sections.

## Acceptance Criteria

AC1: Add Team Role = “Risk Management”

Given I am a system administrator within the Corporate Support apps

When user wants to add a Account or Deal Team (opportunity team) member

Then they should be able to add Team Role = “Risk Management”

AC2: User with Permission Set Group = ‘Risk Management Group’ can edit specific Service Account fields (dependent on story [+PR10505585823+|https://salesforce.atlassian.net/browse/PR10505585823])

Given I am a user with **Permission Set Group = ‘Risk Management Group’**

When I have been added to the Account Team on Service Account with **Account Access = Read/Write** (example: Team Role = ‘Risk Management’)

Then I can edit the following fields in the ‘Insurance Information’ Section

- Insurance Provided By

- Insurance Effective Date

- Insurance Expiration Date

QA Note: for AC2, please be sure to test by adding the following users to the account team:

1. with a user with the Persona = HealthCloud RestrictedUser (Read access to account only) + New Permission Set Group = ‘Risk Management Group’, to ensure users with no edit access on account, if they have this new permission set group then they can edit the above mentioned fields. (can be tested with [+PR10505585823+|https://salesforce.atlassian.net/browse/PR10505585823])

1. with a user with the Persona = MSL (edit access on account), to ensure they too have edit access on the above mentioned fields. 

NOTE: Data Quality is the persona that manages Account Team members. [+PR10505585493+|https://salesforce.atlassian.net/browse/PR10505585493]+.+ Based on the user’s permissions which fields they have access to view or edit can vary.
