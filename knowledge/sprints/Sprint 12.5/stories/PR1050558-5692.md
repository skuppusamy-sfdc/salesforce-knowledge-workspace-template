## Solution

### **AC1: Start New Contract Request**

** User enters **Contract Start Date** and **Description* (required). (first screen - existing functionality)

** When the message _"No previous contract found to perform a change to clone from"_ appears, add a **"Start New Contract Request"** option **when the query don't return anything to clone from*.

** **Finish/Save:* Create a new PO Contract (Draft, Practice Operations, Owner = Account Owner, Previous Contract = blank) and navigate to it.

** **Cancel:* Close the guided experience and return to the Service Account.

** **Implementation:* Add a conditional branch in `Clinician_ChangeRequest_English_2`, create an Integration Procedure to create the contract, and wire Finish/Save and Cancel.

### **C2: Clone from Superseded Contract**

** **Selection rule:** Use PO contracts with Status = Executed or Superseded, ordered by **Start Date dec* (newest first).

** Choose the contract where the user’s **Contract Start Date* falls within its period: `StartDate <= user date` AND (`EndDate >= user date` OR `EndDate` is blank).

** **Implementation:* Update `DRGetPOContract` to use `ORDER BY StartDate Dec` and adjust filters for EndDate (including blank).

- Step 2 already shows Contract Number, Start Date, Status, and Coverage Change; confirm it uses the new selection.

### **AC3: Update End Date and Status on Execution (use a FLOW) and**
