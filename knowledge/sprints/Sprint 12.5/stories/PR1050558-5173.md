## Description

As a SFO, I want the system to automatically update related Clinician executed Contracts based on specific type of updates made to the **Practice Operations** contract record, so that Clinician Contracts stay in sync with the Client contract and contract generation in QGenda can be appropriately triggered

## Acceptance Criteria

+AC1: Check Practice Operations Contract Change+

Given I am a SFO in the Corporate Support Apps

When a Client Contract (RT: Practice Operations) Status gets updated to ‘Executed’

Then the type of change (manual, agent, or automation) is identified based on the following logic ([+link+|https://docs.google.com/spreadsheets/d/19DsMwfjs0JSxk6DUu0KoafnznoQSOS83XHbvXCKMEA4/edit?usp=sharing] to supporting document)

+AC2: Check Clinician Contract Change+

Given I am a SFO in the Corporate Support Apps

When a Client Contract (RT: Practice Operations) Status gets updated to ‘Executed’

Then the system shall generate one task for each of the affected HPFs that follow the Manual path:

- Task record type: Default

- Assigned To: Account Team - Clinician Experience Specialist

- Subject: New Clinician Contract Needed - Coverage Change

- Name: [Clinician]

- Related To: [Client Change Management] _note: this has HPFs linked_

- Comments: Review clinician rate for coverage only change. This clinician has rates that are different from other rates with the same type. If an adjustment is needed, request a new clinician contract record. 

+AC3: Skip Agreement Generation (Contract Record Creation Only)+

Given I am a Recruiter in the Corporate Support Apps

When I generate a clinician contract

Then I am able to use the Coverage Change Flag below the SCP Book Travel field (default Unchecked)

_Note: Contracting Specialists can manually update Contract records to Executed Status when they see that this flag is checked_
