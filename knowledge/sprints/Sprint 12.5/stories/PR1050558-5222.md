## Description

**Tester:** Matthew Hebert

**Role:** Recruiter

**Test Case ID:** IC NPPA terminates and new Contract start date creates a gap in coverage

**Description:** 

For an IC NPPA, the original Clinician Contract record/HPF will have an End Date/Effective To Date one year from the original effective date. As built, if the Recruiter submits a request for the new future contract and uses an effective date of one month after the original end date, then the system will  push the end date of the original contract to the day before the effective date of the new contract. While that is appropriate for contracts that do not already have an End Date (evergreen contracts) it cannot do this for a fixed term contract, since the result would be an inconsistency between the paper contract and the system data

## Acceptance Criteria

As a recruiter that amends an existing active IC NPPA agreement, I want the system to NOT extend the End Date by one year if an End Date exists, so that the terms of the original agreement are still compliant.
