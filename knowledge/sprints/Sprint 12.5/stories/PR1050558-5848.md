## Solution

+**AC1 -- HR Setup: remove 2 tasks, conditionalize I9 + case creation**+

  Flow 1: `Case_Creation_After_Insert_Flow`

  • Add a new Get Records element: query HealthcarePractitionerFacility where:

    • PractitionerId = Clinician's ContactId

    • Employment_Type__c = 'Employee'

    • Status__c ≠ 'Inactive'

    • Status__c ≠ 'Superseded'

  • Add a Decision before adding "HR Setup" to subjectListSCP:

    • If HPF record found then add "HR Setup" to list

    • If NOT found then skip "HR Setup" entirely (no child case, no tasks)

  Flow 2: `Subflow_to_create_Task`

  • +Remove+ the following elements from the HR Setup branch:

    • Assign_Task_Fields_for_W2_Completion

    • Add_task_record_to_task_list_for_W2_Completion

    • Assign_Task_Fields_for_Workday_Termination

    • Add_task_record_to_task_list_for_Workday_Termination

  • Rewire the HR Setup decision branch to go directly to Assign_Task_Fields_for_I9_Update

  • I9 Update is now the only task under HR Setup (no additional condition needed here since the parent case is already gated)

  +**AC2 -- Add "Initial Clinician Outreach" to Training & SCP University**+

  Flow 2: `Subflow_to_create_Task`

  • After Add_task_record_to_task_list_for_compliance (end of current Training node), add:

    1. New Get Records: query HealthcarePractitionerFacility where:

      • PractitionerId = Clinician's ContactId

      • Employment_Type__c IN ('Locum', 'Leased', 'Managed')

      • Status__c ≠ 'Inactive'

      • Status__c ≠ 'Superseded'

    2. New Decision: "Has Active Locum/Leased/Managed HPF?"

      • If TRUE then skip (back to loop)

      • If FALSE then create task

    3. New Get Records: query AccountTeamMember where:

      • TeamMemberRole = 'Clinician Experience Specialist'

      • AccountId = relevant facility account

    4. New Assignment: Assign_Task_Fields_for_Initial_Clinician_Outreach

      • Subject = 'Initial Clinician Outreach'

      • WhatId = Training & SCP University child case Id

      • WhoId = Clinician ContactId

      • OwnerId = CES Account Team Member UserId (fallback: Case Owner)

      • Priority = 'Normal'

      • RecordTypeId = Default Task RT

    5. New Assignment: Add task to taskList

    6. Connector: back to Loop_on_Child_Case

  +**AC3 + AC4 -- Legacy records**+

  • Inherently covered. Removing task creation logic doesn't affect existing records. No data cleanup.
