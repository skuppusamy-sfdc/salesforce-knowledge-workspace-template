## Description

**Submitter:** Matthew Hebert

**Role:** Clinician Contracting Specialist

**Description:** 

Syed Tayyab-Contract No. 00471798.  As a specialist viewing a contract record submitted by a recruiter, under the Summary tab there is no visual indication that a rate is new from the rate identified on the Previous (Executed) Contract. It seems to indicate that nothing has changed for that site when there are numerous changes

**Steps to Reproduce:**

Navigate to Syed Tayyab Person account. Navigate to the Agreements Tab. Select the approved Contract. Click the link to open the Previous Contract to open side by side with the new. Navigate to the Summary Tab on both. Compare the rates listed on each and compare. Note that there is no visual clue as to what is new and what is carried over.

**Expected vs. Actual:**

The rates that are crossed out do not seem to indicate that the rate changed. I also expected there to be a visual indicator of what was carried over and what was new

**Record Link:** 

[https://corehc.lightning.force.com/lightning/r/Contract/800Ps00001GDU0AIAX/view?ws=%2Flightning%2Fr%2FAccount%2F001Ps00001KlY3eIAF%2Fview|https://corehc.lightning.force.com/lightning/r/Contract/800Ps00001GDU0AIAX/view?ws=%2Flightning%2Fr%2FAccount%2F001Ps00001KlY3eIAF%2Fview]

**Business Impact:**

Cannot tell what is new and needs a new agreement or what is still the same without making a line by line comparison.

## Acceptance Criteria

AC1: Update the how the Rates are displayed on the Summary Tab

Given I am a Recruiter or Clinician Contracting Specialist within the Corporate Support Apps

When I am viewing a Clinician Contract in any Status

Then on the Summary tab, I can see the following information displayed for each rate (Preventive Care Agreement) record: (see mocks below)

** **First Line:* <Facility Title> | <Time Type>

** Only display “| <Time Type>” if time type if available

** **Second Line*: Current: <Rate> <Unit of Measure> (Display in bold)

** **Third Line*: Previous: <Previous Rate> <Unit of Measure>

** **Forth Line:* Facility: <Facility Level Rate> <Unit of Measure>

AC2: Update how the system compares/determines change in Rates for Clinician Contracts

Given I am a Recruiter or Clinician Contracting Specialist within the Corporate Support Apps

When I am on a Clinician Contract

Then on the Summary tab, the system will identify if there is a rate change by comparing:

- Clinician’s current rate (field = ‘Rate’) to previous rate (field = ‘Previous Rate’, populate via the cloning process)

**OR** if ‘Previous Rate’ is blank, then

- Clinician’s current rate (field = ‘Rate’) to the facility rate (field = ‘Facility Level Rate’, rate from most relevant Practice Operation contract)

And if a rate change is detected (over or under rate), then the system shall: **(BASED ON SOLUTION DESIGN, update AC, able to do preferred.)**

- For cloned rates: Cross out/strikethrough the previous rate & unit of measure listed in “Previous”

- For newly added rates: Cross out/strikethrough the facility rate & unit of measure listed in “Facility”

- next best: Cross out/strikethrough the whole 3rd line (Previous: <Previous Rate> <Unit of Measure>)

- last option: No cross out/strikethrough

## Solution

**AC1 - Update rate display layout**

• DataRaptor (loads PCA records into Summary tab) -- add Previous_Rate__c, Facility_Level_Rate__c, Unit_of_Measure__c, Time_Type__c to extract output for UI Card

• OmniUI Card SpecifyRates_SummaryContractScreen (new version) -- update rate card template to display 4 lines per approved mock: header shows Facility Title (and Time Type for NPPA if populated), followed by Current rate + UoM, Previous rate + UoM (blank if none), and Facility rate + UoM. Format currency as $X.XX

• GetContractRates.cls -- add Previous_Rate__c, Facility_Level_Rate__c, Unit_of_Measure__c, Time_Type__c to SOQL queries on lines 10 and 14

• For the FLS, verify Clinician Contracting PSG has Read access on PreventiveCareAgreement.Previous_Rate__c and PreventiveCareAgreement.Facility_Level_Rate__c if not present please add to permission set

**AC2 - Rate change comparison + strikethrough (preferred option)**

• In OmniUI Card template, add conditional CSS to rate + UoM only

• If Previous_Rate__c is not blank, compare Rate__c vs Previous_Rate__c; if different, strikethrough Previous rate [$X.XX only] + UoM; if same, no strikethrough

• If Previous_Rate__c is blank, compare Rate__c vs Facility_Level_Rate__c; if different, strikethrough Facility rate + UoM; if same, no strikethrough
