## Description

**Tester:** Nicole Fontenot

**Role:** Legal – Client Contracting

**Description:** 

User mistakenly activated an amendment that is  not yet signed. When attempting to correct the status from "Executed" back to "Submitted" the system is not permitting the change.

**Steps to Reproduce:**

Need edit functionality for Status and Category post Executed status. 

JIRA PR1050558-3351

Given I am a Legal Team Member within the Corporate Support Apps

When navigate to a Contract (RT: Client Contract) record

Then I have the ability to select the ‘Activate Contract’ guided experience

Finish/Save

User is returned to the Client Contract

And the system shall make the following updates:

Populate “Activated Date” with current/today’s date

Populate “Activated By” with the user who initiated ‘Activate Contract’

Updates “Status” = “Executed”

And the system shall locate the previous contract in status = Executed (only if it was “Amended” - Previous Contract populated) and make the following update:

Update “Status” = “Superseded”

**Expected vs. Actual:**
