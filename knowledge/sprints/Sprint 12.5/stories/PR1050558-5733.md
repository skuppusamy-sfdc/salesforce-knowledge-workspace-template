## Description

**Tester:** Lisa Kleyla

**Role:** Payroll

**Description:** 

Under the shift rate, we are not able to select a unit of measure of "per shift" for a NPPA call rate.

**Steps to Reproduce:**

From the PO contract link below, click specify rates for an employed NPPA and select Call 1 (NPPA) and attempt to select the unit of measure of "per shift".

**Expected vs. Actual:**

We need to select per shift for certain rates, such as call shifts.

## Acceptance Criteria

AC1: Updates story [+PR10505582018+|https://salesforce.atlassian.net/browse/PR10505582018], updates in blue to update Shift Type = ‘Call’

Given I am a SFO in the corporate support apps

When I configure rates for a Practice Operations contract in Status = Draft

And I am specifying shift rates for Employee NP/PA (Type of Rate = ‘Shift', Shift Type = ‘NP/PA’, & Employment Type = ‘Employed’)

Then for ALL Shift Types **EXCEPT** for the following, I must select Unit of Measure = ‘per Hour’, if not display error message = ‘Unit of measure has to be per Hour NP/PA Employee Shift rate'

** Shift Type (AKA 'Shift Name') = ‘**Shift Bonus*’, then user must select Unit of Measure = “per Shift” (see story 2430 - ”Shift Bonus requires 'Per Shift' as the Unit of Measure. Please update to proceed. Alternatively, select individual shifts to choose a different Unit of Measure.”)

** Shift Type (AKA 'Shift Name') = ‘**Call*', then user can select Unit of Measure = ‘per Shift’ or ‘per Hour’. 

** Error message 1 (unique): ‘Unit of measure for ‘Call’ has to be ‘per Hour’ or ‘per Shift’ for NP/PA Employee Shift rate'

** Error message 2 (generic): ‘Unit of measure has to be ‘per Hour’ for NP/PA Employee Shift rate; except for ‘Shift Bonus' or ‘Call’ which can be ‘per Shift’.’

## Solution

**Dev Notes (Updated):** 

**OS:** ServiceAccFacilityRateCreation

1. Shift Type “**Call**“ will allow NPPA only Unit per Measure: “per Shift“ **AND** “per Hour“ 

## Error message 1 verbatim for validation 

**Notes:** Shift Type “Call“ is  available and active in Dev1 and higher orgs
