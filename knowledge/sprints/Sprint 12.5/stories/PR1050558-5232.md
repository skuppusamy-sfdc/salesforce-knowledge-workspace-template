**Description:** 

Validation Rule Needed: Any Parent Case should not be able to be completed until all associated child cases and/or activities have been completed. The users should work and complete all child cases timely and allow the system to auto-close (complete) the parent case. 

Robert Dobson in UAT2 was our example

## Acceptance Criteria

Given I am a MSL or data quality team member in the Corporate Support apps

When the Clinician offboarding process is initiated as part of the guided experience [PR1050558306: Clinician Offboarding  Termination Details|https://salesforce.atlassian.net/browse/PR1050558-306]

And Clinician Offboarding case(s) are created (Type = SCP, Facility)

Then the system shall prevent the user from updating the case ‘Status’ = ‘Closed’ until all of the child tasks are closed (task Status = ‘Completed’ or 'Canceled')

And display error message: Please ensure all child tasks are completed.  This will close the Offboarding case.

See tab = ‘[Clinician Offboarding|https://schumachergroup.sharepoint.com/:x:/r/teams/UnifiedPlatformReGenesis/Shared%20Documents/Program%20Documentation/Program%20Execution/1.%20Build,%20QA,%20Refinement/Client%20%26%20Clinician%20Experience/Clinician%20Onboarding_Offboarding%20%20Cases%20and%20Tasks.xlsx?d=w22ea8633b18a4e0cb5220132e36559eb&csf=1&web=1&e=1X5hpM]’ for list of tasks under each Type of Case

## Solution

When creating the Child case Use the Case.ParentId field to populate the Parent Case 

Record Triggered Flow Use in Case after update ((Type = SCP, RT = _Clinician_Offboarding_, Case.Status = Closed)  use Case.ParentId to query the child records status Status != Closed (count) if the Count> 0 and also check the child Task for Case (Type = SCP, RT = _Clinician_Offboarding_) then prevent the  Parent Case  (Type = SCP, RT = _Clinician_Offboarding_) closure
