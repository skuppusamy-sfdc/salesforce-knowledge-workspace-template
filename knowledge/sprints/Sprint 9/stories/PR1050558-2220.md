## Description

As a Data Quality team member, I need an automated task created when clinicians complete their intake form, so that I can have a reminder to review the information provided for accuracy and completeness

## Acceptance Criteria

Given I am a Data Quality team member in the Corporate Support apps

When an intake form is completed for a clinician where their Contact.NPI__c does not match the NPI provided on the intake form (Contact NPI can be blank or populated)

Then I am able to see that a task has been automatically created with the following information:

- Subject = Review Clinician NPI

- Record Type = Default

- Assigned To = Data Quality Queue

- Status = Open

- Due Date = 1 business day from automation start

- Type = Other

- Priority = Normal

- Name = clinician record

- Related To = assessment record (intake form)

## Solution

As part of the Omniscript used to display NPI form ( part of [231|https://salesforce.atlassian.net/browse/PR1050558-231]) , 

Retrieve the Contact’s NPI information

when the NPI field from form does not match the Contact’s NPI 

At the end, after the assessment record is created (as it is used as RelatedTo/WhatId) 

      Use IP to create task with details mentioned above.

If Data Quality queue does not exist, please create one.
