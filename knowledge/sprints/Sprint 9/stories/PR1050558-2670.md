## Description

As an internal corporate support user, when a Clinician’s name changes, I want the ability to create a case for the Data Quality team so that internal manual processes can begin.

## Acceptance Criteria

AC1: Ability to create a case for Data Quality team for Name Change

Given I am a internal corporate support user within the Corporate Support apps

And I have access to the Clinician record

When there is a Clinician Name change

Then from the Clinician record, I have the ability to create a Case with Record Type = “Clinician Change” 

And upon initial case creation of record type = Clinician Change:

- Subject = Clinician Name change for <Clinician> (system to set upon save)

- Description = <user inserts additional information as needed> 

- Contact Name = <system pre-populates with the Clinician’s name>

- Type = User can select “Name Change” (required)

- Status = “Not Started” (default)

- Priority = “Medium” (default)

- Case Origin = “Manual” (default)

- Case Owner = “Data Quality” queue

NOTE: once case is created for the data quality team, internal manual actions will need to occur as done currently.  Data team will need to manually close the case once all actions are complete. 

QA Team: per the design, “Type” will be the first field to answer.

## Solution

Create a new Record type of Case: Clinician Change

Add picklist option to Type  : Name Change

Add the fields mentioned below to layout and LRP.

- Subject = Clinician Name change for <Clinician> (system to set upon save)

- Description = <user inserts additional information as needed>

- Contact Name = <system pre-populates with the Clinician’s name>

- Type = User can select “Name Change”

- Status = “Not Started” (default)

- Priority = “Medium” (default)

- Case Origin = “Manual” (default)

- Case Owner = “Data Quality” queue

Before Save flow on Case: 

Filter based on Record type : Clinician Change

Set Case Owner to Data Quality queue

Conditions if Type = Name change

Set the Subject Clinician Name change for <Clinician>

Trigger Points: 

Standard Option : Clinician > Cases related list > new single related list added to LRP - Preferred
