## Description

As a HR team member, I need to see clinician attributes from Workday in Salesforce, so that I can complete necessary tasks with all information

## Acceptance Criteria

Given I am a HR team member in the Corporate Support or Corporate Support Console apps

When I navigate to a Clinician (Person Account) record

Then I can see the following attributes:

- Workday Employee Id _note: team is not aware of business need to have this in Salesforce_

- Hire Date

- I-9 Order Status

- E-Verify Status

What happens when clinician is terminated in Workday - the following dates are cleared in Salesforce

- Workday Employee Id

- Hire Date

- I-9 Order Status

- E-Verify Status

Clinician switches employment types

- IC to EE - fields are populated (AC1)

- EE to IC - fields are cleared only if clinician doesn’t have any outstanding EE HPFs (AC2)

- EE to Leased - fields are cleared only if clinician doesn’t have any outstanding EE HPFs (AC2)

## Solution

Create the fields mentioned on clinician person account.

FLS : Manage Clinician Recruiting permission set, add read permission

- NO Integration needed in this story.

- Confirm Account fields match the labels specified in the user story

- Create I-9 Completed field on Contact object & add to Person Account layout

- Update permission sets (anyone who can see Account (Service Account) can see these fields); includes the Data integration permission set.
