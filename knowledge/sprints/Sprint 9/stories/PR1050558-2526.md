## Description

As a Recruiter, I want the system to hide/show the “Submit Contract Request” button, so that I am only allowed to select when appropriate.

## Acceptance Criteria

Feedback from 9/3 onsite

- Display the “Submit Contract Request” when Status = “Draft”

- Hide the “Submit Contract Request” when Status = “Submitted” 

----

Given I am a Recruiter in the Corporate Support apps

When I navigate to a clinician record

And I have specified all necessary rates through the Contract Request experience

Then the system should dynamically hide/show the select **Submit Contract Request, Add HPF, Configure Rates** on the Summary tab of the Contract record based on:

** Contract (RT: Clinician) Status **= “Draft”*, then show/allow user to select and proceed

** Contract (RT: Clinician) Status **≠ “Draft”*, then hide/do not allow the user to select and proceed

## Solution

On the ClinicianContractSummaryScreen flexcard,

add conditional visibility for the three buttons ( Add HPF, Submit COntract request, Configure rates) to show the buttons only when 

Where Contract.RecordType Name = Clinician AND

Contract.status = Draft
