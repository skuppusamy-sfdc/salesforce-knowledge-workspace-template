## Description

As a system administrator, I want to update my Clinician Onboarding permission set group to support our Open Shift APAY Request process, so that it’s faster and simpler to set up users.

## Acceptance Criteria

Given I am a System Administrator in the Corporate Support apps

When I am in Setup > Permission Set Group 

Then I can see that Permission Set Group = Clinician Onboarding contains Permission Set = Manage Clinician Contracts

Given I am a System Administrator in the Corporate Support apps

When I am viewing Test User set up for:

- Liz Bennet (Clinician Experience Specialist)

- Zelta Fitz (Scheduler)

Then:

- Permission Set = blank; “Manage Clinician Contracts” has been removed (included in Clinician Onboarding)

- Permission Set Group = “Corporate Support” & “Clinician Onboarding”

## Solution

- See Security Matrix:

** Tab = User Setup

*** Liz Bennet (Clinician Experience Specialist)

*** Zelta Fitz (Scheduler)

** Tab = Permission Set Groups

*** Clinician Onboarding

- Remove permission set 'Manage Clinician Contracts' from User

Add permission set 'Manage Clinician Contracts'  to permission set group Clinician Onboarding

- Ensure permission set group Clinician Onboarding is assigned to user
