## Description

As a Recruiter, when I need to create a new contract based off of a Clinician’s existing contract, then I want the system to clone all rates except for Bonus rate & clone the “Initial Effective Date”, so that I can ensure the bonuses are added back on as needed and I can see the initial effective date.

## Acceptance Criteria

Given that I am a Recruiter in the Corporate Support apps

When I navigate to a **Clinician** record

And the Clinician has an Executed Contract

Then I can click on the **Contract Request** button 

And the system determines options to display

And user selects “**Start From Previous Active Contract**”

And upon the user completing the guided process, then upon Finish, the system shall:

** Clone Clinician Contract & rates, **EXCEPT for Bonus rates* (clone: contract, PNC and rate records. Do not clone HPF)

** Include Cloning the new “Initial Effective Date”

- Direct the user to the newly created Clinician Contract

- Appropriate records are created per the solution

Given that I am a Recruiter in the Corporate Support apps

Recruiter in the Corporate Support apps

When I navigate to a **Clinician** **Contract** record

Then if “Initial Effective Date” field **is populated**, than I **can NOT edit**. Error Message: `The 'Initial Effective Date' field cannot be edited. Please contact your administrator for further assistance.`

And if “Initial Effective Date” field is **NOT populated**, than I **can edit**.

NOTE: Bonus will live on the original contract it was added to, so that we don’t restart the bonus effective date when a new contact is created.

## Solution

Trigger : Contract Request OS > Start from Previous active contract

Condition : If executed contract already exists,

Clone Functionality : 

Query the Contract, PractitionerNetworkContract, PreventiveCareAgreement, Contract Payment Agreement. EXCEPT : PCA where Type__c = Bonus and its child records.

Clone InitialEffectiveDate from Contract record, if it is populated.

Save the new Contract record and redirect the OS to the newly created Contract record.

On the UI, Validation rule to prevent the Editing of Initial Effective Date if it is already populated
