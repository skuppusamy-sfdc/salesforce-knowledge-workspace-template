## Description

As a Clinician Experience Team member, I need to create tickets for clinicians, so that they can receive timely support once they raise an issue

## Acceptance Criteria

Given I am a Clinician Experience Specialist in the Corporate Support apps

When I navigate to a clinician record

Then I am able to select Create CET Inquiry Case in the header

And I am able to populate the following:

- Subject (required)

- Description

- Case Owner

And upon save, a case record is automatically created with the following: 

- Case Record Type = Support

- Case Owner = user who created the record

- Status = Not Started

- Type = CET Inquiry

- Contact = (Clinician)

As a Clinician Experience Specialist in the Corporate Support apps

When I open the Cases tab

Then I am able to see the following list view:

- Open CET Inquiry Cases

## Solution

Create a new button on Person Account (Clinician)

The button launches a record create action, which has the following elements

Subject: Text box

Description: Long text

Owner : Auto-populate current user 

On submit, Create Case

- Case Record Type = Support

- Owner = user who created the record

- Status = New

- Type = CET Inquiry

Create list view on case , ‘Open CET Inquiry Cases’, 

Filter : 

Type : CET Inquiry

Record Type : Support

status != Completed AND status != cancelled

Create new record type for case. and LRP for Case (see DD tab = Case (Support)) 

Layout:

- Header

** Subject is displayed in the top line

** Additional fields in highlights panel below: Priority, Type, Case Number

- Main Region:

** Tabs component: Details (default)

*** Details tab: fields are displayed according to data dictionary
