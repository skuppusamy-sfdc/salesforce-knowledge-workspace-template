## Description

As a SCP Corporate Support user, I need to receive an email when a clinician is inactivated through automation or an offboarding request, so that I can action this update accordingly

## Acceptance Criteria

Given I am a Corporate Support team member

When a HPF record is updated so that Status = Inactive 

Then I receive an email with a link to a Salesforce report 

frequency = daily

recipients:

- [QGendaCredentialingDQ@scphealth.com|mailto:QGendaCredentialingDQ@scphealth.com] (this is a production email)

- [eeinactivations@scphealth.com|mailto:eeinactivations@scphealth.com] (this is a production email)

- [inactivations@scphealth.com|mailto:inactivations@scphealth.com] (this is a production email)

- Account Team Medical Staff Coordinator

- Account Team Clinician Experience Specialist

- Account Team Provider Coverage Coordinator

- Account Team Medical Staff Liaison CPA

- Account Team Recruiter Primary

report includes the following columns: Account, Practitioner, Termination Reason, Healthcare Practitioner Facility, Staffing Team (Account), Professional Title (Clinician), Facility Title (HPF), Last Date Worked (HPF), Company Signed Date (Contract), Hospital Privileged Date (HPF), First Scheduled Date (HPF), NPI (Clinician), Termination Date

report is grouped by Account (sort with alphabetical A at the top)

secondary sort by Practitioner Full Name

report is filtered by Termination Date = yesterday 

Email Details: 

- Subject: HPF Inactivation [yesterday’s date]

- Body: 

Hello, 

The following clinicians have been inactivated. Please navigate to the following report for additional details. 

For any questions or concerns with the data listed, do not reply to this email. Reach out to DL.SalesforceDQ@SCPHealth.com

## Solution

Create report on HPF, with the fields mentioned and filter on Status = Inactive and Termination Date = YESTERDAY. Group report by Service account and sorted by practitioner name.

Flow : 

Object: HPF  

Query for HPF where status = inactive. If there are atleast one record, then send email action, else do nothing. 

Scheduled Flow : runs daily once.

Use custom metadata to store the emails, ( ; separated/ , separated) so its easier to use in the email list.

Get the account team members by the roles specified and their email address.

Send email action with link to the report.
