## Description

As a VPO, I need to initiate leadership changes for Service Accounts, so that I can ensure leadership personnel are in the correct roles at the correct locations

## Acceptance Criteria

Given I am a VPO (VP Operations) in the Corporate Support apps

When I navigate to a Service Account record

Then I am able to select Leadership Change in the highlights panel

And I am presented with a guided screen to fill out the following:

- Adjust Current Leadership

** Date selection

** Then, display a list of HPFs for this Service Account where status does not equal Inactive, Effective From and Effective To dates are within the date selected, and Facility Title = Assistant Medical Director, EE Assistant System Medical Officer, EE Asst Hospital Med Program Director, EE Hospital Medicine Program Director, EE Lead Midlevel Provider, EE System Medical Officer, Interim Medical Director, or Medical Director

*** Columns to Display: Practitioner, Facility Title, Effective From, Effective To, Status

*** If no HPFs meet the criteria, display the following “No leaderships HPFs are available for the date selected. Proceed forward to add a new leader.”

*** _Note: usually a Service Account will only have one HPF that meets these requirements but two is possible (ex. Interim Medical Director AND Medical Director)_

** Optional to select what change is needed to any of the HPFs - Leaving SCP, Leaving Facility, or Title Change

*** If Leaving SCP: assign case to Account Team Clinician Experience Specialist

*** If Leaving Facility: assign case to Account Team Clinician Experience Specialist

*** If Title Change: assign case to Data Quality Queue

** Based on what’s selected, create a case (RT: Leadership Change) with the following:

*** Subject: Service Account Leadership Change - [Leaving SCP or Leaving Facility or Title Change based on what was selected]

*** Case Owner: described above

*** Contact Name: [Selected Clinician/Practitioner]

*** Type: [“Leaving SCP”, “Leaving Facility”, or “Title Change” based on what was selected]

*** Account Name: [Service Account]

*** Healthcare Practitioner Facility: [select HPF]

- Add New Leader

** Display a list of Facility Titles (from Code Set records listed above) for user to multi-select which to add for this service account

** If code set(s) are selected: specify Type: Existing Clinician, New Hire

*** If Existing: assign case to Account Team Clinician Experience Specialist

*** If New Hire: assign case to Account Team VP Operations

** Based on what’s selected, create a case (RT: Leadership Change) with the following:

*** Subject: Service Account Leadership Change - [New Hire or Facility Title Change] for [Facility Title] 

*** Case Owner: described above

*** Type: [“New Leader  New Hire” or “New Leader  Existing Clinician” based on what was selected]

*** Account Name: [Service Account]

NOTE: Assumption that the Account Team role = “Clinician Experience Specialist” & “VP Operations” will be setup on the Service Account

## Solution

On Service Account - Create a new button that launches a omniscript (error message if user is NOT a VPO: Only the VP Operations can initiate this change. )

Omniscript - Display two options as radio button

Adjust Current leadership

Add New Leader

Next button

Based on selection,

**Adjust Current Leadership**

**Display** Date field

Query the HPF

select id, EffectiveFrom, EffectiveTo, Account.Name, Practitioner.Name, Status__c from HealthcarePractitionerFacility where Status__c != 'Inactive' and AccountId = <CurrentServiceAccount> AND <DateAbove> >= EffectiveFrom AND <DateAbove> <= EffectiveTo and Facility_Title__r.Name <filter>

If no records found display message “No leaderships HPFs are available for the date selected. Proceed forward to add a new leader.”

no further action - user has to cancel/exit the screen.

If records found, **Display** the HPF in table format with HPF records as rows, and the following fields as columns:

Practitioner, Facility Title, Effective From, Effective To, Status, Change

Change options (picklist): Leaving SCP, Leaving Facility, or Title Change, 

Action - Create case, based on selection in Change field, (for each HPF record ) – **see AC for mapping**

Subject: Service Account Leadership Change - [Leaving SCP or Leaving Facility or Title Change based on what was selected]

Case Owner:

- If Leaving SCP: assign case to Account Team Clinician Experience Specialist

- If Leaving Facility: assign case to Account Team Clinician Experience Specialist

- If Title Change: assign case to Data Quality Queue

Use Default values for:

- Status: “Not Started”

- Case Origin: “Manual”

- Priority: “Medium”

**Add New Leader**

Query the Code Set for the Facility Titles mentioned in the AC (Facility Title = Assistant Medical Director, EE Assistant System Medical Officer, EE Asst Hospital Med Program Director, EE Hospital Medicine Program Director, EE Lead Midlevel Provider, EE System Medical Officer, Interim Medical Director, or Medical Director)

Display the list as mulit-select picklist

Display picklist Type : Existing Clinician, New Hire

Create case, for each facility title selected – **see AC for other mapping**

Subject: Service Account Leadership Change - [New Hire ] for [Facility Title]

assign case to Account Team VP Operations

OR

Subject: Service Account Leadership Change - [Facility Title Change] for [Facility Title]

assign case to Account Team Clinician Experience Specialist

Use Default values for:

- Status: “Not Started”

- Case Origin: “Manual”

- Priority: “Medium”
