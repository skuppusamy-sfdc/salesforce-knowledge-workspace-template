## Description

As a Recruiter, I need contract dates to be updated automatically, so dates follow specified business rules

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I request a contract for a clinician who already has one executed and active (no end date or end date is not in past)

Then the end date for the already executed & active contract is populated with the new contract’s start date - 1 day

-also update if the End Date on the active contract is already populated and exclude the end date update if the Employment Type/Facility Title has not changed-

Note: the New Contract must be in Status = Executed before setting the End Date on the old contract.

## Solution

Record Triggered Flow on Contract, When a new Contract is Executed, and there is an existing contract that is executed,

If Contract.CompanySignedDate >= Contract.StartDate

If Existing Active Contract. End Date is populated and Employment Type of old contract = Employee and New Contract is IC or vice versa, then do the End date update on old contract.

update the Existing Active contract’s End date to newly created contract’s Start date - 1. Use the PreviousContractId to query the Executed contract.

As part of this , update the Old Contract’s status to Cancelled and CancellationReason = Terminated

For scenarios where old contract where EE and new Contract emp type is IC, and vice versa, EE<-> IC

If New IC Contract has PreviousContractid - Use that Id to update the Contract

If New IC Contract does not have previousContractId  Query for Employee Contract with No End Date or EndDate in future  and use that Contract to update if If Contract.CompanySignedDate >= Contract.StartDate

-Scheduled Flow on Contract (to update Contract status)-

-When previous Contract End date = today, then update executed Clinician Contract status to “Canceled”, Cancelled Reason = Superseded-

-When Contract status becomes Executed ( SignedDate is populated), then if there are existing active contracts with End Date in the past, then update the Contract status to Cancelled and Cancel Reason as Terminated.  same employee type , Yes.

-Scheduled flow on Contracts, Check if a contract EndDate = Today and Contract status != Cancelled and Query for Executed Contracts of the same Employee Type for that Clinician. If its exists, then cancel the contracts.-
