## Solution

On the existing, Contract request omniscript - (when a new contract for clinician, it displays HPFs for selection and the Add HPF omniscript). Check if the Employment_Type__c is Employee Time_Type__c is Full time, and clinician Professional_Title__c title is NP or PA.

Query the NPPA_Tier__c from the SA’s Contract. ( via HPF) (Query for Contract of rec type practice operation for AccountId = HPF.AccountId

Get NPPA Tier from the Client Contract (for that Clinician, Clinician > HPF> SA, Query  Practice Operations contract of status executed (there should be only one , Order by LastModifiedDate DSC)

if all of the contracts for selected HPF have the same value in the NPPA_Tier__c field, then display that field as read only.

If they do not have the same value,

Display Message “Upon Submit Contract Request, then an approval will be sent to review NP/PA Tier mismatch“ 

Display a picklist with the drop down value of the selected HPF’s client Contract ( from query above), so that the user selects it. 

Map it to the Clinician contract’s WaitingForApproval_NPPA_Tier__c field. This should be added to DD. Will not be displayed on the layout. 

When Contract status = Submitted or as part of the Submit contract flow, Create a approval record ( the existing process for Clinician escalation), and assign to Account team regional director of nppa services.

After Approved, stamp the value from WaitingForApproval_NPPA_Tier__c to NPPA_Tier on clinician contract.
