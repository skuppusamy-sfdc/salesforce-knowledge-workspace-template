## Description

As an Active Clinician, I need to login to the Health Cloud portal through SSO, so that if I have been provisioned Okta (SSO) then I can easily login without using personal email and password to login

## Acceptance Criteria

Given I am an Active Clinician where Federation ID on my user record is populated

When I navigate to the HC Portal login screen

Then I see the following sign in options:

- Standard User name & Password

- Sign in with Okta

And if I am an Active Clinician with a Federation ID, then I can click on Sign in with Okta to access the HC Portal

And if I am **not** an Active Clinician with a Federation ID, then I am unable to Sign in with Okta

for testing: confirm that login does not work, Okta testing will be done separately
