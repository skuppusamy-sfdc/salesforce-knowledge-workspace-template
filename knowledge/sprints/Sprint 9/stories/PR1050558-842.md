## Description

As a Corporate Support user, I need to log tickets for IT support, so that I can have Salesforce be uptodate with latest business requirements

## Acceptance Criteria

Given I am a Corporate Support user in the Corporate Support apps

When I click on the IT Support global action

Then I am able to populate a case record with the following:  (note fields that do not say “user input” are set by the system)

- Case Record Type = Support

- Case Owner = IT Support Queue

- Type = IT Support

- Status = Not Started

- Priority = Medium (default)

** Subject  (required upon creation) = <**user input*>

** Description (required upon creation) = <**user to input issue*> 

And upon save, a case record is automatically created

As an IT user in the Corporate Support apps

When I open the Cases tab

Then I am able to see the following list view: 

- Open IT Support Cases

** Filter: Case Record Type = Support, Status ≠ Closed, Assigned To = IT Support Queue

** Fields to display: Case Number, Subject, Status, Priority, Owner Name, Date/Time Opened

NOTE:  If decision is to continue with Service Now process, we can add a link to create a ticket in the home page

## Solution

Ensure IT Support Queue exists, if not create one for Case object.

Add IT Support to Case.Type  if it does not exist

Create Global Action  Create record  Case. Record Type : Support

Use Before trigger flow on Case, to check the Type = IT Support and assign to IT Support queue

Configure layout

Add the action to global publisher layout.

Create list view for Case 

Open IT Support Cases

- Filter: Case Record Type = Support, Status ≠ Closed, Assigned To = IT Support Queue

- Fields to display: Case Number, Subject, Status, Priority, Owner Name, Date/Time Opened

Check sharing permission for Case, as IT user they should be able to view the case records (Clinician Change Support RT)

Ensure Corporate support user - has create case permission

** Add “Read” & “Create” for Case Record Type = Support to Permission Set = Corporate Support. (see security matrix, Permission Set tab, for Object = Case)

** Add Case Record Type to System Administrator Permission Set

** Update FLS for Permission Set = Corporate Support & System Administrator (as needed)
