## Description

As a MSL, I need clinicians to keep their First Clinical Shift and Credentialling Status when a new HPF is created for the same facility, so that accurate data can be tracked in Salesforce

## Acceptance Criteria

Given I am a MSL in the Corporate Support apps

When there is an existing Healthcare Practitioner Facility record for a clinician

And a new HPF record is created where:

- the Practitioner and Account fields have the same value as the existing record AND

- there is no gap between the Effective From of the new HPF and the Effective To of the original HPF

Then I am able to see that the First Clinical Shift (QGenda WF SOR), Credentialling Status (QGenda Cred SOR), Temporary Inactive Reason (QGenda Cred SOR), First Scheduled Date (Salesforce SOR), Hospital Privileged Date (QGenda Cred SOR), Hospital Privilege Expiration Date (QGenda Cred SOR) fields have the same value as the existing HPF record

And fields that are QGenda SOR can continue to be updated on the current HPF via integration as needed

Given I am a MSL in the Corporate Support apps

When there is an existing Healthcare Practitioner Facility record for a clinician

And a new HPF record is created where:

- the Practitioner and Account fields have the same value as the existing record AND

- there is no gap between the Effective From of the new HPF and the Effective To of the original HPF AND

- Facility Title changes from Employee to Independent Contractor OR Independent Contractor to Employee

Then I am able to see that the Certificate of Insurance Start (date), Certificate of Insurance Expiration (date), and Insurance Provided By (picklist, values: copy from Corp org) fields have the same value as the existing HPF record

- Note for testing: 

** Locums/Leased to EE or IC  COI should be blank  JM to check onboarding task sheet and SF to create enhancement story if not covered

** IC/EE to Locums or Leased - COI should be blank

## Solution

New fields to be created: Certificate of Insurance Start, Certificate of Insurance Expiration, and Insurance Provided By, Hospital Privilege Expiration Date 

FSL : MSL persona 

Record triggered flow on HPF (new record creation)

Retrieve existing HPF records for the Clinician

Check if the practitionerId and accountid have the same value

Check if the End date of existing HPF is the one day before the  Start date of new record (HPF)

If so, copy the values for the following fields from existing HPF record to new HPF record.

If more than one matching record exits, use the latest HPF that matches.

First_Clinical_Shift__c, Credentialling_Status__c , Temporary_Inactive_Reason__c, First_Scheduled_Date__c, , Hospital_Privileged_Date__c,   Hospital Privilege Expiration Date 

Retrieve existing HPF records for the Clinician

Check if the practitionerId and accountid have the same value

Check if the End date of existing HPF is the one day before the  Start date of new record (HPF)

Get HPF that match the above criteria, if more than one, use the latest record.

If the Facility title on the newly created record is IC , and existing record is employee, or vice versa, 

then , update the following fields on the new HPF with the same values as the matched HPF record.

Certificate of Insurance Start, Certificate of Insurance Expiration, and Insurance Provided By
