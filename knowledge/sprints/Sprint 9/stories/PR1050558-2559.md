## Description

As a Recruiter, I need to specify accurate data in clinician contracts, so that they can be contracted with the correct information

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I request a clinician contract (contract creation guided experience = Contract Request)

Then I am able to specify: (same screen as Employment Type selection)

- Time Type (picklist, required)

** Values: Not applicable, Full Time, Part Time, PRN

** If Contract Employment Type = Locums, valid value(s): “Not applicable”

** If Contract Employment Type = Leased, valid value(s):  “Not applicable”

** If Contract Employment Type = Employee, valid value(s):  “Full Time”, “Part Time”

** If Contract Employment Type = Independent Contractor, valid value(s): “Full Time”, “Part Time”, “PRN” 

** If Contract Employment Type = Partner, valid value(s): “Full Time”, “Part Time”, “PRN”  

** If Contract Employment Type = Managed, valid value(s):  “Not applicable”

## Solution

Add ‘Not Applicable’ to the picklist Time Type field.

On the existing submit Contract request button/omniscript on clinician record, 

Add Time type picklist value on the first screen

Based on the Employee Type selected, the Time Type value is defaulted / pre-populated

- If Contract Employment Type = Locums, set to “Not applicable”

- If Contract Employment Type = Leased, set to “Not applicable”

- If Contract Employment Type = Managed, set to “Not applicable”

- If Contract Employment Type = Employee, “PRN” & “Not applicable” is not a valid value

- If Contract Employment Type = Independent Contractor, could be any value except “Not applicable”

- If Contract Employment Type = Partner, could be any value except “Not applicable”

//Create a Validation Message (if needed)  “The entered Time Type value is not a valid entry for Employee”
