## Description

As a SFO, when the Pro Forma Model has been confirmed, I want the system to automatically generate the Contract (Record Type: Practice Operations), so that the appropriate individual(s) will have a record(s) available to load the shift map (shifts) & rates to.

## Acceptance Criteria

AC1: Auto create **Contract**(s) [Record Type: Practice Operations]

Given I am a SFO within the Corporate Support Apps

When the Pro Forma Model Status = Approved for Contracting

Then the system shall automatically create **one Practice Operations Contract per Opportunity Service** in Status = Active

1. Record Type = Practice Operations

1. Owner = <Opportunity Owner>

1. Status = Draft

1. Account Name = <Service Account>

1. Source Opportunity = <Opportunity Related to the Pro Forma>

## Solution

Create a Record Triggered Flow in Pro_Forma_Model__c Object 

Event: When a Record is Updated to the Status__c = Approved for Contracting

Create Contract Records 

Query all Opportunity Service Record(s) for the  Opportunity 

Services__c.Opportunity__c = Pro_Forma_Model__c.Opportunity__c AND Services__c.Status__c = Active

Create Contract Record(s) based on all Opportunity Service Record(s) that are Active Status with Fields

- Contract Object

** Record Type = Practice Operations

** Owner = Pro_Forma_Model__c.Opportunity__r.Owner

** Status = Draft

** Account Name = Services__c.Account__c

** Source Opportunity = <Opportunity Related to the Pro Forma)

-Security Matrix:-

- tab = Permission Set

** Permission set = Corporate Support, Add Read to Services

** 11/3 - per Sakthi, we do NOT need read access on Services under the Corporate Support perm set. Removed from Security Matrix
