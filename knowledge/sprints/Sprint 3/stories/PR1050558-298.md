## Description

As a SFO (Staffing Fulfillment Optimization) user, I want the ability to generate facility rates by designating the type of rate and specified rate for each Shift Rate & Facility Title combinations, so that we have clearly defined rates for each Service Account.

## Acceptance Criteria

Given I am a SFO in the Corporate Support or Corporate Support Console app

When I navigate to an **Account** record (record type = Service Account) 

Then I have the ability to click the **Agreement Request** button

And I am presented with a Rate Type screen to capture:

- Type of Rate (Contract Payment Agreement, field = Type)

** Required to select one from the following list of values to proceed to the next screen:

*** Stipend

**** **Shift*

*** Variable

*** Target

*** Bonus

*** Locums Travel

*** Locums On Call

Then when I select Shift, I see the Rate Details screen and I have the following option to:

- Go back to the previous screen (button = Previous) 

** And if I selected go back to the previous screen (button = Previous), then user is returned to the Type of Rate screen with the previously selected value

- Populate the following to proceed:

** Shift Type (Code Set; Code Set Type = Shift)

*** System to display all shifts where Active = true

*** Users need to see: Shift Name, Shift Start Time, Shift End Time, Shift Day of the Week, Shift Type

*** Required, MultiSelect; user shall have the ability to select multiple values.-

** Facility Title (Code Set; Code Set Type = Facility Title)

*** System to display all Facility Title where Active = true

*** Required, Multi-Select; user shall have the ability to select multiple values.

** Rate

*** Required, editable Currency field (18,0); user to capture whole number

*** Unit of Measure; per Metric, % Salary Reduction, per Monthly Tier, per Hour, per Shift, per Month

And upon selecting button = Finish, then the system shall create the appropriate Contract, Contract Payment Agreement, and Preventive Care Agreement records as specified in the solution

The OmniScript is closed automatically and I am navigated to the newly created Contract record

**_If user selects any option besides Shift on the Rate Type screen, then they receive an error message and have to exist the guided experience. Other Rate Types will be configured in a separate story._**

## Solution

unit of measure? object or **picklist** (updated in data dictionary)

And upon selecting button = Finish, then the system shall:

- Create a Contract (record type = Client) record

** Status = Draft

** Account Name = <Service Account record>

** _do not create duplicate Contract if one exists in Draft status_

- Create a Contract Payment Agreement record related to the Contract (Client) record.

** Name = Account name

** Status = Draft

** Contract = <Contract (record type = Client) record created above>

** Account = <Service Account record>

** Agreement Type = Preventive Care Agreement

** Type = <Type of Rate selected by the user>

** _do not create duplicate Contract Payment Agreements in Draft status for the same Type_ 

- Create Preventive Care Agreement record(s) related to the Contract Payment Agreement record; create one per Facility Title & Shift Type captured by the user.

** Facility Title = <Facility Title selected by the user>

** Shift = <Shift Type selected by the user>

** Rate = <Rate inputted by the user>

** Bonus Measure Type = NA

** Target Measure Percent = 0

** Bonus Payout = 0
