## Description

As a SFO (Staffing Fulfillment Optimization) user, I want the ability to generate facility rates by designating the type of rate and specified rate for each Shift Rate & Facility Title combinations, so that we have clearly defined rates for each Service Account.

## Acceptance Criteria

Given I am a SFO in the Corporate Support or Corporate Support Console app

When I navigate to a **Service Account** record (Account record type = Service Account) 

Then I have the ability to click Specify Rates button

And I am presented with Type of Rate selection  configured in [https://salesforce.atlassian.net/browse/PR1050558298|https://salesforce.atlassian.net/browse/PR1050558298|smartlink] 

Then when I select Shift, I see the Rate Details screen  configured in [https://salesforce.atlassian.net/browse/PR1050558298|https://salesforce.atlassian.net/browse/PR1050558298|smartlink] 

And if I select to proceed, then the summary screen is displayed 

Then I have the ability to view a summary of existing Shift Type, Facility Title, and Rate combinations that are relate to the Service Account

And I have the ability to view a summary of new Shift Type, Facility Title, and Rate combinations before I finish the workflow

And on the Summary screen, I have the ability to “Save and Continue”, where I am brought back to the Rate Details screen for the previously selected Type of Rate and able to specify additional Shift/Facility Title/Rate combinations

And upon selecting button = Finish, then the system shall create the appropriate Contract, Contract Payment Agreement, and Preventive Care Agreement records as specified in the solution

## Solution

when Service Account has a Contract (record type = Client) in Status = Draft AND Contract Payment Agreement in Status = Draft, then display related Preventative Care Agreements

And upon selecting button = Finish, then the system shall:

- Create a Contract (record type = Client) record

** Status = Draft

** Account Name = <Service Account record>

** _do not create duplicate Contract if one exists in Draft status_

- Create a Contract Payment Agreement record related to the Contract (Client) record.

** Name = Account name

** Status = Draft

** Contract = <Contract (record type = Client) record created above>

** Account = <Service Account record>

** Agreement Type = Preventive Care Agreement

** Type = <Type of Rate selected by the user>

** _do not create duplicate Contract Payment Agreements in Draft status for the same Type_ 

- Create Preventive Care Agreement record(s) related to the Contract Payment Agreement record; create one per Facility Title & Shift Type captured by the user.

** Facility Title = <Facility Title selected by the user>

** Shift = <Shift Type selected by the user>

** Rate = <Rate inputted by the user>

** Bonus Measure Type = NA

** Target Measure Percent = 0

** Bonus Payout = 0
