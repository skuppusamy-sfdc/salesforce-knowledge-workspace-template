## Description

As a Medical Staff Liaison, Enrollment Specialist, Data Quality team member, I need to create Onboarding cases, so that I can begin clinician privileging before contract execution if needed

## Acceptance Criteria

Given that I am a Medical Staff Liaison, Enrollment Specialist, Data Quality team member in the Corporate Support App

When I navigate to a **clinician** person account record

Then I can click on the **Initiate Early Onboarding** button

Then I am presented with the following:

- +Check if HPF is created+: if this clinician does not have a HPF record where Status = Prospect, then I am presented with the following message: “This clinician does not have a facility where they are a Prospect. We cannot initiate early onboarding.”

And I am able to populate the following: 

- What onboarding type needs to start early?

** Facility specific, SCP, or both

- +If facility specific or both+: select from a list of Healthcare Practitioner Facility records (display Status field) without existing onboarding cases 

- +If SCP or both+: there can only be one open SCP onboarding case at a time

- +For all+: indicate priority for SCP onboarding and each selected HPF (default to Medium)

And upon clicking Finish, the system should direct the user back to the Clinician record. No case creation at this point, it will be covered under: [https://salesforce.atlassian.net/browse/PR1050558562|https://salesforce.atlassian.net/browse/PR1050558562|smartlink] , [https://salesforce.atlassian.net/browse/PR1050558622|https://salesforce.atlassian.net/browse/PR1050558622|smartlink] , [https://salesforce.atlassian.net/browse/PR1050558610|https://salesforce.atlassian.net/browse/PR1050558610|smartlink] , [https://salesforce.atlassian.net/browse/PR1050558621|https://salesforce.atlassian.net/browse/PR1050558621|smartlink]

## Solution

Clinician Onboarding case record type

Account is populated with Service Account record from Healthcare Practitioner Facility lookup

<10 fields to be created

Action to create case

Priority is on the case.

Note: access to create Cases with the specified record types are managed at the permission set level

This guided experience is ONLY for Creation cases, no updates to open cases.
