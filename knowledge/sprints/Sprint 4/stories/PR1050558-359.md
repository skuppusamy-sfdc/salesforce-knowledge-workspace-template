## Description

As a Corporate Support user, I want the ability to create and view a new Task record (Task record type = “Default”), so that I can create and track work needed to support the Client & Clinician workflow process.

## Acceptance Criteria

Given that I am a Corporate Support user using the Corporate Support or Corporate Support Console app

When I view the navigation menu

Then I am able to see the following tabs: Home, Accounts, Contacts, Opportunities, Contracts, **Tasks**, Cases, Reports, Dashboards

Given I am Corporate Support user in the Corporate Support or Corporate Support Console app

When I create a new task record

Then the system shall set the task record type to “**Default**” - _user should not be able to select a task record type_

And I am able to create a record with the fields specified in the data dictionary

Given I am Corporate Support user in the Corporate Support or Corporate Support Console app

When I view a Task with record type = “**Default**”

Then I am able to see:

Header with the Subject only

Section with tabs: Details and Files

Details tab (default) contains fields specified in the data dictionary.

Files tab contains standard Files related list

## Solution

create Default record type on Task object

clean up record type assignment on Corporate Support Group permission set

LRP for tasks - assigned to Corporate Support, System Administrator, and Core HC Customer Community Plus Login User profiles
