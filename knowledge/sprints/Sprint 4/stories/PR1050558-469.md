## Description

As a System Administrator, I need necessary fields to be created, so that integrations and functional processes can run smoothly with necessary information

## Acceptance Criteria

Given I am a System Administrator

When I view objects and page layouts for the following objects:

- Person Account (Clinician)

** add the following fields: Military Status, Military Service Start Date, Military Service End Date

** Also update Clinician layout so that Professional Title is required (already required in Guided Clinician Creation)

- Case (Clinician Onboarding)

** add the following fields: Parent Case, Closed Reason

** Update fields: Status

- Account (Service Account)

** add the following fields: Insurance Provided By, Insurance Effective Date, Insurance Expiration Date, Global Facility ID

** add the following field (not on layout): Global Account ID

- Code Set

** add the following fields: Code, Can Have Stipend?, Can Have Bonus?, Can Have Leadership Incentive?

** update Code Set Type picklist values match data dictionary

- Healthcare Practitioner Facility

** add the following fields: Practice Type

** add help text to: First Scheduled Date, First Clinical Shift

- Contract Payment Agreement

** Update values for Field: Type

- Preventive Care Agreement

** add the following fields: Shift Full Name, Shift Iteration

** update existing field: “Shift” change to “Type” (updated label, name and lookup filter)

Then I can see that the fields and page layouts match what is specified in the data dictionary

## Solution

- All fields listed above should be made visible on permission sets that have object access.

- Subtask to update/add code sets.
