## Description

As a Data Quality team member, I need to create a “Contract” for leased and locums clinicians, so that we can move clinicians from initial data entry to onboarding

## Acceptance Criteria

Given I am a Data Quality team member in the Corporate Support apps

When I navigate to a clinician record

And I am able to select the Contract Request button in the highlights panel

And the following happens:

- Facility Selection screen built in [https://salesforce.atlassian.net/browse/PR1050558893|https://salesforce.atlassian.net/browse/PR1050558893|smart-link] 

- Skip Contract Confirmation built in  [https://salesforce.atlassian.net/browse/PR1050558892|https://salesforce.atlassian.net/browse/PR1050558892|smart-link] 

And User clicks “Finish”

Then the system shall

- Update the selected Healthcare Practitioner Facility records with the “Effective From” date that the user confirmed or updated.

- A Contract record (Record Type = Clinician) is created as defined in the solution

- A Provider Network Contract record is created as defined in the solution

- User shall be navigated back to new Clinician Contract record’s Summary tab.

## Solution

A contract is Locum or Leased if the HPF field Employment_Type__c is. 

ContractRequest OS structure is following:

We modify Clinician_UpdateClinicianHPF to add a new Data Mapper that creates a contract if the Employment_Type__c of the HPF is Locum or Leased. Create as well the Provider Network Contract linking correctly the HPF with the Contract in the newly created PNC.

New introduced components to create Contract,PNCs should be reused in the story 1070 as well.
