## Description

As a Recruiter, I need to request approval for clinician compensation, so that I can offer appropriate pay and incentives for clinicians to join SCP

## Acceptance Criteria

_Story focus on systematically enforcing that the user can only add the additional bonus (types) to one facility/HPF. Split from story_ [https://salesforce.atlassian.net/browse/PR1050558194|https://salesforce.atlassian.net/browse/PR1050558194|smart-link] 

Given that I am a Recruiter in the Corporate Support apps

When I navigate to a **Clinician** record

And completed the **Contract Request** [https://salesforce.atlassian.net/browse/PR10505581070|https://salesforce.atlassian.net/browse/PR10505581070|smart-link] 

And I am on **Clinician Contract’s Summary tab** 

Then the system shall only allow Additional Bonuses (types) to be added to **One** Facility/HPF. 

- The “Type” field that holds the Additional Bonuses values should be optional 

** ONCE a Type (Additional Bonuses) is selected, then Rate and Unit of Measure are Required.

- Once the user has added Additional Bonuses to one Facility, then they are NOT allowed to add to another facility.

- If the user deletes all the Additional Bonuses from a Facility, then they can add Additional Bonuses to a new facility. 

- Help Text: Enter the bonus to the facility at which they are primarily working.

- Error message if the user tries to add additional bonuses to more than one HPF/Facility:  Additional bonuses can only be added to one facility. NOT NEEDED, solution will be to hide Help Text, Type, Rate, Unit of Measure.

## Solution

Remove required from Type field on the omniscript/screen

Use messaging (formula validation) to check if more than one “Additional Bonus” is added and Display Validation message , and prevent the user from adding more than one additional bonus.
