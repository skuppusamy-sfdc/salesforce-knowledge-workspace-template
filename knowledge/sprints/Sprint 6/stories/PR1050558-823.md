## Description

As a Recruiter, I need Healthcare Practitioner Facility records to be automatically created when I associate an applicant to a job, so that I can avoid duplicate work when a clinician is tied to a specific location

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I update the **ATS Stage** field on an **ATS Applicant** record to **Offer**

Then a Healthcare Practitioner Facility record is automatically created with the following attributes:

||**Healthcare Practitioner Facility field**||**value to populate**||

|Practitioner Facility Name|ATS Applicant > Job > Account Name (formula)|

|Practitioner|ATS Applicant > Applicant (lookup - contact)|

|Account|ATS Applicant > Job > Account (lookup - account)|

|Facility Title|ATS Applicant > Job > Facility Title (lookup)|

|Status|Prospect|

|Time Type|ATS Applicant > Job > Planned Status

_On Call and Locums are not on Job but on HPF_|

|Job Applicant|ATS Applicant (Id)|

ATS Applicant object = bpats_ATS_Applicant_c

Job object = bpats_Job_c

_This automation copies information from the Job on the ATS Applicant record. Any changes must be completed prior to changing the ATS Stage to Offer Term Sheet._

## Solution

Create a record triggered flow 

Check if there is a HPF already associated ATS Applicant (bpats_ATS_Applicant_c.Id = HealthcarePractitionerFacility.Job_Applicant__c)

If Exists do Nothing

If Not then Create a HPF record based on mapping in AC
