## Description

As a Medical Staff Liaison, Enrollment Specialist, Data Quality team member, I need to select from a list of open contract, so I have a refined list of Healthcare Practitioner Facility to perform early onboarding.

## Acceptance Criteria

Given that I am a Medical Staff Liaison, Enrollment Specialist, Data Quality team member in the Corporate Support App

When I navigate to a **clinician** person account record

Then I can click on the **Initiate Early Onboarding** button

Then I am presented with the following:

- +Check if a Contract has been requested+: if this clinician does not have a Contract record where Status = New, Draft, Submitted, or Approved, then I am presented with the following message: “This clinician does not have a requested contract. We cannot initiate early onboarding. Please discuss with the recruiter for this clinician’s facility.”

_And I am able to populate the following:_

- _What onboarding type needs to start early?_

** _Facility specific, SCP, or both_

** +_If facility specific or both_+_: select from a list of Healthcare Practitioner Facility records (display Status field) without existing onboarding cases_ **_(filter HPF based on Contract Status = New, Draft, Submitted, or Approved)_*

** If no Healthcare Practitioner Facility record meet the criteria then display the following message: This clinician has an open Facility Onboarding case. Please review existing onboarding case.

- +_If SCP or both_+_: there can only be one open SCP onboarding case at a time_

- +_For all_+_: indicate priority for SCP onboarding and each selected HPF (default to Medium)_

_And upon clicking Finish, the system should direct the user back to the Clinician record_

NOTE: SCP Onboarding message is part of story [https://salesforce.atlassian.net/browse/PR1050558835|https://salesforce.atlassian.net/browse/PR1050558835|smart-link]

## Solution

Initiate Early Onboarding is triggered from OS ClinicianManualOnboardingOS. From what I found current DM only queries Prospect status (this is the ONLY change). We need to add other Status (**_New, Draft, Submitted, or Approved_**)

!Captura de pantalla 20250729 a la(s) 7.53.26 p.m..png|width=50%,alt="Captura de pantalla 20250729 a la(s) 7.53.26 p.m..png"!

Cases are retrieved via following DM (nothing to do here). 

Filter HPF based on: PNC record → Contract status = (**_New, Draft, Submitted, or Approved_**)
