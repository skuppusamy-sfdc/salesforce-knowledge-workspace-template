## Description

As a Sales VP, I need to store outputs from Pro Formas in Salesforce, so that the information can be used for approvals, generating contracts, and history tracking

## Acceptance Criteria

Given I am a Sales VP in the Corporate Support or Corporate Support Console apps

When I navigate to an Opportunity (Growth)

Then I can see the Pro Forma tab

And I am able to create a new record (Pro Forma Model & Pro Forma Facility Detail) with fields as specified in the data dictionary

Given I am a Corporate Support user in the Corporate Support or Corporate Support Console apps

When I navigate to a **Pro Forma Model** record

Then I am able to see the following:

- Header - display Name only

- Main region

** Tabs component: Details, Related

** Details tab displays fields as specified in data dictionary

** Related tab displays standard Files & Pro Forma Facility Detail related list

Given I am a Corporate Support user in the Corporate Support or Corporate Support Console apps

When I navigate to a **Pro Forma Facility Detail** record

Then I am able to see the following:

- Header - display Name only

- Main region

** Tabs component: Details, Related

** Details tab displays fields as specified in data dictionary

** Related tab displays standard Files related list

**NOTE: user’s access to view or create is based on the security matrix.**

- Create Pro Forma Model base on Data Dictionary

** Ensure field visibility based on object permission

- Pro Forma Facility Detail base on Data Dictionary

** Ensure field visibility based on object permission
