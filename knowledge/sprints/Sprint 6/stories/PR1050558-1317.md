## Description

As a Recruiter, I need the clinician contract request to accurately prevent facility titles where employment type is EE/IC are active at the same time, so that our SCP business process rules are enforced.

## Acceptance Criteria

**I Cannot Have EE and IC Contracts Active at Same Time**

Given that I am a Recruiter in the Corporate Support apps

When I navigate to a Clinician record and select Contract Request

Then I am able to select facilities as built in [https://salesforce.atlassian.net/browse/PR1050558893|https://salesforce.atlassian.net/browse/PR1050558893|smart-link] 

When I request a contract for EE when an IC contract is active (and vice versa), then the End Date on the Active IC contract is populated with the Start Date of the EE contract

_Note (built in_ [https://salesforce.atlassian.net/browse/PR10505581389|https://salesforce.atlassian.net/browse/PR10505581389|smartlink]  and [https://salesforce.atlassian.net/browse/PR10505581070|https://salesforce.atlassian.net/browse/PR10505581070|smartlink] _): When a Clinician Contract is created and they already have an active contract, clone all facility associations and their rates for the recruiter to manually adjust as needed_

Active = Status of “Executed”

## Solution

Use Employment_Type to know if Contract will be IC or EE. 

Query ALL the active Contracts for the given Clinician. Per following Diagram we Query using the Person Account Id in Contract object. 

!Captura de pantalla 20250730 a la(s) 9.20.17 p.m..png|width=392,height=222,alt="Captura de pantalla 20250730 a la(s) 9.20.17 p.m..png"!

DON’T USE Clinician__c for this.

Check if Employment Type are different and one is EE and the other IC. 

IF previous condition is true, set the Start Date from new Contract using End Date of Active Contract. 

Create a flow on the Contract Request that runs when the Contract.Status = 'Executed' that updates all Provider Network Contract and Contract Payment Agreements where the Contract is the previous one and set the ContractId to the new ContractId. In order to get the old Contract we query using clinicianId, employment type and the end date should be the start date of the new one. 

In order to CLONE facilities

1. query for all HPF for active Contract. Do this query using PNC. 

1. Remove the HPF Id and,  and save. 

1. Create PNC records, for all the HPF’s from 2. and for the new Contract.
