## Description

**Tester:** Sheila Robson

**Role:** Payroll

**Test Case ID:** Gold Set  Practice Ops shift only change  impact on clinician (Add new shift, cascades down to provider agreements)

**Description:** 

When processed a Practice Operations contract to add a new shift at the facility rate, the cliniciancoverageupdatebatch, it ran (manually triggered) but did not add the new shift to all the providers contract record.  Also, cannot see the HPF under the Client Change Management tab.  

Used Big Ben Regional Medical Center adding Day 2 shift at $250 with effective date of 1/15/26.

**Steps to Reproduce:**

Activate new Practice Operations agreement with new Day 2 shift and rate.  Used 1/15/26 as the effective date.

**Expected vs. Actual:**

Provider contracts would be updated with the new shift and it did not update it.

Not Provided

Fix1: Relax the “Rate Pay Code Validation Rule” in Rate Pay Code to apply only for Practice Operation Contract

Fix 2: The spelling of the contract status in one of the SOQL is wrong correct it

Fix 3: Any one who have Client Change Managment object need to have access to all fields in the object
