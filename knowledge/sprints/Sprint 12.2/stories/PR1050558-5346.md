## Description

As a Clinician Contract Specialist, I need to be able to cancel clinician contracts with the same start dates, so that data can be manually corrected

Given I am a Clinician Contracting Specialist

When I navigate to a clinician contract where the Start Date matches an existing contract for the same clinician AND status is not Canceled

Then I am able to update the status of either Contract record to Canceled

**per story** [https://salesforce.atlassian.net/browse/PR10505585117|https://salesforce.atlassian.net/browse/PR10505585117|smart-link] **team needs to be able to Cancel clinician contracts that are loaded with matching start dates**

## Solution

Flow: Contract Before Save Flow (Same start date error) should be only relevant when Start Date Change add condition to check start date changed

Flow: Case Creation After Flow (Step Update_Contract) should only update Executed Contract)

## Root Cause

The root cause of flow errors was identified as a recursive issue where the flow attempted to update all contracts, including those already canceled, which was fixed by limiting updates to only executed contracts.
