## Description

As a Recruiter, I need to be prevented from creating a clinician contract with the same start date as an existing contract, so that clinician updates are captured appropriately

## Acceptance Criteria

Given I am a Recruiter

When I create a new clinician contract or update an existing contract Start Date

And the clinician already has a contract with the same Start Date (Status does not equal Canceled)

Then I am unable to save and see the following error message: “There is already a contract for this clinician with the same Start Date. Please review and correct the Start Date.”

## Solution

Use Record Triggered Flow Flow  to check  the dates and throw a validation error as per AC

Both Insert and Update
