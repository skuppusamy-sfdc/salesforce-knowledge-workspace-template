## Description

When a clinician’s email address is blank on the Contact, and it is time for them to get experience cloud and/or intake form reminders, there is an error message from a flow error. We need to either add validation that requires an email before they get to the experience cloud site in the flow, or that checks for the user’s email address from experience cloud if Contact email is somehow blank (even though I don’t know how the Contact email could be blank if User is created since it creates automatically from the Contact). In short, it’s a data issue but there is improper error handling and/or validation.

## Root Cause

The issue stemmed from contacts with blank email addresses causing flow errors during reminder processes.
