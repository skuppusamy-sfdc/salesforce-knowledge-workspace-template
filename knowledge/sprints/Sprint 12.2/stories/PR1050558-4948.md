## Description

As a system administrator, when a Clinician Contract is cloned with the same Employment Type (either via ‘Contract Request’ guided experience OR Agentforce), then the system shall clone “Contract_HireDate_Trigger_Ack__c” field value, so that when the new Clinician Contract is Executed we will use this indicator to determine to send trigger for Workday ([+PR10505584861+|https://salesforce.atlassian.net/browse/PR10505584861])

NOTE: the Workday flag should not be cloned if the Employment Type is different/changed & the Qgenda flag should never be cloned (we always want to send to Qgenda).

## Acceptance Criteria

AC1: Workday - Clone flag “Contract_HireDate_Trigger_Ack__c” when Employment Type is the same

Given I am System Administrator

When a **Clinician Contract is cloned** (via ‘Contract Request’ > ‘Start from Previous Active Contract’ OR via Agentforce) with the +**same**+ **Employment Type**

Then they system shall **clone the “Contract_HireDate_Trigger_Ack__c” field** when creating the new Clinician Contract in Status = Draft

AC2: Workday - Do NOT Clone flag “Contract_HireDate_Trigger_Ack__c” when Employment Type is the different

Given I am System Administrator

When a **Clinician Contract is cloned** (via ‘Contract Request’ > ‘Start from Previous Active Contract’ OR via Agentforce) with a +**different**+ **Employment Type**

Then they system shall **NOT** **clone the “Contract_HireDate_Trigger_Ack__c” field** when creating the new Clinician Contract in Status = Draft

AC3: Qgenda - Do NOT Clone flag “Contract_Trigger_Ack__c”

Given I am System Administrator

When a **Clinician Contract is cloned** (via ‘Contract Request’ > ‘Start from Previous Active Contract’ OR via Agentforce) +regardless+ of Employment Type

Then they system shall **NOT** **clone the “Contract_Trigger_Ack__c” field** when creating the new Clinician Contract in Status = Draft

## Solution

Contract_Trigger__chn should fire for all

This should only apply for  Contract_HireDate_Trigger__chn
