## Description

As a Clinician Experience Specialist, when I close my task “Create New Clinician Contract for [Clinician’s Name]', then I want the system to update the status of the related Client Change Management record, so that when users report on the Client Change Management records they know if it was completed or canceled.

## Acceptance Criteria

AC1: Auto update the Client Change Management record Status 

Given I am a Clinician Experience Specialist within the Corporate Support apps

When a Client Change Management record is created with Process Type = Manual and Status = New (original story for task creation [+PR10505581682+|https://salesforce.atlassian.net/browse/PR10505581682]+.+ Related story for additional CCM record creation [PR10505583574|https://salesforce.atlassian.net/browse/PR10505583574])

And I receive a task subject = “Create New Clinician Contract for [Clinician’s Name]” 

And I update the Task Status

Then 

- If Task Status = ‘Completed’, then the system shall update the related Client Change Management record Status = ‘Completed’

- If Task Status = ‘Canceled’, then the system shall update the related Client Change Management record Status = ‘Canceled’

AC2: Update Sharing Setting

Given I am a System Administrator

When I am in Setup

And Object = Client Change Management

Then the Sharing Setting for ‘Contract’ field is set to:

- Read Only: Allows users with at least Read access to the Master record to create, edit, or delete related Detail records.

AC3: Enable History Tracking for ‘Status’

Given I am a System Administrator

When I am in Setup

And Object = Client Change Management

Then History Tracking is enabled for Field = 'Status'

## Solution

AC1: Record Triggered Flow in Object Task, update the Client Change Management records

AC2: Update Sharing Setting in Field

AC3: Enable Tracking History
