## Description

As a user within the Corporate Support app, when a Clinician requires a Facility Title change (New HPF with the same service account as a Active HPF) and a new clinician contract is executed, then I want the system to automatically update the old HPF (Status = Active) record to be inactivated by setting key fields, so that when there is a facility title change there are no Effective date gaps between HPF records and statuses are update to reflect the current state.

## Acceptance Criteria

+AC1: Update Old HPF When New Clinician Contract Executed+

Given I am a Recruiter in the Corporate Support apps

When a clinician contract has a HPF included where the clinician has an active HPF (Status ≠ ‘Inactive’) for that same Service Account (ex. Facility Title change)

Then when the Contract is Executed the following updates happen automatically to the Old HPF*: 

- Contracting Status = ‘Termination In Progress’

- Anticipated Termination Date = <Newly Executed Clinician Contract’s ‘Contract Start Date’ - 1>

And then on the newly executed Clinician Contract ‘Contract Start Date’ (Contract Start Date = Today or in the past), then the system shall update the following fields on the Old HPF*:

- Status = ‘Inactive’

- Contracting Status = ‘Terminated’

- Termination Date = <new contract start date - 1>

*Old HPF record(s) are defined as: the HPF in Status ≠ Inactive for the same Service Account 

- 

** New HPF is defined as: Provider Network Contract (PNC) record related to the Clinician Contract where ‘Deleted?’ = false/unchecked and has the same service Account as the Old HPF.

## Solution

AC1: Use a scheduled flow and move the complex piece to APEX

AC2: To update Status when new HPF Created is Runtime Flow (Key use he HPF.AccountId) Update the field Process_Previous_HPF_Inactivation__c once the contract is processed in the (final method) - Process_Previous_HPF_Inactivation__c will not be updated if the process failed for that contract.
