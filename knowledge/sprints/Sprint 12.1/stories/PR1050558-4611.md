## Description

**Tester:** Nikhil Kokate

**Role:** Recruiter

**Test Case ID:** INT-20

**Description:**

During Clinician Contracting Process, the Contracting Corporate attribute maintained in Salesforce does not align with the corresponding corporate entity value persisted in Workday after integration processing. This results in inconsistent provider master data across source (Salesforce) and downstream system of record (Workday).

**Record Link:**

None Provided

**Steps to Reproduce:**

1.Navigate to clinician record

2.Validate HPF is associated with Valid details

3.Navigate to associated contract and create conga agreement

4.Complete the Docusign approval process and Contract will be marked as executed

**Expected vs. Actual Results:**

Expected result

•	Expected result: For a given provider, the Contracting Corporate value in Workday should exactly match the Contracting Corporate value stored in Salesforce after the integration run. For this record, Workday should display “Schumacher Management” as the Contracting Corporate, aligned to the Salesforce source value.

Actual result

•	Actual result: For the same provider, the Contracting Corporate value in Workday does not match Salesforce. Instead of “Schumacher Management” as in Salesforce, Workday is displaying “NW Florida Emergency Physician, LLC” as the Contracting Corporate, indicating an incorrect corporate mapping or misaligned reference data in the Salesforce–Workday integration.
