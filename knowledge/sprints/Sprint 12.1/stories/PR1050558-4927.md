## Description

Per story 2473 when a Clinician Offboarding case where Type = Facility case is closed (all child tasks are closed), then the following HPF fields are NOT being updated (Ex. UAT2 Offboarding Case 0007484  all task completed, case closed with Last Day  1/8/2026)

- Effective To = Last Day 

- Contracting Status = Termination In Progress

- Anticipated Termination Date = Last Day 

Also, the following HPF fields are not being set based on ‘Effective To’ date:

- Status = Inactive

- Contracting Status = Terminated

- Termination Date = day automation ran

Expected Results:

When a Clinician Offboarding case where Type = Facility case is closed (all child tasks are closed), then the following HPF fields are immediately updated:

- Effective To = Last Day (Last Day from the case record)

- Contracting Status = Termination In Progress

- Anticipated Termination Date = Last Day (Last Day from the case record)

Then, if the HPF’s ‘Effective To’ date is:

** In the **past*, update the following HPF fields immediately:

** Status = Inactive

** Contracting Status = Terminated

** Termination Date = day automation ran

** In the **future** or **current* (if 'Effective To' date = today’s date, that is considered future), update the following HPF fields once the ‘Effective To’ is in the past:

** Status = Inactive

** Contracting Status = Terminated

** Termination Date = day automation ran
