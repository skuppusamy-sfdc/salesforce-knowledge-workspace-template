## Description

As a MuleSoft Integration Developer, I need Salesforce Change Data Capture (CDC) to trigger data transmission to MuleSoft during appropriate times, so that I can perform necessary actions

## Acceptance Criteria

AC1: Add 2 field in SF Contract Object

Given I am a System Administrator

When I am on the Contract Object

Then the following checkbox fields shall be available on for the Clinician Contract record type:

1. Contract_HireDate_Trigger_Ack__c

1. Contract_Trigger_Ack__c

AC2: (QGenda) **Update** the condition for CDC Channel Contract_Trigger__chn

Given I am a System Administrator

When the following conditions are met:

** **Update* the condition for CDC Channel Contract_Trigger__chn

filterExpression: (Status = &apos;Executed&apos; `AND Contract_Trigger_Ack__c = false`)

Then, I need to trigger Contract_Trigger__chn

AC3: (Workday) **Update** the condition for CDC Channel Name: Contract_HireDate_Trigger__chn

Given I am a System Administrator

When the following conditions are met:

** **Update* the condition for CDC Channel Contract_HireDate_Trigger__chn

filterExpression: (Status = &apos;Executed&apos; AND Hire_Date__c != null AND (Employment_Type__c = &apos;Employee&apos; OR Employment_Type__c = &apos;Independent Contractor&apos;) `AND` `Contract_HireDate_Trigger_Ack__c = false`)

Then create a **new** CDC Channel Name: Contract_HireDate_Trigger__chn

NOTE for MuleSoft actions:  adminFlag = startAdministrativeWorkAheadOfClinicianWork

1. **MD :** adminFlag=true, mule to update triggerAckFlag=true & also process the integration

1. **Non-MD:** adminFlag=false & hireDate/firstClinicalShift is NOT NULL, mule will update triggerAckFlag=true & also process the integration.

1. **NonMD:** adminFlag=false & hireDate/firstClinicalShift is NULL, mule will update triggerAckFlag=true & does not process**.  Mule will not receive the event from HC for this scenario.**

## Solution

AC1: Create new field in Contract Object as per Data Dictionary

AC2: Update the CDC event Contract_Trigger__chn with new condition

AC2: Update the CDC event Contract_HireDate_Trigger__chn with new condition

Mule will update the fields respective fields as **TRUE** once they receive the respective event (this change is made to make sure mule received the CDC once once per event per Contract)

1. Contract_HireDate_Trigger_Ack__c

1. Contract_Trigger_Ack__c

Prevent the Contract update record flow from trigger when any of the below fields is set to True

1. Contract_HireDate_Trigger_Ack__c
