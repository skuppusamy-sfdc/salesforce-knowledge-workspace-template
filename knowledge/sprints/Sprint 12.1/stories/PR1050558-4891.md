## Description

C&CUAT285 "No available rates" when trying to configure rates on backdated contract request 

**Tester:** Maci Broussard

**Role:** Recruiter

**Test Case ID:** N/A

**Description:** 

Found defect when trying to set up data for agentforce testing. Confirmed this is a defect with Lisa Kleyla. Appears system is telling me there are "no available rates" when Practice Ops contract record has an end date, but Clinician Contract record has an effective date within the PO start and end dates.

**Steps to Reproduce:**

1. Navigate to Clinician person account, where clinician has HPF for Abbeville General Hospital EM.

2. Create a New Contract Request for 12/1/25.

3. Summary  Configure Rates > Shift

4. Get message "No available rates"

Note: rates for Facility Title are available on PO record, record is just end dated.  But we backdate requests often.

**Record Link:** 

[https://corehc-uat2.sandbox.lightning.force.com/lightning/r/Contract/800bf00000NF4EDAA1/view?ws=%2Flightning%2Fr%2FAccount%2F001bf00000S47vMAAR%2Fview|https://corehc-uat2.sandbox.lightning.force.com/lightning/r/Contract/800bf00000NF4EDAA1/view?ws=%2Flightning%2Fr%2FAccount%2F001bf00000S47vMAAR%2Fview]

**Expected vs. Actual**

Would expect system to display rates from PO (Clinician Contract Start Date falls within start and end date of Practice Ops record)

## Acceptance Criteria

AC1: update PO Contract filter to include Status = Superseded

Given I am a Recruiter user within the Corporate Support Apps 

When I am working on a Clinician Contract is Status = Draft

And I select “**Configure Rates**” button on the Summary tab

Then the system shall pull rates from a Practice Operation contract in Status = “Executed” **or** “**Superseded**”

And same date rules apply for selecting which Practice Operation contract to pull from (Clinician Contract Start Date falls on/within the Practice Operations (PO) contract Contract Start Date).

Example: of current date = `20260106`

PO Contract 1 Start Date `20240101`  End Date `202412-31`, Status = Superseded

PO Contract 2 Start Date `20250101`  End Date `202512-31`, Status = Superseded

PO Contract 3 Start Date `20260101` - End Date BLANK            , Status = Executed

New Clinician Contract Start Date = `20251201`, so user will configure rates based on PO Contract 2 (Start Date `20250101`  End Date `202512-31`, Status = Superseded)

## Solution

Include Status =  Superseded in the PO Contract Query that pulls the Rates
