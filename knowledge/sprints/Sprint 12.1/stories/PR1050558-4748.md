## Description

As a HR employee, I need hire dates to be populated appropriately, so that downstream systems are updated with the proper setup for clinicians

## Acceptance Criteria

**Regular Clinician**

Given I am a Recruiter in the Corporate Support apps

When I generate a contract for a clinician and I **do not select Start Admin Work Ahead of Clinical Work** AND the HPF is updated (automatically) with a First Clinical Shift date

Then the contract record is updated with Hire Date = First Clinical Shift from HPF if it has not already been updated

_example 1: I generate a contract for a MD clinician for Facility A and B where Start Admin Work Ahead of Clinical is NOT selected. At first the Hire Date field is blank. The HPF for Facility A is the first one to receive a First Clinical Shift, so this automatically updates the Hire Date field on the contract. When Facility B receives a First Clinical Shift after Facility A, the contract is NOT updated since Hire Date was already populated._ 

_example 2: I generate a contract for a NP clinician for Facility C and D where those HPFs have already been included in a previous contract. Since the clinician already has a Hire Date, use the Hire Date from the previous contract._ 

**Employment Type Change**

Given I am a HR team member in the Corporate Support apps

When I navigate to a clinician record where:

They have a Contract in Executed status with a previously Executed Contract of a different employment type (ANY employment type change qualifies)

Then their hire date is Contract Start Date (on the new EE contract)

**Start Admin Work**

Given I am a Recruiter in the Corporate Support apps

When I generate a contract for a clinician and I **do select Start Admin Work Ahead of Clinical Work** 

Then I must populate Administrative Start Date

And the Hire Date field is automatically populated with the date value from Administrative Start Date when the Contract changes to Executed Status

_see_ [https://salesforce.atlassian.net/browse/PR10505581721|https://salesforce.atlassian.net/browse/PR10505581721|smart-link]  _for functionality when Admin Start is available to select_

**Contract Field Edit Access**

Given I am a Recruiter in the Corporate Support apps

When I view a contract for a clinician in any Status

Then I am **NOT able to edit** the following fields:

- Hire Date

- EE NP/PA Full Time Weekly Blended

- EE NP/PA Hourly Blended

- EE NP/PA Full Time Overage Rate

- EE NP/PA Full Time Monthly Blended

Given I am a Recruiter in the Corporate Support apps

When I view a contract for a clinician in any Status +and I do not select Start Admin Work Ahead of Clinical Work+

Then I am **NOT able to edit** the following fields:

- Start Admin Work Ahead of Clinical

- Administrative Start Date

## Solution

**Regular Clinician**

Record Triggered Flow for Update in HPF

When HPF.First_Clinical_Shift__c is Updated Check if the Contract.Hire_Date__c = NULL then Update  Contract.Hire_Date__c = HPF.First_Clinical_Shift_Date__c

Query to Identify Contract: Select Id, Hire_Date__c from Contract WHERE Contract.Employment_Type__c = HealthcarePractitionerFacility.Facility_Title_r.Code_Set_Sub_Type_c and HealthcarePractitionerFacility.PractitionerId = Contract.AccountId  and Status = Executed  Order By Start Date DESC LIMIT 1

**Start Admin Work**

Record Triggered Flow for Update in Contract (Fast Field Update)

When the Contract.Status = Executed and Contract.Start_Admin_Work_Ahead_of_Clinical_Work = true then update Contract.Hire_Date__c = Contract.Administrative_Start_Date__c

**Contract Field Edit Access**

Update the Record Page as per AC
