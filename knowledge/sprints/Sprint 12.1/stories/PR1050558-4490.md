## Description

Steps to Reproduce:

1-Login as Legal team member into the org.

2-Navigate to Corporate support apps and open case having record type Client Agreement.

3-Start the Configure Contract flow and navigate to the Agreement category field after populating all the fields.

4Select any value (BAAProvider) and verify all the details and submit it.

5-An error message is displayed.

When verified in DD, for Agreement Category field there are certain active and inactive values and as of now Inactive values are also displayed in the picklist field and hence error is dispalyed upon selection of those values.

Expected Result - Inactive values shouldn't be displayed on Agreement Category field.

Actual Result - Both the values are displayed on Agreement Category field.

## Root Cause

The BAA-Provider was not removed from active pick list values. Fixed that via this BUG
