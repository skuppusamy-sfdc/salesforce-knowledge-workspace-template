## Description

As a Recruiter, I need to pull in HPFs with the same Corporate Entity on clinician contracts, so that clinician contracts are generated appropriately 

**Tester:** Tyler Anderson

**Role:** Recruiting

**Test Case ID:** N/A

**Description:** Clinician had other active facilities tied to the same SCP Contracting entities that did not pull into the contract request.  Recruiter was able to submit the contract request and it went to the approved status without the requirement of including the additional active facilities.

**Steps to Reproduce:**

Create Contract request

Select 2 locations with different corporate entities

configure rates

submit rates

**Record Link:** [https://corehc-uat2.sandbox.lightning.force.com/lightning/r/Contract/800bf00000MXrfBAAT/view?ws=%2Flightning%2Fr%2FAccount%2F001bf00000S4Gr0AAF%2Fview|https://corehc-uat2.sandbox.lightning.force.com/lightning/r/Contract/800bf00000MXrfBAAT/view?ws=%2Flightning%2Fr%2FAccount%2F001bf00000S4Gr0AAF%2Fview]

**Expected vs. Actual:** Shouldn't allow you to pull one location and not the others that have the same corporate entity

## Acceptance Criteria

**automatically add HPFs with same corporate entity**

Given I am a Recruiter in the Corporate Support apps

When I **request a clinician contract** OR **add HPF to a draft contract record**

And the clinician has more than one HPF with the same Corporate Entity where Status does not equal Inactive

Then if I select HPF(s) without selecting all that have the same Corporate Entity, the HPFs that were not selected are automatically added to the contract record

## Solution

Check for the Corp entity for the selected HPF in the Contract during contract process

Get the Selected HPF get the HPF.Contracting_Corporate_Entity__c (create a list)

Query for All HPF for the Clinician where [HPF.I|http://HPF.Is]d Not in (Selected HPF In Contract) and Contracting_Corporate_Entity__c in List from Above And HPF.Status != Inactive 

Add all the HPF to the contract by creating PNC records
