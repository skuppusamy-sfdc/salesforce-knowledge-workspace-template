## Description

As an Implementation Specialist, I want the system create Action Plan Template tasks based on the related Service Account’s Service Line, so that only appropriate tasks get assigned.

## Acceptance Criteria

AC1: Option 2 - Delete tasks based on Service Line

Given I am an Implementation Specialist in the Corporate Support Apps

When I updated the PMT Task Status is set to “In Progress” 

And tasks have been automatically created via the Action Plan Template

And a within a Task’s ‘Applies to Service Line’ text field it contains different service lines separated by commas “<service line>,<service line>_”_ 

Then I want the system will determine if the Service Account’s Service Line value, matches the Service Line(s) listed in the ‘Applies to Service Line’:

** If the system determines that it **does NOT* match, then the system shall delete the task

** If the system determines that it **does* match, then the system shall NOT delete the task

- If 'Applies to Service Line’ field is blank, then do NOT delete.

Example of verbiage in ‘Applies to Service Line’ = CC,CM,CCC

(NOTE: if the Service Account’s Service Line is not populated, then the system will not delete any tasks)

AC2: Ability for the Implementation Specialist persona to create, clone & publish Action Plan Templates

Given I am an Implementation Specialist in the Corporate Support Apps

When an existing task requires a change or a new task needs to be added

Then I shall access to the following actions:

- Clone Template

- Publish Template

- Deactivate Template ( Action Plan Templates(Old) when new Action Plan Templates  is crated)

And when the Action Plan Template is in Status = Draft, then on the Items tab for Tasks I have the ability to:

- Add “New Task”

- Edit

- Delete
