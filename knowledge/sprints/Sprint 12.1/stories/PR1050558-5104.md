## Description

**Tester:** Matt Hebert

**Role:** Clinician Contracting Specialist

**Test Case ID:** N/A

**Description:** Add B for Leadership Roles generated incorrectly. Upon review of data on Clinician Contract Record, the Stipend Tier field was NULL and the Admin Hours field was NULL. We checked the Practice Ops records for Stipend and Leadership Incentives and these values were present.

**Steps to Reproduce:**

1. Navigate to Clinician Contract where there is stipend and leadership incentives configured.

2. Review PCA record and validate Stipend Tier and Admin Hours fields are NULL

3. Navigate to associated Practice Ops/Facility Rate

4. Review PA records and validate Stipend Tier and Admin Hours fields have values

**Expected vs. Actual:** Would expect data to be on Clinician Contract record, as templates use this data for Add B.

**Record Link:** [https://corehc-uat2.sandbox.lightning.force.com/lightning/r/Contract/800bf00000NefiLAAR/view?ws=%2Flightning%2Fr%2FAccount%2F001bf00000UI59dAAD%2Fview|https://corehc-uat2.sandbox.lightning.force.com/lightning/r/Contract/800bf00000NefiLAAR/view?ws=%2Flightning%2Fr%2FAccount%2F001bf00000UI59dAAD%2Fview]

As a Recruiter, I need to Stipend Tier and Admin Hours automatically populated on clinician contracts, so that contracts are generated appropriately

## Acceptance Criteria

Given I am a Recruiter

When I configure rates on a clinician contract

And I select Type = Stipend or Leadership Incentive

Then Stipend Tier and Admin Hours is automatically added to Preventive Care Agreement (from PO contract) once I save the rate

NOTE: Type of Rate = “Leadership Incentive” does NOT have “Stipend Tier” and “Admin Hours” captured at a PO level, these are fields only associated with “Stipend”.

## Solution

Copy the field from PO to Clinician Contract
