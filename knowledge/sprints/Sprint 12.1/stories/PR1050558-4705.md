## Description

As an Integration User, I need to know when an EE clinician is leaving SCP, so that I can remove that clinician from Workday.

Add “Global Provider ID” on the Platform Event, Name = Clinician Offboarding Request, Type = **“Employee”**

## Acceptance Criteria

AC1:

Given I am an Integration User in the HC org

When we have a case record meeting the following criteria:

- Case Record Type = ‘Clinician Offboarding’

- Type = ‘SCP’

- Employment Type = ‘Employee’ (EE)

AND the Case meeting the above criteria is in Status = Closed and Closed Reason = Complete

Then create a Platform Event, Name = Clinician Offboarding Request, Type = “Employee”:

And add “Global Provider ID” (Global_Provider_Id__c)

## Solution

Update the Case flow with the existing platform event (Create or Update a Record Triggers Flow Case for **Update)**

- Condition: Record Type = Clinician Offboarding and Type = SCP is updated so that Status = Closed and Closed Reason = Complete

1. Case > Contract > Employee Type = “Employee” Then

## Create a Platform Event record Clinician_Offboarding_Request__e

### Type = “Employee”

### Employee ID = Case.Contact.Employee_Id__c

### Termination Date = Case.Termination_Request_Date__c 

### Last Day of Work = Case.Last_Day__c

### Primary Reason = Case.Reason_for_Termination__c

### Reason Description = Case.Description

### Pay Through Date = Case.Last_Day__c 

### Rehireable__c = Case.Contact.Rehireable__c

### Global Provider Id = Case.Contact.Global_Provider_ID__c

Add the Global Provider Id Logic
