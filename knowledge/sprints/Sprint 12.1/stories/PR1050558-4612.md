## Description

**Tester:** Nikhil Kokate

**Role:** Recruiter

**Test Case ID:** INT-20

**Description:**

Description: The Salesforce Health Cloud to Workday integration is generating an error "Invalid ID value. '' is not a valid ID value for type = 'Compensation_Plan_ID'" during provider synchronization.

**Record Link:**

None Provided

**Steps to Reproduce:**

1.Navigate to clinician record

2.Validate HPF is associated with Valid details

3.Navigate to associated contract and create conga agreement

4.Complete the Docusign approval process and Contract will be marked as executed

**Expected vs. Actual Results:**

Expected result

•Compensation plan data flows from Salesforce to Workday with a valid Compensation_Plan_ID, completing sync successfully.

Actual result

•Empty Compensation_Plan_ID ('') causes Workday error "Invalid ID value", blocking data flow from Salesforce to Workday.

## Root Cause

Code defect.

The system relies on the facilityTitle field to correctly map to and retrieve a valid Compensation_Plan_ID. The failing records had a facilityTitle value that did not correspond to any valid, configured Compensation_Plan_ID in the system's mapping logic.

When the integration attempted to send the invalid (null/empty string `''`) Compensation_Plan_ID to Workday, Workday's strict validation (schema) rejected the data. This failure was a mismatch between the data being passed and the expected format/value range of Workday's Compensation Plan field.
