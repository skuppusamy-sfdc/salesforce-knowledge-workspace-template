## Description

As a system administrator, I want the system to ensure that a Contract Start Date is before the Contract End Date for any Contract record type when created or updated, so that we have clean data and related records such as PNC records are created created correctly without error.

## Acceptance Criteria

AC1: Clinician Contracts

Given a internal Corporate Support user with access to Contract (RT:Clinician)

When the user is working with a Clinician Contract 

And sets the “Contract Start Date” and/or the “Contract End Date”

Then the system shall validate that the “Contract Start Date” is not after the “Contract End Date”; the “Contract Start Date” shall always be before the “Contract End Date”. 

And display the following error message:

The 'Contract Start Date' must be before the 'Contract End Date'. Please ensure that the start date is earlier than the end date.
