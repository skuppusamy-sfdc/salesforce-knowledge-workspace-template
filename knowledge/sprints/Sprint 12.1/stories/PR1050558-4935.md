## Description

As an Integration User, I need to be notified (platform event = HPFUpdates) when rates should be sent to downstream systems (Qgenda)

Per discussion on 1/9/2026, we will be removing First Clinical Shift from the trigger.  Instead we will create a new indicator field to identify if an HPF has ever been set to Credentialing Status = ‘Active’ or ‘Scheduled Pending’.  If yes, then when a clinician contract is in executed status, then we will create the event to send rates for that HPF.  We are now only looking at the indicator (driven off of Credentialing Status of  ‘Active’ or ‘Scheduled Pending’) because that is means a clinician has been setup in Qgenda and can be scheduled, so we can send rates.

For maintenance scenarios we will still follow the same rule (above paragraph) where we check the new indicator to ensure that the HPF credentialing status at some point was set to ‘Active’ or ‘Scheduled Pending’ to send rates. In the event it’s a new HPF, we should NOT send rates (not fire event) if the indictor is false because this means that Credentialing Status was never set to ‘Active’ or ‘Scheduled Pending’.
