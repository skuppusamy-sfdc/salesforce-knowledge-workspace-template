## Description

As a Recruiter, I need to request $0 rates for clinicians, so that their compensation can be accurately represented in their contract

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I navigate to a clinician contract where I am configuring rates

Then I am able to change allow the Rate to be 0 for the all rate types

And when I have a 0 rate, then the following message displays “You have entered a $0 rate, please confirm this is accurate before continuing” 

_all rates cannot be negative/less than 0_

## Solution

Allow default be 0 in the omni script for 

Shift Rate Type(change) can be 0

 All other Rate Type  _(Bonus, Leadership Incentives, Stipend, Target, Variable, Locum_) it cannot be 0(as built)
