## Description

As a Corporate Support user, I need to remove the duplicate Primary Reason field, so that I am not entering data in multiple fields

## Acceptance Criteria

Given I am a Corporate Support user with access to offboarding cases

When I navigate to a Clinician Offboarding case (Type = Facility or SCP)

Then I am **not able to see the Primary Reason field**

_this field was created in error as part of_ [https://salesforce.atlassian.net/browse/PR10505582182|https://salesforce.atlassian.net/browse/PR10505582182|smartlink] and [https://salesforce.atlassian.net/browse/PR10505582462|https://salesforce.atlassian.net/browse/PR10505582462|smartlink]  _integrations should map to the Reason for Termination field_ 

Given I am a Corporate Support user with access to offboarding cases

When I navigate to a Clinician Offboarding case (Type = **SCP**)

Then I am able to see that the **Reason for Termination field** is populated with the following logic from the termination request (on submission):

** If all facilities have the **same reason*: Reason for Termination = same value from termination request

** If one or more facilities have the **different reasons*: 

*** Pick the facility termination with the farthest in the future **Last Day*

*** If Last Day is the same, pick the HPF with the most recent **Last Modified Date*

Given I am a Corporate Support user with access to offboarding cases

When I navigate to a Clinician Offboarding case (Type = Facility or SCP)

Then I am able to update the Reason for Termination field values independently - user can change the value from what was initially selected during termination request AND these manual updates do not trigger additional automation to update other records

## Solution

Repoint the Field  Case.Primary_Reason__c and use the field Case.Reason_for_Termination__c

Check if the fields Case.Primary_Reason__c is used in Omni Studio Code base like IP, Flex Card etc.

Impacted 

Flow: Case Creation After Flow 

Logic: Update the Reason for Termination when immediately the case is created from the Child Facility cases 

ValidationRule: Rule_Offboarding_Case_Close_Scenario

FlexipageFieldInstance: Case_Clinician_Offboarding_Record_Page

Replace field Case.Primary_Reason__c with Case.Reason_for_Termination__c in layout or filters or record page

Delete the Field Case.Primary_Reason__c
