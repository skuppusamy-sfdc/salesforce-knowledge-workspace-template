## Description

As a Recruiter, I need to withdraw contracts that I have submitted for approval, so that I can make updates if needed

Tester: Matt Hebert

Persona: Recruiter

Test Case ID: n/a

Description:  Recruiters need the ability to withdraw Contract requests. If not the records will become cluttered up with unneeded requests (see Conga project earlier this year).

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I **submit a Clinician Contract for approval**

And the Contract record now has the Status field with value Submitted 

Then I am **able to select Recall**

And the Contract record returns to **Draft status**, I am able to make updates in the Summary tab, all outstanding Approval Work Items records are updated so that Status is Recalled, and I am able to Submit for Approval again

## Solution

Update the Clinician Contract Approval Flow to perform the update 

NOTE: Steps to view Recall Comments:

1. Click on the ‘Approval Submission’ record for the recalled work item

1. On the Approval Submission record, click on the Related tab

1. Click on the Approval Submission Details record for Action Name = Recall

1. See “Comments” field
