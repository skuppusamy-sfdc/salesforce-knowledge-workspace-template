## Acceptance Criteria

AC1: Create new "EE NP/PA Hourly Blended" field on Preventive Care Agreement (PCA) object 

Given I am a System Administrator 

When I navigate to the Preventive Care Agreement (PCA) in Setup 

Then there is a new read only field = "EE NP/PA Hourly Blended"

AC2: Add new "EE NP/PA Hourly Blended" field to the Preventive Care Agreement (PCA) layout for specific Clinician rates 

Given I am a System Administrator 

When Corporate Support users with access to view Clinician Contract Preventive Care Agreement (PCA) navigate to a record 

Then the "EE NP/PA Hourly Blended" field shall ONLY be visible on the PCA layout for Clinician Contract when the below criteria are met: (should NOT be visible when view PCA for Practice Operations contracts)

- Type of Rate = ‘Shift’ 

- Shift Type (Type) ≠ ‘Shift Bonus’ 

And the "EE NP/PA Hourly Blended" field on the PCA record shall be populated with the value from the Clinician Contract’s "EE NP/PA Hourly Blended" field, when the below criteria are met: 

- Type of Rate = ‘Shift’ 

- Shift Type (Type) ≠ ‘Shift Bonus’ 

And if the Clinician Contract’s "EE NP/PA Hourly Blended" field is blank/null, then on the PCA the "EE NP/PA Hourly Blended" field shall display “0” (zero). 

NOTE: 

- Logic for MuleSoft (MS): Pull the new PCA "EE NP/PA Hourly Blended" field if value ≠ = 0/zero. If 0/zero, then pull value from “Rate” field. 

OR do the reverse:  Pull the ‘Rate’ unless the PCA "EE NP/PA Hourly Blended" field is NOT 0/zero.

- Assumption is that on the Clinician Contract the Blended Rates ("EE NP/PA Hourly Blended") field is populated when appropriate (Lisa confirmed that was working and passed testing), so we will just use a formula field to pull info from contract to PCA. Pulling to PCA, so that the rates can be pulled consistently from a single object.

- Expected results in Qgenda:

** For EE NPPA:

*** For all facilities related to the clinician contract, we should send the hourly blended rate ("EE NP/PA Hourly Blended") for Full and Part Time.

*** Blended Rates ("EE NP/PA Hourly Blended") should NOT be sent for Shift Type = Shift Bonus
