**Description:**

Workday Part time EE Nurse Practitioners and EE Physician Assistants should not have a work shift listed. This should be blank.  

Test Case Description :

1)Verify if Employee is Hired at Workday Successfully when Facility title is "EE  Physician" and time type is Part time  

2)Verify if Employee is Hired at Workday Successfully when Facility title is "EE Nurse Practitioners" and time type is Part time

**Steps to reproduce:**

"Pre Condition :

Facility or Client account is created with all the valid data Associated

Test steps :

1)Clinician is created with valid details(Unique Global provider ID,NPI,Address,email,phone) 

2)HPF is created when  Clinician is moved to offer with status as Prospect

3)New Contract is created  with Facility type EE and Start Admin work flag is set to true

4)Contract is executed and Hire providers event is triggered to workday

After above steps is performed Employee should be created in workday successfully and validate the record at workday"

**Expected vs Actual Result:**

Expected : In Workday Part time EE Nurse Practitioners and EE Physician Assistants should not have a work shift listed. This should be blank  Actual : This field has value 120 now for record below (Attached screenshots in document)  "Data Set 1 :  NANA ABRAMS Status Prospect Professional Title NP Mobile 7183535970 Email NPI 1508963638 Global Provider ID 140012  "

## Root Cause

Code Defect

The condition that determined when the Work Shift field should be left blank for part-time employees contained a typo. This error in the conditional logic caused the system to incorrectly evaluate the rule and assign the default or previous value ("120") instead of leaving the field blank as required by the business rule.
