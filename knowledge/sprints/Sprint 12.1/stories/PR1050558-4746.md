**Expected vs. Actual**

expected: the new contract would pull in the information from the most recent contract

actual: new contract pulling in information from oldest contract

## Solution

For Clinician Contract all Query that sorts Contract records should be using Start Date not Created Date

Update all DR that pulls Creates Clinician Contract and Clones Clinician Contract

## Root Cause

The system's "Contract Duplication" feature contained a selection algorithm which instead of identifying the contract with the most recent effective date or highest version number, the code was defaulting to the oldest record in the database.
