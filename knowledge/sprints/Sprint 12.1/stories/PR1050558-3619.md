## Description

As a Legal team member, when a Client Contract is in status = Draft, I want to add new Provider Network Contract (PNC) record(s), so that I can include any PNC record(s) I accidentally forgot to add during initial creation.

## Acceptance Criteria

AC1:

Given I am a Legal Team Member within the Corporate Support Apps

When I select a **Client Contract** record 

And **Status =  Draft, Submitted, or Executed.**

And I am on the “Provider Network Contracts” tab

Then I have the ability to ‘**Add Provider Network Contract**’ (system will create PNC records based on Service Account record(s) that are related to the Opportunity’s Services that are in Status = Active)

** And the Service Account record(s) have NOT already been linked to a Client Contract **(in status = Draft, Submitted, or Executed)* related to the Opportunity.  And the Service Account has not already been linked to this Client Contract [Service Accounts will be linked to the Client Contract via Provider Network Contract records]

- And user must select at least one Service Account to proceed.

** Display - Service Account & Type

- And if no Service Accounts are available, 

** Display message: No available Service Accounts. 

** Navigation option is to Cancel

And navigation options are:

- Cancel

** No records are created

** Guided experience is closed

** User is returned to the Client Contract record

- Finish

** Provider Network Contract record(s) are created

*** Record(s) are visible in the Provider Network Contract tab:

**** Related List  Fields displayed: Service Account Name, SCP Corporate Entity, Name (Provider Network Contract auto number link), Client Termination Date

** Guided experience is closed

** User is returned to the Client Contract record

And ‘Add Service Account(s)’ shall NOT be available (hide button or error message) when the Client Contract Status ≠ Draft, Submitted or Executed

## Solution

- Remove the New Button in Provide Network Contract Related List 

- Screen Flow based on Action: Add Provider Network Contract

** Display all the Opportunity Service Records not linked to current Contract PNC 

** Query: Select Account__r.Name, Account__r.Service_Line_r.Name from Service__c where Oppotunity__c = Contract.SourceOpportunityId and Service__c.Id not in (Select Related_Opty_Service_Id__c from ProviderNetworkContract Where Contract.SourceOpportunityId = Contract.SourceOpportunityId )

** When Record Found:

*** Display records so User can select multiple PNC records

** When No Record Found:

***  No available Service Accounts.

- When user select 1 or more service account add the Service Account to ProviderNetworkContract

** ProviderNetworkContract

*** ContractId = Current.Contract.Id

*** Related_Opty_Service_Id__c = Service__c.Id

*** Service_Account__c = Service__c.Account__c

**** ProviderNetworkContract.HealthcarePayerNetworkId = Select Id from HealthcarePayerNetwork WHERE Name = “**Corporate Entity TBD*“

*** ProviderNetworkContract.HealthcareProviderId = Select Id from HealthcareProvider Where AccountId = Service__c.Account__c)

*** StartDate = Current.Contract.StartDate

*** If Contract.EndDate != null then use PNC.EndDate = Contract.EndDate else 01/01/2099

**NOTE: Do not Show the End Date Flex Card**
