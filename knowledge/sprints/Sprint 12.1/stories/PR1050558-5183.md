## Description

**Tester:** Maci Broussard

**Role:** Recruiter

**Test Case ID:** N/A

**Description:** When Term Sheet is not required (based on criteria provided by business) Term Sheet status should be automatically populated to Not Applicable. 

Record linked: please review, has Previous Contract linked. History of record shows manual update to Term Sheet status field.

**Steps to Reproduce:**

1. Log in as Recruiter

2. Create Clinician Contract via Clone (Start from Previous Active)

3. Clinician Contract Record is approved

4. View Term Sheet Status (in test case, value was "Not Sent"

**Expected vs. Actual:** Actual Result above. To see the approved request in Contracting queue, I had to manually update the Term Sheet status to Not Applicable. 

Would have expected the system to apply the correct value to the field when Term Sheet is not required/not applicable. In this test case, the record would have sat and gotten stale, as it didnt meet the list view criteria to show in Clinician Contracting queue for them to take action on it.

## Acceptance Criteria

**No Term Sheet Needed**

Given I am a Recruiter

When I request a clinician contract that meets any of the following criteria:

- Clinician already has a contract for this employment type

** Previous Contract is not null

AND

Employment Type has not changed OR Employment Type change is not IC to EE

*** Employment Type has not changed: Previous Contract Employment Type = Current Contract Employment Type

*** Employment Type change is not IC to EE: Previous Contract Employment Type = Independent Contractor AND Current Contract Employment Type = Employee

And I submit the contract for approval

Then I can see that the **Term Sheet Status field is automatically populated to ‘Not Applicable’**

_negative test scenarios for reference:_

- First clinician contract: no update to Term Sheet Status field

- Clinician converting from IC to EE: no update to Term Sheet Status field

- Clinician converting from Locums to EE: no update to Term Sheet Status field

## Solution

Update the Flow:  Clinician Contract Review Action Handler

When the Status is set to Approved then 

if PreviousContract.Employment_Type__c = Same as Current Contract then Set  **Term Sheet Status** = **‘Not Applicable’ (only for EE and IC)**

if PreviousContract.Employment_Type__c = “Employee” and the new Contract.Employment_Type__c = “Independent Contractor” then Set  **Term Sheet Status** = **‘Not Applicable’**
