## Description

As a Legal team member, when I receive a Client Agreement Request from the Sales VP, I want to create a new Client Contract(s) so that I can capture the necessary information and track it through the execution process.

## Acceptance Criteria

AC1: **Case** List View [APPROVED]

Given I am a Legal Team Member the Corporate Support Apps

When I am on the Case tab

Then I can see a List View = Client Contracting Queue

- Filters: Record Type = Client Agreement Request, & Case Owner = Client Contracting Queue, and Status ≠ Closed

- Fields to display: Case Number, Subject, Status, Priority, Date/Time Opened

AC2: **Case** **layout** Record Type = Client Agreement Request [APPROVED]

Given I am a Legal Team Member the Corporate Support Apps

When I navigate to a Case Record Type = Client Agreement Request

Then I am able to see: 

- Header

** Subject is displayed in the top line

** Additional fields in highlights panel below: Priority, Type, Case Number

- Main Region:

** Tabs component: Details (default), Related, Files

*** Details tab: fields are displayed according to data dictionary

*** Files tab: standard files related list

*** Related tab: 

**** Client Contracts related list

***** Filter: Contract Record Type: Client Contract 

***** Fields to Display: Contract Number, Contract Name, Status, Category, Account Name

## Solution

AC1:

1. New List view for Case 

AC2:

1. Update Case Layout as per AC for Record Type Client Agreement Request
