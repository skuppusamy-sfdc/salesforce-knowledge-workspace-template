## Description

As a Data Quality team member, I need to receive a task to clear QGenda data when a clinician has been reactivated, so that I can ensure integrations work appropriately

## Acceptance Criteria

Given I am a Data Quality team member in the Corporate Support apps

When a clinician contract is **Submitted for Approval**

And that clinician has an HPF in the contract where they have previously been inactivated (**HPF for same service account exists in Inactive Status**)

Then the following Task record is automatically created:

- Subject = Clear QGenda Data - Previously Active

- Record Type = Default

- Assigned To = Data Quality Queue

- Status = Open

- Due Date = 3 days from automation start

- Type = Other

- Priority = Normal

- Name = clinician record

- Related To = contract record 

_Note: if a contract has already been submitted for approval (and this task was created) and the contract is submitted for approval again (ex. approval was rejected and contract needed an update) then this task is re-created_

## Solution

When a clinician contract is **Submitted for Approval**

Check all the HPF Record in the Contract via PNC> HPF and collect the IDs HPF.AccountId and HPF.Id
