## Description

“EE NP/PA FullTime Blended (Monthly/weekly) and Overage Rates Appearing for PartTime Contracts”

**Steps to Reproduce:**

1. Create/Open Clinician with  Prof title = NP/PA,  HPF time type = Part Time.

1. Create a contract with Employment type = Employee , Time type = Part Time.

1. Configure the Rate & Check NPPA calculation. 

**Expectation:** For Part time contract, Only EE NPPA Hourly Blended should be calculated.

**Actual Result:**

NPPA tier will be present in clinician contract  irrespective of contract's Time type  i.e.  Full time / part time contract. (As we are defaulting it from PO contract)

**Below statement from AC is failing:** 

**_EE NP/PA Full Time Weekly Blended, EE NP/PA Full Time Overage Rate and EE NP/PA Full Time Monthly Blended are for Full Time only_**

## Root Cause

There was an understanding gap that the NPPA Tier would be populated only for FullTime records. However, as per the current implementation, it is populated for both PartTime and Full-Time.

So now, the fix is as follows:

Time Type = Part-Time: Only EE NP/PA Hourly Blended is calculated.

Time Type = FullTime: EE NP/PA FullTime Weekly Blended, FullTime Overage Rate, FullTime Monthly Blended, and EE NP/PA Hourly Blended are calculated, provided the NPPA Tier is populated.
