## Description

As a Clinician Contract Specialist, I need to view NP/PA tiers and where clinicians are expected to practice, so that I can generate contracts accurately

## Acceptance Criteria

AC1:  Given I am a Recruiter in the corporate support apps

When I request a contract for a **full time employee NP/PA** clinician (Contract’s Employment Type = ‘Employee’ AND Contract’s Time Type = ‘Full Time’, and the Clinician’s Professional Title = ‘NP’ or ‘PA’)

And I am either +creating the contract+ OR +adding additional HPFs+

Then I can see the NP/PA Tier value from the client practice operations contract

And I can specify NP/PA Tier for that clinician:

- If Service Accounts from selected HPFs have the same NP/PA Tier

** Default the value of the Clinician Contract record NP/PA Tier field 

** User selection is NOT needed since there is only one possible value

- If Service Accounts from selected HPFs do not have the same NP/PA Tier

** Recruiter must select one NP/PA Tier from what values (unique values) are available at those facilities. (Design is to implement an validation rules with error if the user selects a NPPA tier values that is NOT applicable = “Please select an NPPA Tier value based on your chosen facilities”.

*** Display a message that Upon Submit Contract Request, then an approval will be sent to review NP/PA Tier mismatch

** And an approval record is created for Account Team Regional Director of NPPA Services to review NP/PA Tier mismatch (NOTE: upon clicking ‘Submit Contract Request’ button, then send approval)

AC2:  Given I am a Recruiter in the corporate support apps

When I clone an Executed Clinician Contract (selecting “Start from Previously Active Contract” in Contract Request OS)

Then the following fields are cloned:

- NPPA Tier

- EE NP/PA Full Time Weekly Blended

- EE NP/PA Hourly Blended

- EE NP/PA Full Time Overage Rate

- EE NP/PA Full Time Monthly Blended

AC3: And if I add new Facilities to the Clinician Contract

Then the newly added and cloned Facilities are included in the EE NP/PA field re-calculations exclude deleted PNC from the Calculation

## Solution

AC1: The solution is similar of how the NPPA Tier is auto populated or user entered when there is a mismatch.  

Check if the Contract.Employment_Type__c is Employee Contract.Time_Type__c is Full Time, and clinician Contract.Account.Professional_Title__c title is NP or PA (same as new Contract use case no change here)

When all the NPPA Tier has the same value then defalut Contract.NPPA_Tier__c = Client PO contract tier else force the user to input NPPA Tier (similar solution for new Contract)

AC2: When cloning the contract clone the fields below Contract Fields

- NPPA Tier

- EE NP/PA Full Time Weekly Blended

- EE NP/PA Hourly Blended

- EE NP/PA Full Time Overage Rate

- EE NP/PA Full Time Monthly Blended

AC3: Re Calculate NPPA Tier field (already built) when PreventiveCareAgreement.Is_Deleted__c = false or PreventiveCareAgreement.Cloned_From__c = Null or PreventiveCareAgreement.Cloned_From__c != Null (new change) and exclude when PreventiveCareAgreement.ContractPayementAgreement.Provider_Network_Contract__c.Is_Deleted__c = true (deleted use case for facility title change - Contract PNC Gray use case)

- EE NP/PA Full Time Weekly Blended
