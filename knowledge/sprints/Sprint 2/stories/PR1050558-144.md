## Description

As an Applicant, I need to log in to SCP’s prospect portal, so that I can track the status of my applications and find information about my recruiter

## Acceptance Criteria

Given I am an Applicant who has been provisioned a prospect portal login

When I login to the prospect portal

Then I am able to see the following on the home page:

- Name: Core Connect

- SCP Health banner

- Open Tasks record list

** List of Tasks where Assigned To is my Person Account record

** List displays: Subject, Status, Due Date

Given I am a Recruiter in the Corporate Support or Corporate Support Console apps

When I view a Clinician person account record

Then I am able to view:

- Enable Customer User

** Available for clinician records where a portal user has not yet been created

- Log in to Experience as User

** Available for clinician records where a portal user has been created

## Solution

- Create Portal User permission set from security matrix

- Create a Build Your Own aura site

- Add Record List component to home page

** Layout = Full

** Object Name = Task

** Filter Name = Open Tasks

*** _Create Open Tasks list view with the columns specified in the AC_ 

*** deploy Open Tasks list view to remove filters for Pardot and SCP business users

** Allow list pinning and Allow inline edit = unchecked

** Header properties: 

*** Show image icon, Show object name, Show list search, Show refresh button, Show filter button = checked

*** Show list actions, Show charts button = unchecked

** Show display options = Table Only

- Members

** Portal User permission set

** Clone Customer Community Plus Login User to create “Core HC Customer Community Plus Login User”

- Corporate Support users can enable customer users (create Corporate Support role)
