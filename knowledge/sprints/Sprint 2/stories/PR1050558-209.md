## Description

As a Recruiter, I need to use a guided wizard for my screening calls with prospects, so that I can quickly update prospect data and have prompts for items to cover on the call

## Acceptance Criteria

Given that I am a recruiter on the Corporate Support app

When I open a clinician record and click the “Create HPF & Job Association” button

Then I am able to 

Enter clinician information as built in [https://salesforce.atlassian.net/browse/PR1050558123|https://salesforce.atlassian.net/browse/PR1050558123|smart-link] 

Then when I navigate to the next screen,

-I am able to search and select Account (Facility/Hospital) records to associate the clinician with-

-And existing Healthcare Practitioner Facility records for this clinician are displayed with the HPF Name, Facility Name, and HPF Status- 

I am able to search and select Record Type: Service Account by Account  Name and associate them with the Clinician record by created the records as HPF (Health Care Practitioner Facility) with Practitioner = Clinician and Account = Service Account

Display the existing HPF (Health Care Practitioner Facility) record already available 

Then when I select save, the populated fields are updated (regardless of whether all the field requirements were met).

## Solution

OmniScript

Reuse the solution built for [https://salesforce.atlassian.net/browse/PR1050558192|https://salesforce.atlassian.net/browse/PR1050558192|smart-link] 

Healthcare Practitioner Facility Type Ahead 

Choose only Account Record Type Service Account

**Mapping:**

HPF Name:  Truncate 255 (Service Account.Name) 

Account = Service Account

 Practitioner = Person Account (Clinician)

Filter those Service Account records that already part of HPF
