## Description

As Data Quality team member, I want the ability to view Cases (RT: Clinician Change), so that I can what changes are being request for Clinician.

## Acceptance Criteria

AC1: Case List View for Open Clinician Changes

Given I am a Data Quality team member within the Corporate Support apps

When I am on the Case tab

Then I am able to see List View = Open Clinician Changes

- Filter: Case Record Type = Clinician Change, Status ≠ Closed

- Fields to Display: Case Number, Subject, Type, Status, Priority, Owner Name, Date/Time Opened

AC2: Clinician Change case layout

Given I am a Data Quality team member within the Corporate Support apps

When I navigate to a **Clinician Change** case record

Then I am able to see:

- Header

** Subject is displayed in the top line

** Additional fields in highlights panel below: Priority, Type, Case Number

- Main Region: 

** Tabs component: Details (default)

*** Details tab: fields are displayed according to data dictionary

AC3: Update “Data Quality” queue to support cases

Given I am a System Administrator with the Corporate Support apps

When a Clinician Change case is created

Then the system needs the ability to set the Case Owner to the “Data Quality” (queue)

NOTE:  “Data Quality” is an existing queue, but we need to update to support cases.

## Solution

Create list view on Case

- Filter: Case Record Type = Clinician Change, Status ≠ Closed

- Fields to Display: Case Number, Subject, Type, Status, Priority, Owner Name, Date/Time Opened

Create Case page layout/LRP fields as mentioned in DD

- Header

** Subject is displayed in the top line

** Additional fields in highlights panel below: Priority, Type, Case Number

- Main Region:

** Tabs component: Details (default)

*** Details tab: fields are displayed according to data dictionary

Ensure existing Data Quality queue has Case object, if not add it. (The assigning of case to data quality queue is part of other stories 2673, 2672, 2670)
