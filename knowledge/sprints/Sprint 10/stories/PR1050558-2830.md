## Description

As an Integration User, I need to know when Integration Failures are Re-Triggered so that Exception can be handled

## Acceptance Criteria

Given I am an IT User

When I update Status on a Integration Log record

Then the following updates happen automatically: 

- Update Status = Retry

- Trigger the following platform event: 

** Platform Event Fields, Name = Retry Integration Exception

*** IntegrationLogId

Picklist Value (Integration_Log__c.Status__c )

- Delete: Pending, In Progress, Retried

- Add New Values: Retry

## Solution

Integration_Log__c.Status__c 

Picklist Value

- Delete: Pending, In Progress, Retried

- Add New Values: Retry

Create new Platform Event: Retry Integration Exception

- Platform Event Fields, Name = Retry_Integration_Exception__c

** IntegrationLogId

Create a Record Trigger Flow for Update when Integration_Log__c.Status__c = “Retry” create a Platform Event

- Retry_Integration_Exception__c

** IntegrationLogId = Integration_Log__c.Id
