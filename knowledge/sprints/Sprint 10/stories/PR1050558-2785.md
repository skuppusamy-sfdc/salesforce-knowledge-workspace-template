## Solution

As part of the existing Clinician Contract submit request OS,

**non full time employee NP/PA clinician** // Clinician Prof title = NP or PA (Code Set Shift Type = NP/PA), Contract.Employment_Type__c = Employee and Contract.Time_Type__C != full time

In the HPF selection screen, where we display the list of HPF (as rows) with fields as columns  ( in data table),

Add the column Hour Commitment. [~accountid:61c036c29ee70a0068954daf]  need to know which field this should be mapped to

Contract > CPA > PCA 9/15: Map to HPF field  (added by data migration) = [**No_of_Hours_Anticipated_to_Work__c**] with **History Tracking enabled.**

Flow on PCA record - each time a record is added,  

If NPPA_Tier__c field is specified on the Clinician contract, then calculate the following and update the values of on the fields.

Hourly rate = Get the Rate__c from PCA where Unit of measure = per hour, for the Contract

Total Hourly rate = Calculate the sum of the rates

Avg Hourly rate = Total Hourly rate  / number of HPF records.

EE_NPPA_Hourly_Blended__c = Avg Hourly rate

EE_NPPA_Full_Time_Overage_Rate__c = Avg Hourly rate + 2

EE_NPPA_Full_Time_Monthly_Blended__c = Avg Hourly rate * ClinicianContract.NPPA_Tier__c

EE_NPPA_Full_Time_Weekly_Blended__c = Avg Hourly rate * 40

The same calculation should also take place the first time NPPA tier is added the first time - on contract request oS

rate should be computed when rates are changed

Consider only CPA of type Shift. Do not include shift of type Bonus
