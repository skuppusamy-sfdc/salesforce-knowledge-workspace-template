## Description

As a MSL, I need the Insurance Provided By field to have validation rules defined, so that the field is populated with the valid values

## Acceptance Criteria

Given I am a MSL in the Corporate Support apps

When I attempt the following changes on a Healthcare Practitioner Facility record

Then I am unable to save

** **Non-External Locums using Locums Insurance*

** IF Facility Title does not equal External Locums, Insurance Provided By = SCP Insured Locums OR Locum Company

** **External Locums using Non-Locums Insurance*

** If Facility Title equals External Locums, Insurance Provided By = SCP OR Commercial OR Hospital

And I receive the following error message at the Insurance Provided By field: “The selected value is invalid for this clinician’s HPF”

Given I am a MSL in the Corporate Support apps

When a HPF record is created

Then the Insurance Provided By field is populated according to the logic below: 

- If Facility Title is External Locums - Locum Company

- If Facility Title is Leased Employee or Leased Medical Director - Hospital

- If Facility Title is Managed - Hospital

- For all others - use the Insurance Provided By value on the Service Account record

## Solution

Create new field Insurance Provided By on HPF. Add to layout.

Add a row to existing Custom Metadata, to capture the table of Facility title and Insurance provided by.

When HPF is created ( using flow trigger),

Use the custom metadata, to auto-populate the Insurance provided by field on HPF.

Validation Rule on HPF

HPF> Facility title ! = Locum && HPF. Insurance Provided by ==  SCP Insured Locums OR Locum Company

OR

HPF> Facility title == External Locums, HPF.Insurance Provided By = SCP OR Commercial OR Hospital

Throw validation error “The selected value is invalid for this clinician’s HPF“
