## Description

Steps to Reproduce:-

1-Login as MSL user and navigate to a Clinician Record.

2-Start the Initiate Clinical Match flow and ensure to select Initiate Termination Request checkbox.

3-Initiate Termination Request flow is displayed.

4-Complete the flow and select the checkbox for the Initiate Clinical Match and click on Next.

5-Again Initiate Clinical Match flow is started.

Issue 1 - A loop is going on if user keeps on selecting the checkboxes on both the flows Initiate Clinical Match and Initiate Termination Request flows. 

Expected Result - A loop for the flows shouldn't happen.

Actual Result - As of now loop is going on for both the flows.

Issue 2 - If user navigates from Initiate Clinical Match to Initiate Termination Request flow then for nanoseconds dialog box of Initiate Clinical Match is being displayed with Finish button, Ideally it shouldn't happen.

Expected Result - For nanoseconds too dialog box of Initiate Clinical Match shouldn't be displayed with Finish button.

Actual Result -  For nanoseconds dialog box of Initiate Clinical Match is being displayed with Finish button,

Issue 3 - Everytime user completes the Initiate Clinical Match then a new Clinical Match case is being created, Ideally if one Open cases is present then a new duplicate case shouldn't be created.

Expected Result - If one Open cases is present then a new duplicate case shouldn't be created.

Actual Result - As of now duplicates cases are being created everytime user completes the Initiate Clinical Match flow.
