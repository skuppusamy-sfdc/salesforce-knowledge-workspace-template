## Description

As a Clinician Contracting Specialist, I the ability change the Clinician Contract Status to “Contracting - In Progress“, so that a contract generation can be triggered in Conga.

## Acceptance Criteria

Given I am a Clinician Contracting Specialist within the Corporate Support apps

When I have a clinician Contract record “Term Sheet Executed” = checked 

And I reviewed the signed Term Sheet 

Then I have the ability to change the Status to “**Contracting - In Progress**”

NOTE for the Perfcient Team: When the Contract (Record Type = Clinician) Status = “**Contracting - In Progress**”, then that will be the trigger for Conga CLM to generate a Contract. 

Also, there is NO validation rule that Term Sheet Executed has to be checked for a Clinician Contracting Specialist to change the status from “Approved” to “Contracting - In Progress”, since they manage both contracts that have Term Sheets or not.

## Solution

Add a New picklist value for Contract.Status = “Contracting - In Progress” , only for Clinician record type. 

Contracting Status = Signed

Add a validation only can change from Status = “Approved” to “Contracting - In Progress”

Add Custom Permission = “**Clinician_Contracting_Specialist_Bypass_Validation**” to new Permission Set = **Clinician Contract Bypass Validation**

Add new permission set to Permission Set Group = **Clinician Contracting**
