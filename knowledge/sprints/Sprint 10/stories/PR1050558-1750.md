## Description

As a Recruiter, I need to submit contracts for approval, so that I can ensure rates are aligned with SCP standards before sending out a term sheet

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I navigate to a clinician record

And I have specified all necessary rates through the Contract Request experience

Then I can select **Submit Contract Request** on the Summary tab of the Contract record (only when Status = Draft)

And the following happens: 

** If HPFs have **more than one Contracting Corporate Entity** (if Contract has more than one Contracting Corporate Entity) AND **Employment Type = Employee* 

** Built in [https://salesforce.atlassian.net/browse/PR1050558301|https://salesforce.atlassian.net/browse/PR1050558301|smart-link] 

** If there are **special terms*

** Built in [https://salesforce.atlassian.net/browse/PR1050558301|https://salesforce.atlassian.net/browse/PR1050558301|smart-link] 

** If any **clinician rates do not match facility rates*

** Contract record updates to Status = Submitted

** Contract record is NOT locked from being edited

** Approval Types

**** **Bonus*: if there is a bonus specified outside the range of the job (fields on bpats__Jobc object are Commencement_Bonusc, Locums_Buyoutc, New_Grad_Bonusc, Relocation_Bonusc, Retention_Bonusc, Sign_On_Bonus__c)

**** Approval Work Item records are created for the following approvers per applicable HPF: 

***** Account Team Role = VP Operations (with the ability to delegate)

****** when associated Service Accounts have different VPOs, only send approval to indicated facility on bonus request 

******  if multiple VPOs, assign approval to the most recent VPO on the Account Team

****** NOTE: Since Bonus can ONLY be added to one HPF (look for the CPA related to the HPF), send approval to the VPO listed on the related Service Account. If there are more than 1 VPO listed on the Service Account, then assign to the most recently added VPO to the account team. 

**** **Other Bonus**: in addition to routing above to VPO per applicable HPF, **Tactical Team Queue* must also approve in parallel 

**** **Over Rate*: if clinician rates do not match facility rates

**** Approval Work Item record is created for the following approver based on the applicable HPF: 

***** Account Team Role = VP Operations (with the ability to delegate)

****** when associated Service Accounts have different VPOs, only send approval to indicated facility for overrate

******  if a service account has multiple VPOs, assign approval to the most recent VPO on the Account Team

- If multiple steps are required, they occur in parallel and all are required 

** If **none of the above criteria are met*

** Contract record updates to Status = Approved

- Approve/Reject Actions

** When users approve/reject they can add comments

** Send notification to request submitter and all approvers with comments

** Contract Status updated to Approved

** If Contract is not approved

*** Contract Status updated to Draft

## Solution

## **Use same Autolaunched Orchestrator (no trigger) Approval Flow process**

extend and reuse flow built for [https://salesforce.atlassian.net/browse/PR1050558301|https://salesforce.atlassian.net/browse/PR1050558301|smart-link] 

Step 3: 

Flow on Contract : when status changes to submitted 

NOTE: to choose/compare against client contract; query for executed contract where start date < today and end date > today or no end date. 

1. **Bonus**

If Clinician Contract’s bonus rate for the bonus type ( derived from CPA & Preventive care agreement) is not within the range of the bonus specified in bpats_Job__c object,( match by bonus type ) 

## submit for  Approval

## Query ContractPaymentAgreement WHERE Type__c = Bonus WHERE ContractId = Id of the Contract when button was clicked  (clinician contract)

##  Get the service account team role, from clinician  HPF  SA.

### Query AccountTeamMember WHERE AccountId = Provider_Network_Contract__r.Healthcare_Practitioner_Facility__r.AccountId AND TeamMemberRole = “VP Operations” LIMIT 1

#### If Record Found Then Create a Approval Work Item and Approval Work Item Owner = User from Step 1.b.i

#### If No Record Found Raise an Error to the “For Approval of Bonus Request “VP Operations” role is required to be setup for Provider_Network_Contract__r.Healthcare_Practitioner_Facility__r.Name”

## If No Records Found Move to Next 

1. **Other Bonus**

## If there is value in Other bonus field, then send approval to VPO per HPF’s SA and Tactical Team Queue

1. **Over Rate**

Background : If Client contract’s rate for the ( from HPF > SA > Contracts > CPA > PCA )  is lesser than Clinician Contract’s rate (from HPF > SA > Contracts > CPA > PCA ) (match based on facility title) then submit for Approval

## Query ContractPaymentAgreement WHERE Type__c = Stipend, Shift, Variable, Target WHERE ContractId = CurrentContractId and check if the rate is higher than the client contract’s rate.

## If Records Found

### Query AccountTeamMember WHERE AccountId = Provider_Network_Contract__r.Healthcare_Practitioner_Facility__r.AccountId AND TeamMemberRole = “VP Operations” LIMIT 1 - query the most recent record and use it. 

### If Record Found Then submit for approval to VPO (role above)

### If No Record Found Raise an Error to the “For Approval of Bonus Request “VP Operations” role is required to be setup for Provider_Network_Contract__r.Healthcare_Practitioner_Facility__r.Name”

1. If none of the criteria are met, 

## update the current clinician contract status to approved.
