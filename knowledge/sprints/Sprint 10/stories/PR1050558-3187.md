## Description

As a SFO, I want the system to automatically update all related Clinician executed Contracts based on specific type of updates made to the **Practice Operations** contract record, so that Clinician Contracts stay in sync with the Client contract and contract generation in Qgenda can be appropriately triggered. 

** **Coverage Change  Only Shift changes*: no new shift record created. New Shift is add with the same Type and same Rate. [See story [+PR10505582916+|https://salesforce.atlassian.net/browse/PR1050558-2916]]

** **Pay Code Change*: Only paycode change, new paycode added to shift (no rate change)

## Acceptance Criteria

AC1: Coverage Change  new shift with same shift type and same rate [See story [+PR10505582916+|https://salesforce.atlassian.net/browse/PR1050558-2916]]

AC2: Rate Pay Code Change - New pay code added to Shift (no Rate Changes)

Given I am an SFO within the Corporate Support Apps

When I clone a **Practice Operations contract** in Status = Executed

And I add new **Rate Pay Code(s)** to Shift(s)

And No rate changes have occurred 

Then the cloned Practice Operations contract Status gets updated to Status = “Executed” & Start Date equals current date, the system shall automatically Add the Rate Pay Code to all related **Clinician** contracts meeting the following criteria:

- Clinician Contract Status ≠ Canceled

- HPF Service Account matches Practice Operations Contract’s Service Account 

Note: Before the Practice Agreement Contract Status = Executed, the pay code must be established in Qgenda. (will be addressed as part of the business process within the existing approval process.)

AC3: Platform event trigger for HPFTrigger (HPFUpdates__e) for Qgenda [See story [+PR10505583190+|https://salesforce.atlassian.net/browse/PR10505583190] ]

## Solution

As part of the batch class built for 2916, in the start method, we would have gotten the PO CPA where Rate_Pay_Code_Remove_c or Rate_Pay_Code_Added_c > 0.

Add method to Get Client RatePayCodes

Query for RatePaycode, where Is_Newly_Added > 0 or Is_Deleted = true.

Get the RatePayCode>PCA>Shift>Shift_Full_Name__c  Code and RatePayCode.PayCodeName

Add method to match the Clinician Shift PCA

Match the PO RatePayCode using the Shift_Full_Name__c with the existing Clinician PCA > shift>Shift_Full_Name__c and PO.RatePayCode > PCA > FacilityTitle with Clinician PCA.FacilityTitle

For the matched Clinician PCA

to add RatePayCode

if PO RatePayCode has Is_Newly_Added true , create a new rate paycode for clinician PCA and insert

to delete RatePayCode

if PO RatePayCode has Is_Deleted to true, query for the ratePaycode by matching the RatePayCode.PayCodeName and delete the record from clinician PCA.
