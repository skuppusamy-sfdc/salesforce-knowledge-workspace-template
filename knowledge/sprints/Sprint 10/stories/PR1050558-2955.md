## Solution

- Field creation part of 2924

Ensure the field SCP to Book travel is already created on Contract. If not create it as per [https://salesforce.atlassian.net/browse/PR10505582924|https://salesforce.atlassian.net/browse/PR10505582924|smart-link] 

In the Contract Request omniscript, add SCP to book travel as part of the second screen , below the date fields ( both the Contract selection screens in OS). This UI input will be mapped to the Contract.SCP to book travel field at backend

As part of Clone process ( Clinician_CloneContract IP), ensure this field is cloned over in the new contract.
