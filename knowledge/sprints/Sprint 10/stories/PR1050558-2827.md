## Description

As an Integration User, I need to know when Healthcare Practitioner Facility are updated to Active status in Salesforce, so that I can update downstream systems

## Acceptance Criteria

Given I am an Integration User

When I update First Clinical Shift on a Healthcare Practitioner Facility record

When  Healthcare Practitioner Facility, Facility Title is one of below

Employee

Independent Contractor

-Locum-

-Leased-

Managed

Partner

Then the following updates happen automatically: 

- Status = Active

- Trigger the following platform event: 

** Platform Event Fields, Name = HPFUpdates

*** HPF Id

**** Contract Id = [Contract.I|http://Contract.ID]d WHERE Contract.Employment_Type__c = HealthcarePractitionerFacility.Facility_Title_r.Code_Set_Sub_Type_c and HealthcarePractitionerFacility.PractitionerId = Contract.AccountId  and Status = Executed  **Order By Start Date ASEC*

*** Global Provider Id

*** Global Facility Id

*** Type = “Rate for HPF”

All contract records the qualified the below condition need to have one Platform Event per contract records

** [Contract.I|http://Contract.ID]d WHERE Contract.Employment_Type__c = HealthcarePractitionerFacility.Facility_Title_r.Code_Set_Sub_Type_c and HealthcarePractitionerFacility.PractitionerId = Contract.AccountId and Status = Executed  **Order By Start Date ASEC*

## Solution

Update the Flow to HPF Updates Triggered Flow

Change 1:

Add new Condition with below Criteria to restrict Platform Event Type (Note: The original condition If_date_First_Clinical_Shift_is_changed should remain unchanged)

Employee

Independent Contractor

-Locum-

-Leased-

Managed

Partner

**Trigger the Platform Event within the LOOP (so that sequence of dates are maintained in the target system) -** **NO BULK Processing**
