** '**Type of Rate*' field I can only select “Shift”

** I should NOT have the ability to add or edit any other type of rate (Variable, Leadership Incentive, Stipend, Target)

## Solution

All Changes to Apply to Practice Operation Contract

Create a validation rule in Shift__c to make sure only same shift type can be added when Contract.Shift_Coverage_Change__c = true

- Validation Rule:

** Validation Rule to prevent the user from editing any shift Shift_Name__c when Contract.Shift_Coverage_Change__c = true

** Error Message: “For Coverage Changes Shift Type changes not allowed”  

Create a record triggered before save flow that will throw error when a new Shift__c records is created with a Shift Type that is not already available when Contract.Shift_Coverage_Change__c = true and is_cloned = null

- For a new record when Shift_Name__c entered check with Query: Select Shift_Name__c from Shift__c where Shift_Name__c = <Newly entered Shift> 

** if the query returns the record then allow save 

** if no result is returned error Error Message: “For Coverage Changes new Shift Type changes not allowed”  

 

Preview Updated to UI

When Contract.Shift_Coverage_Change__c = true 

Update the Flex Card ContractSpecifyRates

1. Type of Rates: Filter and display only Shift

1. In the next screen display only the new Shifts that was added previously identified by Shift__c.Cloned_From__c == ““ 

1. When saving the **Preventive Care Agreement** record check if the rate entered matched with already exiting rate for the same shift type and facility title

## Query: Select Max(Rate__c), Min(Rate__c) from **PreventiveCareAgreement** WHERE FacilityTitle__c in (Facility Title Selected) AND Shift__r.Shift_Name__r.Code__c in (shift selected) group by FacilityTitle__c, Shift__r.Shift_Name__r.Code__c

### When Max(Rate__c) != Min(Rate__c) then throw error “The ‘Rate' must match the previous rate entered for the selected Facility Title(s) & Shift Type(s). Select the Facility Title in such a way the rates will match”

### When Max(Rate__c) != newly entered rate in UI throw error “The ‘Rate' must match the previous rate entered for the selected Facility Title(s) & Shift Type(s). Select the Facility Title in such a way the rates will match”

### When Max(Rate__c) = newly entered rate in UI then allow save
