## Description

As a SFO, when working with a cloned client contract, I want to re-add a deleted rate so that I can continue making edits as needed for the new contract.

## Acceptance Criteria

Given I am an SFO in the Corporate Support apps

When I navigate to a Contract (RT: Practice Operations) record in Status = Draft that I had created from “Start From Previous Active Contract” (**clone** option)

And on the Summary tab, I clicked Delete (trash icon) on a rate

And the system displays the rate under “Removed” (Is Deleted = checked/true)

Then on the rate record I have the option to “**Re-add**”

And when I select “Re-add” then:

- The system unchecks “Is Deleted”

- Rate is taken out of the “Removed” section

- “Re-add” option is hidden from the rate record. 

!image20250908230750.png|width=33.333333333333336%,alt="image20250908230750.png"!

## Solution

We can identify cloned contracts using “PreviousContractId“  field

For those contracts, If IsDeleted = true, display “Re-Add” button on the flexcard.

When ReAdd is selected, IsDeleted = false, and hide “ReAdd” button.
