## Description

### ✅ **Preconditions:**

1. A Client Contract is executed with Rate Pay Code configured.

1. A Clinician Contract (CPA) is created with PCA having the **same rate** as the client contract.

1. PCA record is automatically created under CPA.

### 🔄 **Steps to Reproduce:**

1. Execute a Client Contract with a configured Rate Pay Code.

1. Create a new Clinician Contract (CPA) using the same rate.

1. Let the system auto-create the first PCA under this contract.

1. Observe the PCA record → **Related tab**.

### 🔍 **Expected Result:**

** The PCA record should automatically display the **Rate Pay Code* (via flow) on creation, assuming all preconditions are met.

- When batch run then flow was running and creating rate pay code under wrong pca

### ❌ **Actual Result:**

** The **first PCA record does not show the Rate Pay Code*.

** After cloning the contract and creating a **second PCA**, and running the **batch job**, **both PCA records* get Rate Pay Codes populated.

- This indicates the flow did not work as expected on initial PCA creation.
