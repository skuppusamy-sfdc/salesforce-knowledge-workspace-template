## Description

As a DMO, I need to indicate a clinician’s final offboarding decision, so that updates can be made automatically as needed

## Acceptance Criteria

Given I am a Corporate Support user in the Corporate Support apps

When I navigate to a Clinician Offboarding case where Type = SCP

Then I am able to update the Offboarding Decision field to **No Longer Exiting**

And the following updates happen automatically:

- Offboarding Case updates

** Status = Closed

** Closed Reason = Canceled

- Child Task updates

** Status = Canceled

Given I am a Corporate Support user in the Corporate Support apps

When I navigate to a Clinician Offboarding case where Type = SCP

Then I am able to update the Offboarding Decision field to **Confirmed Exit**

And the following updates happen automatically:

- HPF updates

** Effective To = Last Day

** Contracting Status = Termination In Progress

** Anticipated Termination Date = Last Day

- Offboarding Case updates (occur when all child tasks are closed)

** Status = Closed

** Closed Reason = Complete

- HPF updates (occur when Effective To is in the past)

** Status = Inactive

** Contracting Status = Terminated

** Termination Date = day automation ran

- Person Account updates (occur when Effective To on all HPFs is in the past)

** Status = Inactive

-Given I am a Corporate Support user in the Corporate Support apps- (removing, adding to a new backlog story) 

-When I navigate to the guided offboarding experience AND a Clinician Offboarding case where Type = Facility-

-Then I am not able to view/edit fields for HPF specific Last Day, Termination Request Date, and Termination Request Received since this lives at the SCP level for full termination-

Given I am a Corporate Support user in the Corporate Support apps

When I navigate to a Clinician Offboarding case where Type = Facility

Then when all child tasks are closed and the case is closed

And the following updates happen automatically:

- HPF updates

** Effective To = Last Day

** Contracting Status = Termination In Progress

** Anticipated Termination Date = Last Day

- HPF updates (occur when Effective To is in the past)

** Status = Inactive

** Contracting Status = Terminated

** Termination Date = day automation ran

## Solution

Updating of the newly created Offboarding Decision is manual by the user

Use one of the existing flow on Case , Record update flow

Filter for offboarding case record type, where case type = SCP,

If Offboarding Decision is updated to “No longer Exiting”, 

Offboarding Case updates

- Status = Closed

- Closed Reason = Canceled

- Child Task updates

** Status = Canceled

If Offboarding Decision field to **Confirmed Exit**,

Get all the HPF for the clinician. Get Case> Account. For the Account, get all HPFs.

Check if last day > TODAY

HPF.Effective To = Case.Last Day

HPF.Contracting Status = Termination In Progress

HPF.Anticipated Termination Date = Last Day

If Case.Last_Day__c < TODAY

HPF.Status = Inactive

HPF.Contracting Status = Terminated

HPF.Termination Date = Last Day

Use Dynamic form in Lightning record page on Case, and display these fields only if Type = SCP and RecordType = Clinician Offboarding

Last Day, Termination Request Date, and Termination Request Received

Check if there is already a story where this was implemented.
