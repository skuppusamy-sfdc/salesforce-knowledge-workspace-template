## Description

As an Integration User, I need clinician contracts to be associated with Rate Pay Codes, so that data can be sent from Salesforce to QGenda

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I create a Clinician Contract

Then the following happens automatically: 

- Rate Pay Code records from client are created for the clinician

** To find client contract: from clinician contract, find the client contract records associated with the clinician via the Account field on the Healthcare Practitioner Facility records

** Create Rate Pay Code records

## Solution

Use the existing omniscript of creating PCA record, where we get the rates from the client contract (that are executed, and there is a Create PCA and CPA step.

For these Client Contracts, Get Contract Payment Agreement > Preventive Care Agreement > Rate Pay code (for Shift PCA records)

Use PCA’s Facility Title . Shift, Type and CPA’s type to match the records on which to insert rate pay code

ContractPaymentAgreement - has both Service Account and Contract

Rate Pay code is child of  Preventive Care agreement

Preventive care agreement is child of Contract Payment Agreement (MD)

Preventive care agreement is child of Contract and has lookup to PNC
