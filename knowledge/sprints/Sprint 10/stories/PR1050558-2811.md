## Description

As a recruiter, I the ability to generate a term sheet so that I can expedite the contracting process prior to engaging the Clinician Contracting Specialist team.

## Acceptance Criteria

Given I am a Recruiter within the Corporate Support apps

When I navigate to a clinician Contract record 

And the following criteria is met:

- Contract Status = “Approved“

And 

- [This is a new Contract (Previous Contract field is blank) 

OR 

- Clinician is an Independent Contractor (IC) moving to Employee (EE) employment type]

Then I shall have the ability to click “Generate Term Sheet”, (should NOT be displayed if the criteria above is not met)

Then a document is generated with the following template (attachment = **Conditional Offer Letter (****5****)**), 

And the following fields are pulled into the merge fields with the initiating Clinician/Contract details

**Conditional Offer Letter (****5****) ** delete the following:  “The other terms that govern this agreement can be found at: [**insert hyperlink**]” per Greg Bianchi-

- Date = [today’s date]

- Clinician/Clinician Name = Contract.AccountID 

- Clinician contact info = [Clinician’s > Mailing Address]

- Insert Title = [ATS Applicant’s > Job > Job Title]  (Note: multiple values should NOT be an issue, should only have one job title.)

- Insert Name = [ATS Applicant’s > Hiring Manager’s Name] (Note: multiple values should NOT be an issue.)

- Position = [ATS Applicant’s > Job > Job Title] 

- Position Type = Contract.Time_Type__c

- Start Date = Contract.StartDate

** Locations = [Display the contract’s active HPFs **Service Account* name]  

- Compensation = [Display the Rates, see below for logic to display] 

** If Employment Type = Employee (EE) AND Time Type = Full Time AND Shift Type = NP/PA, then display:

*** Contract’s “EE NP/PA Full Time Monthly Blended” 

*** Format = [EE NP/PA Full Time Monthly Blended] per Month

**** Example: $200.00 per Month

** If Employment Type = Employee (EE) AND Time Type = Part Time AND Shift Type = NP/PA, then display:

*** Contract’s “EE NP/PA Hourly Blended”

*** Format = [EE NP/PA Hourly Blended] per Hour

**** Example: $55.00 per Hour

** Else, display

*** Type/Shift Full Name

*** Rate 

*** Unit of Measure

*** Format = [Type/Shift Full Name] [Rate] [Unit of Measure]

- Anticipated Monthly Clinical Hours = [Contract> Anticipated Clinical Hours]

- Insert Hyperlink = [NEED HYPERLINK]

And a preview of the generated document is displayed to the initiating user

And Additional actions:

- Cancel

- Link to template [https://docs.google.com/document/d/1gM9YFc0Fu9zy1aaFFZDoWHgCtqD3INIO/edit?usp=sharing&ouid=107332607889089087961&rtpof=true&sd=true|https://docs.google.com/document/d/1gM9YFc0Fu9zy1aaFFZDoWHgCtqD3INIO/edit?usp=sharing&ouid=107332607889089087961&rtpof=true&sd=true|smart-link] 

- Field = Term Sheet Executed, creation part of [+PR10505581794+|https://salesforce.atlassian.net/browse/PR10505581794]

Doc Gen, Use the sample document provided to generate template.

DR and Transform

From Clinician Contract, extract PNC from which you will get the HPF and ServiceAccounts.

Query the CPA of the Contract and group them by the HPF.

To get the Job details, HPF > Job_Applicant__c > Job > Job Title

|Contract.Employment_Type__c|Contract.Time_Type__C|Shift Type|Compensation| |

|Employee (EE)|Full Time|NP/PA (HPF> FacilityTitle > CodeSetType.)|Contract.EE_NPPA_Full_Time_Monthly_Blended__c| |

|Employee (EE)|Part time|NP/PA (HPF> FacilityTitle > CodeSetType.)|Contract.EE_NPPA_Hourly_Blended__c| |

|All others| |PCA.Shift__r.FullName or PCA.Type__r.Name (whichever is populated)| |, PCA.Rate__c, PCA.UnitofMeasure__c|

NOTE:

SCP mentioned there would be only one HPF in this scenario. However we have to accommodate for multiple HPFs, hence group them per HPF. Use the _OR repeat per HPF format in this document_
