## Description

As a Legal team member, I need to track a clinician’s visa information, so that I can coordinate with appropriate teams

## Acceptance Criteria

Given I am a **legal team member, MSL, data quality team member, or recruiter** in the Corporate Support apps

When I navigate to a clinician record

Then I am able to populate the following: 

- Visa Number - text, 25

- Type - picklist

** Values: Birth Abroad Cert, Citizen/Naturalized, E2, EAD, H1B, H2B, H3B, H1C, J1, J2, L, O1, P, Perm (Green Card), R, TN

- Visa Expiration - date

_other personas can see but not edit these visa fields_

Given I am a System Administrator

When I navigate to object manager

Then I cannot open the Visa object

- Delete custom Visa object

- Update OmniScripts to remove all references to Visa object (ex. when Sponsorship Need? is populated with Yes, redirect picklist to the clinician record rather than Visa object)

## Solution

Delete the Visa object and any reference to it. Create destructive changes.

Create new fields on Clinician Person account as mentioned in the DD

Visa Number - text, 25

- Type - picklist

** Values: Birth Abroad Cert, Citizen/Naturalized, E2, EAD, H1B, H2B, H3B, H1C, J1, J2, L, O1, P, Perm (Green Card), R, TN

- Visa Expiration - date
