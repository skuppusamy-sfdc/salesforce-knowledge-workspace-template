## Description

As a Data Quality team members, I need Salesforce to enforce requirements to collect NPI for some clinicians, so that identifiers are populated when available

## Acceptance Criteria

Given I am a Recruiter with access to “Create HPF” guided process in the Corporate Support apps

When I navigate to a clinician record

And I use the “Create HPF” guided experiences which ask for NPI 

Then I am required to populate NPI when the following criteria are met: 

- Clinician’s Professional Title equals “MD”, “DO”, “NP”, “PA”

Display the message ““NPI number is required for clinicians with the professional title MD, DO, NP, or PA.””

And when the above criteria are not met, I am able to populate NPI but it is not required

And have Checkbox = “No NPI number” (NEW) - Conditionally visible based on the criteria 

Professional Title = NP, PA, MD, DO and

(below) Task is not completed

And if NPI is required (Professional Title equals “MD”, “DO”, “NP”, “PA”) and “No NPI number” is checked, then system creates a task:

- Subject = No NPI number for <Clinician>

- Assigned To = Data Quality Queue

- Status = Open

- Due Date = Today

- Priority = High

- Type = NPI Number Missing

- Name = <Clinician>

And if the NPI number is required and there is a “Closed “ task Type = NPI Number Missing, then allow the user to proceed with “Create HPF” guided experience. 

Given I am a clinician with access to the HC Portal

When I am completing an **intake form**

Then I am required to populate NPI when the following criteria are met: 

- Professional Title equals “MD”, “DO”, “NP”, “PA” and Professional Status Practicing Status does not equal student or new graduate. Display the error message “ NPI is required”

And when the above criteria are not met, I am able to populate NPI but it is not required.

## Solution

In the existing Create HPF omniscript, in Clinician Information screen ,  

Placement of the checkbox and validation  Update NPI  screen, (existing)

Add a new checkbox ‘No NPI number’

Query for task related to Clinician of type “NPI Number missing “  and status Closed. 

Create a validation error message if Clinician Prof title is one of these  “MD”, “DO”, “NP”, “PA” and No NPI number checkbox is unchecked and if no task if found of type NPI Number missing. Msg  “NPI number is required for this Clinician’s professional title”

If if Clinician Prof title is one of these - “MD”, “DO”, “NP”, “PA” and No NPI number checkbox is checked, create the task mentioned in AC (via IP from the OS)

NOTE: Future attempts to create HPF  if task of NPI Number missing is closed  no need to display the validation error message, as it was bypassed once already.

As part of Clinician Profile Update intake form - make NPI required if Clinician Professional Title equals “MD”, “DO”, “NP”, “PA” and Practicing Status does not equal student or new graduate
