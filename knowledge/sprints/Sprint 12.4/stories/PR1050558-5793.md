## Solution

As a Clincian Contacting Specialist I can view a list view of Tasks called “Transition Specialist Queue” that shows the name of Task.WhatID.Case.ContactName

**SOLUTION:**

Update Task creation for tasks “Update Records and Notify Teams” and “Employee Review” to include WhoID as well as WhatID on creation. WhoID should be populated with the related Case.ContactName

Flow: CaseCreationAfterFlow

SubFlow: Create_Task_for_SCP_Offboarding_Cases

Add WhoID assignment to the following TaskFieldAssignment elements

Is SCP Offboarding Decision Path: 

- Assign Task Fields for Employee Review

- Assign Task Fields for Conduct Stay Interview / Offer Stay Incentive

- Assign Task Fields for Exit Interview

- RN/LPN Manager Assign Task Fields for Exit Interview

- Regional Medical Officer Assign Task Fields for Exit Interview

- Assign Task Fields for Compensation Recoupment

Is Facility Offboarding Decision Path: 

- Assign Task Fields for CPA Review

- Assign Task Fields for Review Contract for Termination for other Employee Type

- Assign Task Fields for QGenda Credentialling Inactivation

- Assign Task Fields for Update Malpractice Insurance

- Assign Task Fields for Compensation Recoupment 2

On each assignment element Add 

**{!taskObj.WhoId} = {!Loop_on_Offboarding_SCP_Cases.AccountId}** for the SCP Client Cases

OR

**{!taskObj.WhoId} = {!Loop_on_Offboarding_Facility_Cases.AccountId}** for the Facility Offboarding Cases

to each element above.

## Root Cause

Original automation creating: Tasks with CaseID in WhatID but not ContactID in the WhoId
