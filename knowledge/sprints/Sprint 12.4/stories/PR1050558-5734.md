**Business Impact:**

Incorrect data flowing into downstream systems and on clinician contracts

## Acceptance Criteria

Given I am a SFO persona within the Corporate Support Apps

When I am working on a **Practice Operations contract in Status = Draft**

And from Summary tab, I click “Specify Rates” and select Type of Rate = Stipend

Then in the “**Admin Hours**” field, I should be able to enter number with up to 3 digits and 2 decimal places.

And if the user does NOT input a valid value then display the following error message and prevent the user from proceeding

- Error Message: “Please enter a value with no more than 3 digits and 2 decimal places.”

And upon clicking “Finish” when I navigate to the **Preventive Care Agreement** record, I shall see the value I inputted in the **“Admin Hours” field**. 

Ex. 

100.75 - Pass

40.50 - Pass

40 - Pass

40.175 - Fail, system should not let them proceed/finish. Display error message

1000 - Fail, system should not let them proceed/finish. Display error message

## Solution

** **Update the following fields to be Number(3,2)*

- PreventativeCareAgreement.Admin_Hours__c Number(3,0)

** **Update the Validation Rule* to allow for 2 decimals and update failure message to read “Please enter a value with no more than 3 digits and 2 decimal places.”
