## Description

As a System Administrator, when a CCM record is created for Process Type = Agent, I want the system perform additional Clinician Contract checks, so that the Client Change Management record’s ‘Status’ and ‘Cancel Reason’ can be appropriately set when a Clinician either doesn’t have an Executed Contract or they have an Executed Contract, but it doesn’t currently contain that rates that got updated on the Practice Operations Executed Contract.

## Acceptance Criteria

Given I am a System Adminstrator within the Corporate Support Apps

When a Practice Operations contract is Executed

And Client Change Management (CCM) records are created with Process Type = ‘Agent’

Then the system shall perform additional checks to set the CCM ‘Status’ = ‘Canceled’ & set a ‘Canceled Reason’:

- If the Clinician does NOT have a Clinician Contract in Status = Executed, then set:

** Status = ‘Canceled’

** Canceled Reason = ‘No Executed Contract’

- If the Clinician has a Clinician Contract in Status = Executed, BUT it does NOT contain the rate(s) that have been updated, then set:

** Status = ‘Canceled’

** Canceled Reason = ‘Clinician’s executed contract does not contain the rate(s) that have been updated.’ 

NOTE: Confirmed that the Clinician Contracting Specialist do have edit access o the CCM ‘Status' field, so if they want to include a record in their Agentforce process they can update the Status to ‘New’ (which will allow Agentforce to pick up and display the clinician record per [+PR10505583478+|https://salesforce.atlassian.net/browse/PR10505583478])

## Solution

**Class:** ClinicianContractSyncService

Add CCM logic block: 

1. Query CCMs: Process_Type = 'Agent'  AND Status = 'New'    

1. Query PO PCAs where Existing_Rate_Increased__c = TRUE (the rates that triggered Agent)

1. Query Clinician Contracts filtered to Status = 'Executed' ONLY 

1. For each Agent CCM:   

   **4.1** NO Executed Contract? 

         Status = "Canceled"    

         Canceled_Reason = "No Executed Contract"    

   **4.2** Executed Contract missing ANY of the PO shift types where rates increased?         

         Status = "Canceled"    

         Canceled_Reason = "Clinician's executed contract does not contain the rate(s)..."

         **IMPORTANT:** Look at what shift types got updated on PO (rates increased), then validate the clinician contract has all those exact shift types. If any shift type is missing, cancel. 

   **4.3** Executed Contract WITH matching rates? 

         Leave as "New"  (Agentforce picks up) 

-New Field-

Client_Change_Management__c.**Canceled_Reason__c** (Text)

**Note:** 

1. Add new field to respective PermissionSets **+ DataDictonary**

1. Add new field to Client Change Manager Layout/Flexipage (Next to Status)
