## Description

The DeactivateHPFsForClinician batch is correctly identifying clinicians past the 270day threshold. However, it is failing to respect the Provider Next Scheduled Date exception. Specifically, it is not removing clinicians scheduled within 365 days, nor is it flagging the "Recontract" field for those scheduled beyond 365 days.

Issue 1: Re-contract Flag Not Checked 

Steps to Reproduce:

1-Set Provider Next Scheduled Date to July 15, 2026 (374 days after LDW) on all sister sites.

2-Run Batch DeactivateHPFsForClinician.

3-View 3 Month HPF Inactivation Report.

Expected Result: Clinician appears on report with the Re-contract field checked (because Gap > 365 days).

Actual Result: Clinician appears on report, but Re-contract field is blank.

Issue 2: Records Not Removed from Report

Steps to Reproduce:

1-Set Provider Next Scheduled Date to May 1, 2026 (300 days after LDW / within 365 days) on all sister sites.

2-Run Batch DeactivateHPFsForClinician.

3-View 3 Month HPF Inactivation Report.

Expected Result: Clinician is removed from the report (because they have a scheduled date within the 365-day window).

Actual Result: Clinician still appears on the report despite the valid upcoming schedule.

Issue 3 - On Reports, the Column Staffing team should come.

Expected Result - The Column Name should appear on report.

Actual Report - As of now it is not coming on report.
