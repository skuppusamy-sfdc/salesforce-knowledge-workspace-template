## Description

As a Recruiter, I should not be able to create a new Account with record type = ‘Person Account’ (should be creating record type = ‘Clinician’) and if I am viewing an Account with record type = ‘Person Account’, then I should not be able to change the record type to ‘Clinician’, so that we ensure that when recruiters are manually creating a new account, they are correctly creating a ‘Clinician’ record type which triggers the auto creation of the Healthcare Provider record.

## Acceptance Criteria

AC1:

Given I am a Recruiter within the Corporate Support Apps

When I am on the **Accounts list views**

Then should NOT have the ability create a ‘New’ Account with record type = ‘Person Account’.

And I should only be able to create Account record type = Clinician (original story [+PR105055823+|https://salesforce.atlassian.net/browse/PR105055823])

AC2:

Given I am a Recruiter within the Corporate Support Apps

When I am on a **Account Record Type = ‘Person Account’**

Then should NOT have the ability “Change Record Type” and change to a ‘Clinician’.

Note: 

- User should be using the guided experience = 'Create Clinician' on the Contacts list view 

** Permission set = **Manage Clinicians*, gives Create, assign record type = Clinician.

## Solution

- Remove Person Account from the Assigned Record Types on the ATS Recruiter Perm Set - OK
