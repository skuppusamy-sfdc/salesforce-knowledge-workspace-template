## Description

**Tester:** Maci Broussard

**Role:** Recruiter

**Description:** 

Enhancement to follow Hypercare. Discussed with James B and Zach R. We need to re-evaluate the logic of system determining entity when a Clinician Contract Record is cloned; specifically when the user is adding / removing HPFs. 

Initial discussion we think we should:

- Only populate the Contracting Corporate Entity field for EEs (i.e. all match: field is populated OR mismatch: goes for Entity Review

- System clears field for entity evaluation when HPF removed or HPF added when record is cloned.

**Business Impact:**

Need a long term solution following Hypercare, considering the following short term and Hypercare solutions: 

- Short Term solution: business users are manually adjusting contract record entities when required.

- Hypercare solution: pending  clear field when contract is cloned ([+PR10505585707+|https://salesforce.atlassian.net/browse/PR1050558-5707])

- Impact: 1) Creation of Conga Agrmt Records, 2) How often a record is going for entity review, 3) When is it appropriate for the entity to be reviewed

**Design/solution Assumptions:** The “Contracting Corporate Entity” field data is correct in the Previous Contract that we will be copying over if criteria is met.

## Solution

**AC1: Start New Request — Move Evaluation to Submit**

  Changes:

1. Modify `Clinician_CreateContractAndPNC` (IP):

- Remove the checkCorporateEntityList / contractCorporatingEntity formula logic that pre-populates Contracting_Corporate_Entity__c.

- Always pass blank/null for Contracting_Corporate_Entity__c to DMCreateContract.

1. Enhance `ClinicianContractReviewActionHandlerFlow` (Flow):

- After iterating all PNC→HPF records (excluding Delete? = true):

- All same entity → Populate Contract.Contracting_Corporate_Entity__c with that value. Do NOT set var_AssignCEMQueue.

- Different entities → Leave field blank. Set var_AssignCEMQueue = true.

- This logic already partially exists in the Check_Corporate_Entity decision — it needs to be enhanced to also write back the entity value when all match.

1. No changes to `ClinicianContractReviewApprovalFlow` — it already routes to Corporate_Entity_Mismatch_Queue based on var_AssignCEMQueue.

**AC2: Start from Previous Active Contract — New Branching Logic (UPDATED)**

- At the time to clicking approval butting if If all HPF are cloned then copy the previous contract cope entity and no approval (Use the PNC.Clone From)

- At the time to clicking approval butting for new HPF Added (Use the PNC.Clone From is null) and if the Copr Entity is Same as Previous Contract.Copr Entity then new cloned Contact.Corp Entity = Previous Contract.Copr Entity Else send for approval

- If any HPF got deleted then send for approval

  **AC3: Continue with Draft / Discard and Start New — Router Logic**

  Problem: These paths may or may not have a Previous Contract. The evaluation must route to AC1 or AC2 logic based on context.

  Changes to `ClinicianContractReviewActionHandlerFlow`:

  ***Add a top-level Decision before the corporate entity evaluation:

- PreviousContractId is blank OR (PreviousContractId is populated AND Employment_Type__c != 'Employee') │ AC1 logic (simple all-same check)       

- PreviousContractId is populated AND Employment_Type__c = 'Employee'                                   │ AC2 logic (previous contract comparison)

  This is a single decision element that gates which evaluation branch runs.
