## Description

**Submitter:** Kristi Bouillion

**Role:** Data Quality

**Description:** 

When a service line account has changes to the following fields, QGenda Credentialing also needs to be updated. This is important as the data between the two systems needs to match, as if there is a variance, this could cause enrollment and billing loss of revenue.

Service Line Field updates to be sent to QGenda Credentialing Location Record:

•	Physical Address (street, city, state, zip code)

**Business Impact:**

Potential for loss of revenue if the service account/locations (our client) addresses are not correct.

**MuleSoft change description:**

New Integration needs to be built with an event trigger from Health Cloud (CDC / Platform event) for phone number update coming from Contact object to be sent to Qgenda (Cred Locations)

Mapping Doc: POST&PUT CredentialingLocation tab (row 1925)

## Acceptance Criteria

- A Platform Event to update QGenda Data is fired any time the following fields are updated on an existing Clinician Account

** Contact.HomePhone

** Contact.MobilePhone

- The platform event includes the

** Contact.Global_Provider_ID__c

## Solution

Platform Event Name:  Clinician Updates

Platform Event Fields:

- HomePhone

- MobilePhone

- Global Provider ID 

- AccountRecordId

When any of the fields is updated

- Create or Reuse a synchronous Brach for Account Record Update Flow

** When the Record Type = Clinician

** Entry Criteria: Account isUpdated && (IsChanged(HomePhone) || IsChanged(MobilePhone) AND there is at least one Contract for the Clinician in Executed Status (Contract.AccountId = [Clinicina.Id|http://Clinicina.Id] (Person Account)

** Send Platform Event with the Field and Value (ALL Fields Listed will be sent even when )

** Always send fields 

*** Global Provider ID

*** AccountRecordId
