## Description

As a system administrator, when an HPF Status is set to “Inactive” I want the system to update all the related HPFs for the same service account from Status = “Superseded” to “Inactive”, so that we have clean data within Health Cloud (HC) & Inactivation to Qgenda will be triggered (story 5648).

## Acceptance Criteria

Relates to 5792 (set HPF status= “Superseded”) & 5648 (update Name = HPFUpdates, Type = Inactive Date).  

Given I am system administrator

When an HPF Status set to “Inactive”

Then the system shall update all of the Clinician’s related HPFs for the same Service Account from Status “Superseded” to “Inactive”. 

And a Platform Event Name = HPFUpdates, Type = Inactive Date shall **NOT** be triggered.

## Solution

When HPF Status = Inactive

Query all HPF with Status = Superseded with PractitionerId = Current.HPF.PractitionerId and AccountId = Current.HPF.AccountId and Id != Current.HPF.Id

Update Status = Inactive

And a Platform Event Name = HPFUpdates, Type = Inactive Date shall **NOT** be triggered.
