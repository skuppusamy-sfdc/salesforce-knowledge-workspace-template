## Description

**Current State:**

- When a clinician forgets login credentials or the welcome email expires, recruiters must escalate to system admins for password resets.

- Clinicians often don't know their Experience Cloud username, making standard self-service reset flows ineffective.

**Business Impact:**

- Admin bottleneck for routine support requests; recruiter cannot resolve in real-time during onboarding calls.

**Requirement (Two Options – Investigate Feasibility):**

- Option 1: Add a button on the Clinician Account page that navigates the recruiter to the Experience Cloud password reset screen for that clinician.

- Option 2: Surface the clinician's Experience Cloud username on the Salesforce Clinician record so recruiters can initiate password resets without escalation.

- Jeff to investigate technical feasibility and recommend the preferred path.

**Suggested Acceptance Criteria (pending option selection):**

-◦     Given a recruiter is on a Clinician Account page, they can initiate a password reset or view/copy the clinician's Experience Cloud username.-

-◦     No admin escalation is required for routine password reset scenarios.-

-◦     Action is logged for auditing purposes.-

## Acceptance Criteria

As a Recruiter I can open a clinician account page and trigger an Experience Site password reset on behalf of the Clinician so I can provide them with a new registration email on-command

As a Recruiter I see a success message that indicates the email has been sent, the email it was sent to, and the username associated to the clinician. 

- Success Message:

Success! The password reset email has been sent to the clinician.

“User.Fullname” has had their password reset. The email was sent to “User.EmailAddress”. The username for this user is “User.Username”.

** Email:

*** Subject: Action required: reset your password

*** Body: 

Hello <insert User’s first name>,

We received a request to reset your password for Core Connect. To set a new password, click the link below. The link expires after 72 hours.

<Link to site>

If you have any questions about this message, contact your administrator. 

- Error Message for Portal User is NOT enabled:

Password reset is unavailable because the Customer Portal is currently disabled. Please enable portal access for this user.

- Error Message for Reset Password again within 24 hours OR Inactive User:

Failed! The password reset email couldn’t be sent to the clinician.

This may be because their password was reset within the last 24 hour or the user is inactive/frozen.

** Email ONLY sent if Password was reset within the last 24hrs: 

*** Subject: We can’t reset your Salesforce password

*** Body:  Your Salesforce administrator recently requested to reset the password for the username <insert username>. We can’t rest your password because it was changed within the last 24 hours.

If you have any questions, contact your Salesforce administrator.

As a Recruiter, a closed task shall be created when I successfully ‘Reset Password’ containing the following details: (success message was presented)

- Task record type = Default

- Subject = ‘Password Reset’

- Comments = “User.Fullname” has had their password reset. The email was sent to “User.EmailAddress”. The username for this user is “User.Username”.

- Status = 'Completed'

- Assigned To = <Initiator, user who clicked the “Password Reset” button>

- Priority = ‘Normal’

- Related To = <Clinician>

## Solution

1. New Action button on Clinician Person Account Page Layout named “Reset Password”

1. OnClick query the User record related to the Clinician person Account and store the user ID

1. Invoke `System.resetPassword(userId, sendEmail)`

1. Log an Activity Type Taskagainst the Person Account

Subject: Password Reset

Description: “User.Fullname” has had their password reset. The email was sent to “User.EmailAddress”. The username for this user is “User.Username”.

**Status: Completed**

Owner/Assigned To: Active User
