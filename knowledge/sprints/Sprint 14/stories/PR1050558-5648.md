## Description

As an Integration User, when any HPF is Inactivated (not from ‘Superseded’) and there are no other HPFs that are NOT in Status = ‘Inactive’ for the same Clinician and same Service Account, then I need a trigger from SF to update Qgenda.

If we are unable to create a Platform Event due to missing Termination Date, then I want the system to create an Integration Error log, so that there is a record of the failure for missing a Termination Date.

## Acceptance Criteria

Given I am an Integration User

When any HPF status becomes ‘Inactive’ (**previous Status was NOT ‘Superseded’**) 

And for the **same Clinician and same Service Account, there are NO HPFs in a status other than “Inactive”** 

Then check if the HPF 'Termination Date' is populated

- If yes, then send a Platform Event to Qgenda with that ID

** Platform Event Fields, Name = HPFUpdates

*** HPF Id

*** Global Provider Id

*** Global Facility Id

*** Inactive Date

*** Type = “Inactive Date”

- If no, then create an Integration Error Log, stating that the Termination Date is missing. 

NOTE: 

- Need to update the flow (HPF Updates Triggered Flow) to send a PE (original story 4840)

- Related to:

** Story 5792, which updates Status to “Superseded”

** Story 6235, which updates Status = “Superseded” to “Inactive”

## Solution

Update the Flow HPF_Updates_Triggered_Flow

|**Is_Inactive**|`Status__c` = **Inactive** **and** `Status__c` **IsChanged** = true

AND Previous `Status__c` !=**Superseded**

AND TerminationDate != null|→ Assign_PE_instance_for_Inactive_HPF|

Send the Platform Event

Scenario for Testing

1. Make sure there are multiple HPF for same service account with different facility titles
