## Description

**Current State:**

- When a clinician is moved between opportunities or facilities, the system may re-trigger the intake process even though the clinician has already completed it.

**Business Impact:**

- Clinician frustration and drop-off; recruiter credibility impacted; redundant work for clinicians already through onboarding.

**Requirement:**

- Once a clinician has a completed intake on record, the system should not re-trigger intake for subsequent opportunity movements.

- The system should recognize a single completed intake as valid across opportunities.

- A mechanism should exist to manually retrigger intake if a valid business reason requires it (e.g., outdated information, timebased expiration rule — rules TBD with business).

**Suggested Acceptance Criteria:**

-◦     Given a clinician with a completed intake is moved to a new opportunity/facility, the intake process is not retriggered.

-◦     Given a recruiter or admin needs to retrigger intake, a manual override or admin action is available.

-◦     Business rules for when intake should be considered "stale" and retriggered are documented and configurable.

## Acceptance Criteria

As a clinician I should only receive intake assessments I have not already recieved completed within the last 180 days as necessary for the current intake.

## Solution

Amend Intake Case creation logic to check for an existing Intake Case with a Completed Status for the same Clinician

If none is found, create new Case and assessmentEnvelopes as is done today.

If an existing Intake Case is Found, create a new intake case with only any Assessments required for this Intake not already assigned completed on the previous intake Case. Only consider past intake assessments that are created within the last 180 days. 

Example (Consider, a physician has moved to California and now needs to provide a CCPA Notice, Or Assessments are assigned completed but are more than 180 days old.)

If no new assessments are required for the new Intake update the case to Closed on creation.
