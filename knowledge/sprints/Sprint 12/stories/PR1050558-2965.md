## Description

As a Recruiter, when cloning a Clinician Contract, I want the system to copy over the associated Rate Pay Codes & track if a new rate pay code is added, so that changes from the original (Cloned From) and new contract can be tracked.

## Acceptance Criteria

AC1: Add “Cloned From” field to Rate Pay Code

Given I am an Recruiter within the Corporate Support apps

When I clone a Clinician Contract that has Rate Pay Codes associated (clone via the guided experience)

Then the system shall:

- Clone the related Rate Pay Codes to the newly created Clinician Contract in Status = Draft

- Populate the “Cloned From” field with the Rate Pay Code record it was cloned from.

AC2: Add “Is Newly Added?” field to Rate Pay Code

Given I am an Recruiter within the Corporate Support apps

When I clone a Clinician Contract (clone via the guided experience)

And the Status = Draft

And I add a new rate

Then the system shall pull the Rate Pay Code from the related Practice Operations contract’s rate

And update the field “Is Newly Added?” = True/Checked

## Solution

Applies to Contract.RecordType = Practice Operation 

AC1:

1. Add new Field Cloned_From__c in Rate_Pay_Code__c 

1. When the Clinician Contract is cloned based on [https://salesforce.atlassian.net/browse/PR10505583017|https://salesforce.atlassian.net/browse/PR10505583017|smart-link]  Set the Cloned_From__c 

AC2:

1. When a new records i created in Rate_Pay_Code__c and when Cloned_From__c = null then populate Is_Newly_Added__c = true
