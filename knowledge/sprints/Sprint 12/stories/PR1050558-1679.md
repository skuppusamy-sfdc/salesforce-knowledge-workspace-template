## Description

As an Implementation Specialist, when a PMT Project is created for the Opportunity and PMT Phase/Tasks have been created for each Service Account, I want the system to automatically link the matching Action Plan Template and trigger task creation, so that either individuals or teams are notified and can take the appropriate actions to complete the Client onboarding process.

## Acceptance Criteria

AC1: Auto create tasks 

Given I am a Implementation Specialist in the Corporate Support Apps

And I created a PMT Project for the Opportunity (RT: Opportunity Growth)

And I “Clone Phase/Tasks” for each Service Account

When I am on a PMT Task

And I update the “Task Status” = “In Progress”

Then the system shall automatically:

- Link the Action Plan Template with the same name as the “Task Name”

*** Populate the NEW read only (Layout) “**Related Action Plan*” lookup field with Action Plan Template

- Trigger the auto creation of the Action Plan task record(s).

** Populate **‘Start Date’* = <Date the PMT Task Status was set to “In Progress”>

** Populate **‘End Date'* = <Due Date of the Last task (find the task with the max/latest Due Date) based on the task created within the Action Plan Template>

** Populate ‘**Total Tasks*’ (NEW, read only, number) = <populate with number of tasks from Action Plan Template>

** Populate ‘**# of Completed Tasks*’ (NEW, read only, number) = <populate as Action Plan template tasks are completed update the number>

AC2: Add Action Plan related list to Service Account

Given I am a user with access to the Action Plans

When I navigate to an Account (RT: Service Account)

Then on the Related tab I can see the Action Plan Related List (add to bottom)

## Solution

Record Triggered Flow for Update in PMT Task object

1. When the Status of inov8__PMT_Task__c.inov8__Status__c = “In Progress”

## Get the PMT Task Name of the inov8__PMT_Task__c.Name and query for matching name in ActionPlanTemplateVersion to find the version Id

### Use SOQL as reference SELECT Id,ActionPlanTemplateId,ActivationDateTime, ActionPlanTemplate.ActionPlanType, ActionPlanTemplate.Name FROM ActionPlanTemplateVersion WHERE ActionPlanTemplate.Name = inov8__PMT_Task__c.Name AND Status = 'Final' Order By ActivationDateTime DESC LIMIT 1 

### If no records found then throw error message “Unable to find Published Action Plan Template with the name that matches PMT Task Name. Review Action Plan Template Setup and try again.” and stop the update.

## Create the Action Plan Record with fields below

### ActionPlanTemplateVersionId = ActionPlanTemplateId from ActionPlanTemplateVersion in above query

### ActionPlanType = ActionPlanTemplate.ActionPlanType from ActionPlanTemplateVersion in above query

### Name = ActionPlanTemplate.Name from ActionPlanTemplateVersion in above query

### Start Date = Today

### TargetId = inov8__PMT_Task__c.inov8__Phase__r.Service_Account__c

## Update the inov8__PMT_Task__c with related data

### **Total Tasks** (inov8__PMT_Task__c.Total_Tasks__c) = Select Count(Id) from Task WHERE Id in (

SELECT ItemId FROM ActionPlanItem WHERE ActionPlanId = [from step 1.b] and ItemEntityType = 'Task')

### **# of Completed Tasks** (inov8__PMT_Task__c.No_of_Completed_Tasks__c) = o (hard code)

### **End Date** (inov8__PMT_Task__c.inov8__Due_Date__c) = Select ActivityDate from Task WHERE Id in (

SELECT ItemId FROM ActionPlanItem WHERE ActionPlanId = '07r7x000000GxLtAAK' and ItemEntityType = 'Task') order by ActivityDate DESC LIMIT 1

### Start Date (inov8__PMT_Task__c.inov8__Start_Date__c) = Today

### **Related Action Plan** (inov8__PMT_Task__c.Related_Action_Plan__c) =  Get Action Plan Id once the Record is created in step b and store in inov8__PMT_Task__c.Related_Action_Plan__c

User should have access to Action Plan Template can only be able to perform the operation of making the Status = In Progress otherwise need to revisit how the flow is run (System Context)
