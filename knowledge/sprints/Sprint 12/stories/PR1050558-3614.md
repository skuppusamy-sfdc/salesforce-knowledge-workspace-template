## Description

As an Integration User, I need to know when a clinician contract is Executed, so that I can send clinician information to Okta to get a user created.

## Acceptance Criteria

This story updates original story [+PR10505583545+|https://salesforce.atlassian.net/browse/PR10505583545], updates in **bold**

AC1:

Given I am an Integration User in the HC org

When a Contract record where Record Type = ‘Clinician’ AND contract Status = ‘Executed’ **AND contract’s related HPFs Provider System Status = ‘Active’**  **(sent every time this criteria is met. MuleSoft will determine which Okta API to call)** **AND ‘Federation ID’ is not populated (send once per clinician)**

Then the system shall create the following record: (Update existing custom Platform Event) 

- Platform Event, Name = Clinician Onboarding

** Global Provider Id

** SSN (last 4 digits)

** Zipcode (Mailing Address)

** Phone (Mobile)

*** **Fed ID (add)*

**** **IF blank, set as null*

Assumptions:

- API handles duplicates

- We are only creating platform event when “Federation ID” is blank. Clinician Offboarding process should be clearing the Federation ID on the user record and deactivating the user in SF. - Per [+PR10505583573+|https://salesforce.atlassian.net/browse/PR10505583573]  -  SCP team will look into either automating or manually clearing the Federation ID when the Clinician deactivated in Okta.

WHEN Contract.RecordType = ‘Clinician’ and Contract.Status = Executed

send a platform event 

Name = Clinician Onboarding

- Global Provider Id → Contract.Account.Global_Provider_ID__pc

- SSN (last 4 digits) → Contract.Account.SSN_Last_4_Digit__pc

- Zipcode → Contract.Account.PersonMailingPostalCode

- Phone → Contract.Account.Mobile

** **Fed ID*  → Select FederationIdentifier from User where ContactId = Contract.Account.PersonContactId

** if blank then set as null per Aneesh
