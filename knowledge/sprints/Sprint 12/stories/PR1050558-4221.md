## Description

Steps to reproduce:

1. Login as System Administrator

1. Open a ATS applicant linked to an inactive clinician record who previously had access to Experience portal

1. Move the ATS Applicant record to ‘Awaiting Intake Form’ stage

1. Verify if the user record of the clinician is reactivated and a reset password email is sent to the latest email on person account

Expected Result:

User should be able to reset the password using the link in ‘Reset password’ email

Actual Result:

1. User receives a reset password and ‘Welcome to SCP Health Core Connect’ email

1. Link on ‘Reset password’ email redirects to Portal login page and throws an error ‘You do not have access to the Apex class named 'CommunityLoginController'.’ when email is entered

1. Link on ‘Welcome to SCP Health Core Connect’ email redirects to password reset page

## Root Cause

he issue is the user is getting 2 emails, first email says " welcome to scp " and is getting auto triggered when the user record is activated and in that there is already a link through which the user is able to reset their password.

The second email is the one which I am manually sending using system.resetPasswordWithEmailTemplate via Apex.... The problem with the second email is the user is not able to proceed further and it says they dont have access to 'CommunityLoginController' class.

After discussing with the Architects, There is a change in the way we handle emails
