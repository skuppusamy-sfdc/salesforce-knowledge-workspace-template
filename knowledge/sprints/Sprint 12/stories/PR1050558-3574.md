## Description

As a Clinician Contracting Specialist, when a Clinician contract is Executed and there is a future dated Practice Operations contract in Status = ‘Executed’, I want the system to automatically create a **Client Change Management** record, so that additional automation based on the 'Process Type' (Manual, Agent, Automation) can occur.

## Acceptance Criteria

AC1: Check for future dated, executed **Practice Operations** **contracts**

Given I am a Clinician Contracting specialist within the Corporate Support apps

When a Clinician contract Status is set to “Executed” 

Then for each of the Clinician contract’s active HPFs (HPF in status ≠ ‘Inactive’), I want the system to check for **Practice Operations** contracts meeting the following criteria:

- Future ‘Contract Start Date' & Status = ‘Executed’

And if:

- Match is found proceed to AC2

- Match is NOT found do nothing

AC2: Check for matching **Practice Operation Change** record

Given a Clinician contract Status is set to “Executed” 

When a matching Practice Operations contract is found in AC1

Then the system shall check for a matching **Practice Operation Change** record based on “Facility Title” of the Healthcare Practitioner Facility (HPF) record

And if:

- Match is found proceed to AC3

- Match is NOT found do nothing

AC3: Create **Client Change Management** record

Given a Clinician contract Status is set to “Executed” 

When a matching Practice Operations contract is found in AC1

And a matching **Practice Operation Change** record if found in AC2

Then the system shall create **Client Change Management** record
