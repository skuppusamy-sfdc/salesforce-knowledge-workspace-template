## Description

Issue - There can be multiple service lines in Action plan templates  Task, delete the tasks that do not have any of the service lines mentioned in Service account level.

But as of current behaviour : The system does text to text compare and deletes the task records that does not have exact match for the service lines in service account records.

Steps to Reproduce:-

1.Create a Service Account and ensure Service Line exists.

- Account team role : Implementation Project director.

2.Create Opportunity RT: Growth Opp. -Link to Service Account #1

- Related:create Services record. link to Service Account.

1. Create PMT project record.

- Associate PMT project Lookup on Opportunity to PMT Project & PMT project.

- Create Phases.

--Under phases, create tasks

1. Create Action plan template record. with the same name of Task name (#3)

- Create task under action plan template and ensure Service Line exists

- Publish the action plan template.

1. PMT project > Phases -> Update Service Account field.

1. All 5 tasks are created.

1. Now change the status of PMT Phase tasks to In Progress.

1. Now navigate back to Service account record and verify ACT Tasks are now displayed on Service Account record.

## Root Cause

The initial understanding was that an exact match of service lines between the account and tasks would be considered. However, upon further clarification, it was confirmed that the service lines will be provided as comma-separated values in the task level , and each service account will contain only one service line item. So modified the logic to handle this
