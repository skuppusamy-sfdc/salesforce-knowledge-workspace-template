## Description

As an Integration User, I need to know when a clinician contract is Executed, so that I can send clinician information to Okta to get a user created.

## Acceptance Criteria

AC1:

Given I am an Integration User in the HC org

When a Contract record where Record Type = ‘Clinician’ AND Status = ‘Executed’ AND ‘Federation ID’ is not populated (send once per clinician)

Create a custom Platform Event with the following:

- Platform Event, Name = Clinician Onboarding

** Global Provider Id

** SSN (last 4 digits)

** Zipcode

** Phone

Assumptions:

- API handles duplicates

- We are only creating platform event when “Federation ID” is blank. Clinician Offboarding process should be clearing the Federation ID on the user record and deactivating the user in SF. 

- Okta will update the User’s ‘Federation ID’ & 'Email'
