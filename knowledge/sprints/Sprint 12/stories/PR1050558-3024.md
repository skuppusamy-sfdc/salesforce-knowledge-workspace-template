## Description

As a Contracting specialist, when I am working with a Clinician contract, I want the ability to pull recent Practice Operations rates, so that I am working with the most up to date Client rates.

## Acceptance Criteria

AC1: “**Refresh Client Rates**”

Given I am an Contracting specialist within the Corporate Support Apps

When I am on a **Clinician contract in Status = Draft**

Then on the Summary tab, I have the ability to “***Refresh Client Rates****” for **all HPFs Type of Rates**; **except for 'Bonus'*

And for each of the Clinician’s rates, 

1. The system shall pull in the rate from the relevant Practice Operations contract and populate the “Facility Rate”. (Status = Executed AND Clinician Contract Start Date falls within the Practice Operations Contract Start Date) 

1. If the “Facility Rate” is greater than the current “Rate”, then system shall update the “Rate”.

1. If the system is unable to find a matching Facility Rate / Facility Rate has been removed on the Practice Operations contract, then the system shall **hard delete the rate** on the Clinician contract. (Rate = Preventive Care Agreement record & related Rate Pay Codes)

1. When the user selects “Configure Rates” button, then on the ‘Rate Details’ step they are able to to view the updated  ‘Facility Rate’ and ‘Rate’ (if #2 was met)

1. If Clinician contract Status ≠ Draft, then the system should hide/disable the “***Refresh Client Rates***” button. 

1. When the user clicks “**Refresh Client Rates**”, then the user can view the Last Synched Date as a timestamp on Summary page of Clinician contract.

## Solution

When the user clicks the “**Refresh Client Rates**”

Apex Code

1. For Each ProviderNetworkContract Record in the Clinician Contract

## Get the latest +relevant+ Practice Operation Contract 

### ProviderNetworkContract.Healthcare_Practitioner_Facility__r.AccountId = [Contract.AccountId|http://Contract.Id] and [**Contract.**|http://Contract.Id]**StartDate <= ProviderNetworkContract.Contract.StartDate**  AND  [Contract.|http://Contract.Id]RecordType= Practice Operation and [Contract.|http://Contract.Id]Status = “Executed” Order By [Contract.|http://Contract.Id]StartDate DESC LIMIT 1

### Get all the **Practice Operation** PreventiveCareAgreement record where ContractPaymentAgreement.Type__c = Stipend OR Shift OR Variable OR Target OR Leadership Incentives and Facility_Title__c = ProviderNetworkContract.Healthcare_Practitioner_Facility__r.Facility_Title__c. Loop through each records 

#### For ContractPaymentAgreement.Type__c = “Shift” 

##### use When Is_Deleted__c = false use PreventiveCareAgreement.Shift__r.Shift_Full_Name__c-PreventiveCareAgreement.Shift__r.Type__c, PreventiveCareAgreement.Rate__c, [PreventiveCareAgreement.Id|http://PreventiveCareAgreement.Id] (use a List<String, Map<Id, Number>>) **[Shift Active Records]**

##### use When Is_Deleted__c = true then use  PreventiveCareAgreement.Shift__r.Shift_Full_Name__c-PreventiveCareAgreement.Shift__r.Type__c, PreventiveCareAgreement.Rate__c, [PreventiveCareAgreement.Id|http://PreventiveCareAgreement.Id] (use a List<String, Map<Id, Number>>) **[Shift Deleted Records]**

#### For ContractPaymentAgreement.Type__c != “Shift”

##### use When Is_Deleted__c = false use PreventiveCareAgreement.Type__c, PreventiveCareAgreement.Rate__c (use a List<Id, Map<Id, Number>>) **[Other Type  Active Records]**

##### use When Is_Deleted__c = true use PreventiveCareAgreement.Type__c, PreventiveCareAgreement.Rate__c (use a List<Id, Map<Id, Number>>) **[Other Type Deleted Records]**

## Get all the **Clinician Contract** PreventiveCareAgreement record where Cloned_From__c != null AND ContractPaymentAgreement.Type__c = Stipend OR Shift OR Variable OR Target OR Leadership Incentives  and Loop through each records

### Process Shift Records

#### For **[Clinician]** ContractPaymentAgreement.Type__c = “Shift” use PreventiveCareAgreement.Shift__r.Shift_Full_Name__c-PreventiveCareAgreement.Shift__r.Type__c, PreventiveCareAgreement.Rate__c compare the Rates from Step 1.a.ii.1 and when the [Practice Operation].PreventiveCareAgreement.Rate__c > **[Clinician]**PreventiveCareAgreement.Rate__c then update the **[Clinician]**PreventiveCareAgreement.Rate__c **(use  [Shift Active Records] Map)**

#### If there are any shift that is not available in the Practice Operation Contract then delete the Shift from Clinician Contract **(Use  [Shift Deleted Records])**

#### Update the **[Clinician].**Facility_Rate__c to the [**Practice Operation.CAP**].Id **when its same** 

### Process Other Records

#### For **[Clinician]** ContractPaymentAgreement.Type__c != “Shift” use PreventiveCareAgreement.Type__c, PreventiveCareAgreement.Rate__c compare the Rates from Step 1.a.ii.2 and when the **[Practice Operation]**.PreventiveCareAgreement.Rate__c > **[Clinician]**PreventiveCareAgreement.Rate__c then update the **[Clinician]**PreventiveCareAgreement.Rate__c **(Use  [Other Type Active Records])**

#### If there are any ContractPaymentAgreement.Type__c that is not available in the Practice Operation Contract then delete from Clinician Contract **(Use  [Other Type Deleted Records])**

#### Update the **[Clinician].**Facility_Rate__c to the [**Practice Operation .CPA**].Id **when its same** 

## Once complete update today to Contract.Last_Synched_Date__c

Button should be visibly only for Draft Clinician Contract
