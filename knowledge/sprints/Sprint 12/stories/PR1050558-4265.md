## Description

### **Description:**

When withdrawing one job application, the system incorrectly withdraws **all active job applications**.

Both jobs belong to **different service accounts** and are **independent**, so withdrawing one should not affect the other.

### **Steps to Reproduce:**

1. Log in to the **HC Portal** as a user with multiple active job applications (each under a different service account).

1. Navigate to the **Job Applications** page.

1. Select one job application and click **Withdraw**.

1. Return to the **Home Page** or **Job Application List** view.

### **Actual Result:**

All active job applications are withdrawn after withdrawing just one job.

### **Expected Result:**

Only the **selected job application** should be withdrawn; other active applications should remain unaffected.

### **Test data :** [https://corehc-qa.sandbox.lightning.force.com/lightning/r/Account/001gP00000A8RWjQAN/view|https://corehcqa.sandbox.lightning.force.com/lightning/r/Account/001gP00000A8RWjQAN/view|smartlink]

## Root Cause

Initially, the condition was set incorrectly, causing the entire Job Application list to hide when a single application was withdrawn. The logic is now refined to hide only the withdrawn application, keeping all other records visible.
