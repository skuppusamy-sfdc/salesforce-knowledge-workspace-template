## Description

### **Description:**

Certain conditional sections and fields (as defined in the **Data Dictionary – SCP Health**) are not visible on **Client Contract** records in Salesforce, despite the Contract Record Type and Contract Type meeting the required display conditions.

This issue prevents users from viewing or populating key fields such as:

** **Termination Notice Issue Date*

** **Financial Model*

** **Volume Terms*

** **Non-Compete*

Steps 

** Log in as **System Administrator*

** Navigate to **Corporate Support Apps → Contracts tab*.

** Create a new **Client Contract* record OR open an existing one (Record Type = Client Contract).

** Update **Contract Type** field to **“Compensation”*/“Protection.”/“Insurance.”/“Termination.”/“Special Terms.”

- Verify the corresponding session visibility 

### **Expected Result:**

When **Contract Type** changes, the respective conditional **section layout** (Compensation, Protection, Insurance, Special Terms, or Termination) should appear dynamically, displaying the mapped fields per Data Dictionary definition.

### **Actual Result:**

The **conditional sections are not rendering** on the Client Contract record page. Fields such as _Termination Notice Issue Date, Financial Model, Volume Terms, Non-Compete,_ etc., remain hidden even though:

- Record Type = Client Contract

- Contract Type matches display condition
