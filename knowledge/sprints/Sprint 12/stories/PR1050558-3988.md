## Root Cause

The reported issue concerning **Global IDs containing commas** in Service and Person Accounts was not a system defect but stemmed from a **misunderstanding of the current data design and the future migration plan.**

** **System Logic Confirmed:** The presence of commas in the Global IDs was confirmed to be **intentional and by design* for the current stage of the project.

** **Misconception:* The issue was raised as a bug because the current data format did not align with the perceived final, clean structure.

** **Resolution:** The team clarified that the final resolution will be handled through a planned **data type change to Auto-number** before migration to production. Since the current behavior is intentional and the issue will be managed via a future migration step, the bug was marked as **"Won't Do"* and closed.
