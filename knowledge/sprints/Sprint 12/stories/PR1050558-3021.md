## Description

As a Legal team member, when I receive a Client Agreement Request from the Sales VP, I want to create a new Client Contract(s) so that I can capture the necessary information and track it through the execution process.

## Acceptance Criteria

AC3: **New** Client Contract [APPROVED]

Given I am a Legal Team Member within the Corporate Support Apps

When I am on a Case (record type = Client Agreement Request)

And Case Status = “Not Started” or “In Progress” 

Then from the Case record, I can initiate the “Configure Contract” guided experience (“Configure Contract”  is NOT available if Case Status = “Closed')

And the system shall allow the user to capture the following information:

- Contract Start Date (required)

- Contract End Date (optional)

- Contract Name (required)

- SCP Corporate Entity (optional, NEW lookup field NOTE SCP Corporate Entity should use party role relationship = Services Contracting Entity) 

** 10/22: REMOVED. Used to hold: Legal entity for SCP. Decision to move field to the PNC level where the user will manually input.  

- Agreement Category (required)

- Agreement Effective Date (optional) REMOVED, this lives on Term & Renewals, not Master

And the system shall allow the user to link Service Account record(s) that are related to the Opportunity’s Services that are in Status = Active

- And have NOT been already linked the Service Account record(s) to a Client Contract related to the Opportunity.  [Service Accounts will be linked to the Client Contract via Provider Network Contract records]

- And user must select at least one Service Account to proceed.

** Display - Service Account & Type

- And if NO service accounts, then display the following message, user will need to cancel out of the flow. 

** No available Service Accounts found. Please verify that your Opportunity has active Services or check if the Service Accounts are already linked to another Client Contract.

And the following navigation options shall be available:

- Cancel

** No contract record is created

** Guided experience is closed

** User is returned to the Case record

- Finish

** a New Contract (Record Type: Client Contract) is created with information captured and the following additional info:

*** Status = “Draft”

*** Contract Owner = <user who initiate the “Configure Contract”>

*** Account Name = <Account Name from the related Opportunity>

*** Source Opportunity = <related opportunity>

*** Pro Forma Model = <Pro Forma Model in Status = Approved for Contracting” from Opportunity>

*** Requested By = <opportunity Owner> 

*** Contract Type = “Master”

** Case Status set to “In Progress”

*** User to update Status to “Closed” & Closed Reason = “Completed” once done creating Client Contracts

** Guided experience is closed

** User is returned to the new Contract record

## Solution

Story will Create below items

1.  Contract RT: Client Contract 

1. PNC Records (with the Opportunity Service Selected)

- New Contract RecordType = Client Contract

** Contract Start Date: User Input

** Contract End Date (optional)

** Contract Name (required)

** SCP Corporate Entity Picklist (Select )-

** Agreement Category (required)

** Contract.SourceOpportunityId = [Opportunity.Id|http://Opportunity.Id] when the Contract is saved

** Contract.Requested_Date__c = Case.CreatedDate (part of solution for  [https://salesforce.atlassian.net/browse/PR10505582807|https://salesforce.atlassian.net/browse/PR10505582807|smart-link] )

** Contract.Contract_Type__c = “Master”

** AccountId = Case.Opportunity__r.AccountId

** Status = Draft

** Contract Owner = Current User 

** Pro_Forma_Model__c = [SELECT Id FROM Pro_Forma_Model__c WHERE Opportunity__c = Case.Opportunity__c AND Status__c = “Approved for Contracting” ORDER BY LastModified DESC LIMIT 1]

** Requested By = Case.Opportunity__r.Owner

- And have NOT been already linked the Service Account record(s) to a Client Contract related to the Opportunity.

** Query for SELECT Account__c FROM Service__c WHERE Status__c = “Active” AND Opportunity__c = Contract.SourceOpportunityId AND Id NOT IN (SELECT Related_Opty_Service_Id__c FROM ProviderNetworkContract WHERE ProviderNetworkContract.Contract.RecorType.DeveloperName = 'Client_Contract' AND ProviderNetworkContract.Contract.Contract_Type__c = “Master” AND ProviderNetworkContract.Contract.AccountId = Contract.SourceOpportunityId.AccountId _AND ProviderNetworkContract.Contract.Contract_Type__c.Status NOT IN (Expired, Abandoned, Terminated)_ )

** When Contract is created Setup ProviderNetworkContract.Related_Opty_Service_Id__c = Service__c.Id ( only for the Parent Contract Type = ‘Master’)

When the Contract Records is created update the Case.Status = “In Progress”

Provider Network Contract Record created with 

ProviderNetworkContract.Service_Account__c = Service__c.Account__c

ProviderNetworkContract.Related_Opty_Service_Id__c = Service__c.Id

ProviderNetworkContract.HealthcarePayerNetworkId = Select Id from HealthcarePayerNetwork WHERE Name = “**Corporate Entity TBD**“

ProviderNetworkContract.HealthcareProviderId = Select Id from HealthcareProvider Where AccountId = Service__c.Account__c)
