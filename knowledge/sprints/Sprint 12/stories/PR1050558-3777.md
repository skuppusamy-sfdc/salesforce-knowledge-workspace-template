## Description

As a system administrator, I want to ensure that when users are working with cloned Practice Operations contracts they are not able to hard delete shifts, rates, and rate pay codes, so that we will have traceability of what was removed from the contract.

## Acceptance Criteria

-AC1: “Cloned From” **is NOT** populated, **hard delete** records [Covered under split story [+PR10505582988+|https://salesforce.atlassian.net/browse/PR1050558-2988]]

AC2: “Cloned From” **is** populated, **soft delete** records [Part of AC Covered under split story [+PR10505582988+|https://salesforce.atlassian.net/browse/PR10505582988]]

Given I am system administrator within the Corporate Support apps

When a SFO is working on a **cloned** contract (RT: **Practice Operations**)

And contract status = Draft

Then the user shall have the option to select the Delete option on a **Shift** record, if Deleted & “Cloned From” is populated:

- System shall update the “Is Deleted” field = True/Checked

- System shall NOT allow hard deletes

- System shall hide/prevent users from accessing the delete option if the contract status is ≠ Draft

** System shall update any related Preventive Care Agreement (**PCA**) records and **Rate Pay Code* record(s) field “Is Deleted” = True/Checked 

And if a Shift “Is Deleted” field = True/Checked, the user has the ability to **“Re-add”** 

- User shall update the “Is Deleted” field = False/Unchecked

** User shall update any related Preventive Care Agreement (**PCA*) records field via ‘Readd’ on Summary tab (then system will update “Is Deleted” = False/Unchecked)-

** User has the ability to **“Re-add”* the Shift (then system will update “Is Deleted” = False/Unchecked)

*** If Shift is updated to “Is Deleted” = True/Checked, then “Re-add” will be **hidden on related Rate Pay Codes(s) and Preventive Care Agreements (PCA) records.*

*** If Shift is updated to “Is Deleted” = False/Unchecked, then the User has the **ability to “Re-add”** **any related Rate Pay Codes(s) and Preventive Care Agreements (PCA) records*. 

-Then the user shall have the option to select the Delete option on a **Rate Pay Code** record, if Deleted & “Cloned From” is populated:-

- System shall update the “Is Deleted” field = True/Checked

- System shall NOT allow hard deletes

- System shall hide/prevent users from accessing the delete option if the contract status is ≠ Draft

** System shall update any related Rate Pay Code (**RPC*) records field “Is Deleted” = True/Checked

-And if a Rate Pay Code (**RPC**) “Is Deleted” field = True/Checked, the user has the ability to **“Readd”,** if Readded:- 

- System shall update the “Is Deleted” field = False/Unchecked

Then on the Summary tab, the user shall have the option to select the Delete option on a **Rate and** **Type of Rate** record(s), if Deleted & “Cloned From” is populated: 

- System shall NOT allow hard deletes

- System shall hide/prevent users from accessing the delete option if the contract status is ≠ Draft

** If the user deletes a **Rate**, then System shall update the related Preventive Care Agreement (**PCA*) record field “Is Deleted” = True/Checked [Existing functionality to delete and “Readd”]-

** And displays record under “Removed” 

** If the user deletes a **Type of Rate**, then the system shall update all the related Preventive Care Agreement (**PCA**) record(s) & **Rate Pay Code** records field “Is Deleted” = True/Checked, within the selected Contract Payment Agreement (**CPA*) record 

** And display Rate record(s) under “Removed” 

And if a Rate (PCA) is “Is Deleted” field = True/Checked, the user has the ability to **“Re-add”** (then system will update “Is Deleted” = False/Unchecked)

** If PCA is updated to “Is Deleted” = True/Checked, then “Re-add” will be **hidden on related Rate Pay Codes record(s).*

** If PCA is updated to “Is Deleted” = False/Unchecked, then the User has the **ability to “Re-add”** **any related Rate Pay Codes record(s).* 

**QA Testing Note: ‘Delete icon’ and ‘Re-add’ should ONLY be available when the Contract status = Draft**
