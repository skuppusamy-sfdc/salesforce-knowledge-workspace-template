## Description

As a Legal team member, when tracking the progress of a new Letter of Intent contract, I want to include tracking key dates so that the status of the contract is updated.

## Acceptance Criteria

AC1: Letter of Intent contract Statuses

Given I am a System Administrator within the Corporate Support Apps

When I on Contract Record Type = Letter of Intent

Then the available ‘Status’ values are: (read only on layout. NOTE: system admin should have the ability to edit. can’t make it read only because it’s required. )

- Draft [Status Category: Draft] (Default value)

- Submitted [Status Category: In Approval Process]

- Executed [Status Category: Activated]

- Superseded [Status Category: Canceled]

- Abandoned [Status Category: Canceled]

- Expired [Status Category: Contract Expired]

- Terminated [Status Category: Contract Terminated]

## Solution

- Update Contract (RT:Letter of Intent)

** Field = Status - update the values

** Read only on page/layout

Record Triggered Flow on Case RT: Letter of Intent

Requested Date → CreatedDate

Date Sent For Internal Review → When Contract.Status = “Submitted” update the date 

Other Party Returned Date → User Updated (Manual)

Outstanding Days → See Data Dictionary

Executed Copy Mailed Out Date  → User Updated (Manual)

Activated Date, Activated By → handled as part of [https://salesforce.atlassian.net/browse/PR10505583468|https://salesforce.atlassian.net/browse/PR10505583468|smart-link] 

Date Distributed → User Updated (Manual)

Total Process Days → See Data Dictionary
