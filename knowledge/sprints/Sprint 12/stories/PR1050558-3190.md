## Description

As a SFO, I want the system to automatically update all related Clinician executed Contracts based on specific type of updates made to the **Practice Operations** contract record, so that Clinician Contracts stay in sync with the Client contract and contract generation in Qgenda can be appropriately triggered. 

** **Coverage Change  Only Shift changes*: no new shift record created. New Shift is add with the same Type and same Rate. [See story [+PR10505582916+|https://salesforce.atlassian.net/browse/PR1050558-2916]]

** **Pay Code Change*: Only paycode change, new paycode added to shift (no rate change) [See story [+PR10505583187+|https://salesforce.atlassian.net/browse/PR10505583187]+]+

## Acceptance Criteria

AC1: Coverage Change  new shift with same shift type and same rate [See story [+PR10505582916+|https://salesforce.atlassian.net/browse/PR1050558-2916]]

AC2: Rate Pay Code Change  New pay code added to Shift (no Rate Changes)  [See story [+PR10505583187+|https://salesforce.atlassian.net/browse/PR1050558-3187]+]+

AC3: Platform event trigger for HPFTrigger (HPFUpdates__e) for Qgenda

Given I am SFO within the Corporate Support Apps

When a **Clinician Contract** record gets automatically updated based on AC1 or AC2

Then the system shall generate a platform event trigger for HPFTrigger (HPFUpdates__e) for Qgenda for the Clinician HPFs that got automatically update. 

And the system shall support 3187 Rate Pay Code delete scenario.

## Solution

As part of the same batch class (related to the associated stories 2916 and 3187),

When either of the conditions (of ac1 or ac2) satisfies, 

Create a platform event for each HPF of the clinician contract (using the PNC>HPF>Account = PO Contract.Account (Service account) record) that is part of Client Change Management (from where the job started)

Contract Id = Clinician Contract Id (where the update was performed)

 FirstScheduleddate  = HPF.First_Scheduled_Date__c, 

global facilityId = ServiceAccount.GlobalFacilityId,  

GlobalProviderId = Clinician.GlobalProviderId, 

HPF Id = HPF Id

Type = “Rate for HPF”

check [https://salesforce.atlassian.net/browse/PR10505582827|https://salesforce.atlassian.net/browse/PR10505582827|smart-link]  for fields related to Platform Event
