## Description

As a implementation specialist, I need pre-defined SUN Orchestration task template to be created associated to the Client onboarding, so that I can continue the SUN process as part of client onboarding

## Acceptance Criteria

AC1: Create PMT Case for SUN  Preliminary’, ‘SUN  Update’, or ‘SUN - Complete’ 

Given I am a Implementation Specialist within the Corporate Support Apps

When a PMT project exists

Then from the **PMT Phase** I have the ability to create a **PMT Case** for: ‘SUN  Preliminary’, ‘SUN  Revision’, or ‘SUN - Final’ (3 actions/buttons) 

And system shall check if “Service Account” field is populated on PMT Phase: 

- If yes, then proceed

- If no, user can not proceed

** Display error: ‘Service Account’ field must be populated to proceed

** Available navigation actions:

*** Cancel

**** Guided experience is closed

**** User is returned to the PMT Phase

**** No case or tasks are created

And if Service Account is populated, then the system shall display:

- Message: By clicking Finish the system shall automatically create new tasks & send the <Dynamically populate based on selection: ‘SUN  Preliminary’, ‘SUN  Revision’, or ‘SUN - Final’ > email.  If there are any open tasks, they will be automatically closed and new tasks shall be created.

And available navigation options: 

- Cancel

** Guided experience is closed

** User is returned to the PMT Phase

** No case or tasks are created

- Finish

** Guided experience is closed

** User is returned to the PMT Phase

And upon clicking Finish: 

** If the **PMT Case** for the selected Type **does Not exist* then: 

** New PMT Case is created [Split/covered under story [+PR1050558905+|https://salesforce.atlassian.net/browse/PR1050558905]]

**** Case Record Type = ‘Client Onboarding’ (**NEW*)

*** Subject = <Type> for <Account Name (Service Account)>

*** Case Owner = <user who initiated the case>

*** Type = <based on button selected: ‘SUN  Preliminary’, ‘SUN  Revision’, or ‘SUN  Final’ >-

*** Status = 'Not Started' (NOTE: Case and task creation will be in a single flow, so you will not see “Not Started”, it will get to set to “In Progress” per AC 2)

*** Case Origin = ‘Manual’

*** Priority = ‘Medium’

*** PMT Phase = <PMT Phase>

*** Account Name = <'Service Account' from PMT Phase>

** Tasks are created based on story [+PR1050558886+|https://salesforce.atlassian.net/browse/PR1050558886]

** If the **PMT Case** for the selected Type **does exist* then:

** NO new PMT Case is created

** Existing Tasks in Status = 'Open' are automatically updated to Status = ‘Canceled’

** New Tasks are automatically created based on story [+PR1050558886+|https://salesforce.atlassian.net/browse/PR1050558886]

- And send the SUN Email notification [Covered under story [+PR1050558887+|https://salesforce.atlassian.net/browse/PR1050558887]+]+

AC2: Auto Update the PMT Case Status 

Given I am a Implementation Specialist within the Corporate Support Apps

When a **PMT Case** for: ‘SUN  Preliminary’, ‘SUN  Revision’, or ‘SUN - Final’ has been created for a PMT Phase

Then the system shall automatically update the PMT Case Status:

- Update ‘Status’ = ‘In Progress’, when there are open related tasks (Status = ‘Open’)

- Update ‘Status’ = ‘Closed’, once ALL related tasks are updated to closed (Status = ‘Complete' or ‘Canceled’)

AC3: Ability to view Tasks from the PMT Case

Given I am a Implementation Specialist within the Corporate Support Apps

When I navigate to a **PMT Case** for: ‘SUN  Preliminary’, ‘SUN  Revision’, or ‘SUN - Final’ 

Then I have the ability to view the related tasks

Design Note: in order to set the task ‘Assigned To’ based on Team Role, we need to automatically create ‘Case Team' based on the related Account Team.  On the Client Onboarding case, we had to add the 'Case Team’ tab. Tabs = Details (Default), Action Plan, Case Team.

## Solution

Action Plan Template for SUN Tasks (built in separate [user story|https://salesforce.atlassian.net/browse/PR1050558-886])

Create the action Buttons in PMT Phase 

1. ‘SUN - Preliminary’

1. ‘SUN - Revision’

1. ‘SUN - Final’

Upon clicking the button check if the Service Account (inov8__PMT_Phase__c.Service_Account__c) 

- is blank “Error as per AC” 

- if Service Account is available then check if 

** Already a case for that PMT is available for that type then do not create a new case else create a new case based on button selected as per AC

- Check if there are open Task for Action Plan for the Case

** when exist Close the all the Task

- Create a new Action Plan as per AC

- Rest the Case Status = Not Started

Case Team Member is copied from the Account Team Role

Flow to associate respective Action Plan Template at ‘SUN Status’ change and set the new status

-Custom field on Client onboarding CASE  “SUN Status“  picklist: Not Started, Ready, In Progress, Completed-
