## Description

As a implementation specialist, I want all tasks that are created as part of the Client Onboarding process to be Record Type = Default, so that users can appropriate access and view tasks.  (related to story 1679)

## Acceptance Criteria

This story enhances what was built in story 1679 and ensures the record type is correctly set.

AC1: Auto create tasks with record type = Default

Given I am a Implementation Specialist in the Corporate Support Apps

When I am on a PMT Task

And I successfully update the “Task Status” = “In Progress” 

Then the tasks created via the Action Plan shall be Task Record Type = Default.

## Solution

Note: Since the Record Triggered flow is running in System User the default RT is created as Sales which was not correct.

Update the flow PMT Task : Action Plan and Record Updation Flow   

Update All the Task Record Type that is selected as part of this step to Task.RT: Default
