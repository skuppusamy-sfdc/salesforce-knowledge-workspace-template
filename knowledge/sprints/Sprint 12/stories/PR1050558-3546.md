## Description

As an Active Clinician, if I have been provisioned Okta access, then I need to login to the Health Cloud portal through SSO, so that I can easily login without using personal email and password to login

## Acceptance Criteria

AC1: Clinician **with** a Federation ID

Given I am an Clinician with an active user record where Federation ID (Is Single Sign-On Enabled = true) on my user record **is** populated

When I navigate to the HC Portal login screen

Then I shall have the ability to input an email address

And if I am an active Clinician with a Federation ID (Is Single Sign-On Enabled = true), then I can Sign in with Okta to access the HC Portal

AC2: Clinician **without** a Federation ID

Given I am a Clinician with an active user record where Federation ID (Is Single Sign-On Enabled = true) on my user record **is** **NOT** populated

When I navigate to the HC Portal login screen

Then I shall have the ability to input an email address

And if I am an active Clinician without a Federation ID (Is Single Sign-On Enabled = true), then I can Sign in with the Password I manually created.

NOTE: per Jose, Okta will NOT update the User’s ‘Email’, Okta will just update the ‘Federation ID’.

## Solution

Input text box - email address ( with the placeholder username@scp.com) 

Apex/Logic - to query for user based on email address. 

If user is found and fedId is present, redirect to okta sso url,

If user is found and FedId is not present, display password - perform Authentication via apex (site login method)

If user is not found, display error message “User not found”

Administrator > Login & Registration - Change the login page type to experience builder page.

Add custom lwc on the login page 

Custom lwc - displays email address input first.

based on logic mentioned above, display password and redirect to Exp cloud home page on successful login

or redirect to SSO okta (check with SSO okta if the redirect happens to exp page home page)

`Auth.AuthToken token = Site.login(username, password, System.URL.getSalesforceBaseUrl().toExternalForm());`
