## Description

As a Legal team member, when I receive a Client Agreement Request from the Sales VP, I want to create a new Client Contract(s) so that I can capture the necessary information and track it through the execution process.

## Solution

AC1: Set the Status value for Record Type = Client Contract 

AC2: 

Requested Date: When the Contract is initially created from Case using “Configure Contract” (handled as part of [https://salesforce.atlassian.net/browse/PR10505583021|https://salesforce.atlassian.net/browse/PR10505583021|smart-link] ) button populate Contract.Requested_Date__c = Case.CreatedDate **(NO BUILD IN THIS STORY)**

Record Trigger Flow for Update when Contract Record Type = Client Contract and the field is initially populated

1. Date Sent For Internal Review: Contract.Date_Sent_For_Internal_Review__c is updated Set Contract.Status = Submitted, Contract.Category__c = External

1. Other Party Sent Date: Contract.Other_Party_Sent_Date__c is updated Set Contract.Status = Submitted, Contract.Category__c = External

1. Activated Date: Contract.ActivatedDate AND Contract.ActivatedById IS Not Null then Contract.Status = Executed
