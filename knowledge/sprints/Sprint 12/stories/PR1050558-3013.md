## Description

As a System Administrator, when a new clinician user is create, then the standard SF email shall be sent with their user name and prompt the clinician to setup their password.

## Acceptance Criteria

AC: Email Template: Experience Cloud: New Member Welcome Email

Given I am a System Administrator

When a new clinician User is created:

- Manual creation - ‘Enable Customer User’ (811)

- Auto creation - ATS Applicant Status = Awaiting Intake Form (1789)

Then the system shall send: 

- Sender: [No-Reply Address]

- Subject: Welcome to SCP Health [Community Name]

- Body: 

Dear [First Name],

Welcome to SCP Health [Community Name]!

To get started, visit **[Community URL]** and log in with:

** **Username:* [Username]

Thank you,

[Organization Name]

----

(NOTE: Community Name = ‘Core Connect' & Organization Name = 'SCP Health’)

Assumption: Welcome Email are only sent when a NEW user is created.

## Solution

Out of the box functionality 

When a new user is created, the welcome email is sent out ( as long as the generate email checkbox is checked on the user)

Ensure the community URL is a link to the word “Community” ( if this is possible)
