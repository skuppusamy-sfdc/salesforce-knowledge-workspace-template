## Description

As a System Administrator, I need to setup access and fields, so that the PMT tool can support our Client Onboarding process.

## Acceptance Criteria

AC1: Create PMT Project

Given I am a Implementation Specialist within the Corporate Support Apps

When a user has access to **create a PMT Project**

-Then there should be a related list to the Opportunity-

- Filter: Record type: Opportunity Growth

- Columns to display: Opportunity Name, Stage, Target Contract Start Date, Closed Date, Incremental Margin

-And the following tabs visible: Chatter & Activity, Details, Summary, Tasks, Files, Related-

User can see have the ability to update the Opportunity in the PMT Project Record using the Opportunity Lookup in Details Tab

When the user relate the Opportunity in the PMT Project the corresponding Opportunity PMT Project gets automatically update by the system

AC2.1: PMT Project - Clone Phases/Tasks

Given I am a Implementation Specialist within the Corporate Support Apps

When a user has access to **create a PMT Project**

Then from the PMT Project record they should have the ability to “Clone Phases/Tasks” when “IsTemplate”= False

AC2.2: PMT Project - Use this Template 

Given I am a Implementation Specialist within the Corporate Support Apps

When a user has access to **create a PMT Project**

Then from the PMT Project record they should have the ability to “Use this Template” when “IsTemplate”= True

AC3: PMT Phase

Given I am a Implementation Specialist  within the Corporate Support Apps

When a user has access to create **PMT Phase**

Then on the PMT Phase record there should be a field:

- Service Account (lookup to Account RT: Service Account) 

And Service Account that is added is related to the opportunity, if not display message:

- Error Message: Service Account must be related to the Opportunity. Service Account must be related to the Opportunity Service and must be in Active Status.

And when the “Service Account” is successfully added, then the system shall update the “Phase Name” field with the Service Account Name.

AC4: PMT Task

Given I am a Implementation Specialist within the Corporate Support Apps

When a user has access to create PMT Task

Then on the PMT Task record the user should not be able to update the “Task Status” = “In Progress” if the PMT Phase is missing a “Service Account”.

- Error Message: Please identify the Service Account associated with the PMT Phase.

And “IsTemplate?” = False/not checked, then do not allow users to edit the PMT Task field = “Task Name”

- Error Message: You cannot edit the Task Name (Milestone).

AC5: View access for internal Corporate Support Users

Given I am a internal Corporate Support User

When I am in the Corporate Support Apps

Than I have the ability to view: PMT Project, PMT Phase, PMT Task, & PMT Gantt View

And I shall have **read only** access to PMT Project, PMT Phase, & PMT Task records
