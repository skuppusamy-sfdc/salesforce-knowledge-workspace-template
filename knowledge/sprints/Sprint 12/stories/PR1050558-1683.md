## Description

As a legal team member, I want the system to automatically create Facility Offboarding case(s) for each Service Account, so that we can track and ensure appropriate offboarding activities have been completed.

## Acceptance Criteria

AC1: Parent Facility Offboarding Case(s)

Given I am a Legal team member within the Corporate Support apps

When I complete the “Begin Facility Termination” guided experience [Story [+PR1050558317+|https://salesforce.atlassian.net/browse/PR1050558317]]

Then for **each** selected **Service Account (PNC),** a Parent Facility Offboarding Case shall be created: (do not create duplicates; update existing open case.)

- Record Type = Client Offboarding

- Status = Not Started

- Priority = Medium

- Origin = Manual

- Type = Facility

- Subject = "Facility Offboarding-<Account Name>"

- Case Owner = Account Team Role: VP Operations

- Account Name = <Service Account>

- Parent Case = BLANK

- Contract = <Contract RT: Client Contract> 

- Termination Request Date = <PNC - Client Termination Date>

- Termination Request Time = <PNC - Client Termination Time>

- Provider Network Contract = <PNC>

And if there are **NO Account Team role member**, then assign to **user who initiated** the “Begin Facility Termination” guided experience so they can manually reassign the case as needed.

And if there are more than one **Account Team role member**, then assign to the **most recently added** member. 

AC2: Child Facility Offboarding Cases

Given I am a Legal team member within the Corporate Support apps

When I complete the “Begin Facility Termination” guided experience [Story [+PR1050558317+|https://salesforce.atlassian.net/browse/PR1050558317]]

And for **each** Parent Facility Offboarding Case that is created [AC1]

Then the system shall automatically create the following child cases:

- Child Case Subject: Clinician Review

** Case Owner = Clinician Experience Specialist

- Child Case Subject: Client Systems Access

** Case Owner = Hospitalist Practice Manager

- Child Case Subject: Client Operations Close Out

** Case Owner = VP Operations

- Child Case Subject: Client IT Operations Close Out

*** Case Owner = IT Infrastructure Queue (**NEW*)

- Child Case Subject: Equipment Return & Proprietary Materials 

** Case Owner = Hospitalist Practice Manager

And child cases are populated with the following:

- Record Type = Client Offboarding

- Status = Not Started

- Priority = Medium

- Origin = Manual

- Type = Facility

- Account Name = <Service Account>

- Parent Case = <Parent Facility Offboarding Case>

- Contract = <Contract RT: Client Contract> 

And if there are **NO Account Team role member**, then assign to **user who initiated** the “Begin Facility Termination” guided experience so they can manually reassign the case as needed.

And if there are more than one **Account Team role member**, then assign to the **most recently added** member. 

AC3: 

Given I am a Internal Corporate Support user within the Corporate Support apps

And I have access to view the Case (RT: Client Offboarding) records & related tasks

Than on the a Case (RT: Client Offboarding) record, I can see the Activities component so that I can view open/upcoming tasks related to this case.
