## Description

**Submitter:** Kristi Bouillion

**Role:** Data Quality

**Description:** 

When a service line account has changes to the following fields, QGenda Credentialing also needs to be updated. This is important as the data between the two systems needs to match, as if there is a variance, this could cause enrollment and billing loss of revenue.

Service Line Field updates to be sent to QGenda Credentialing Location Record:

•	Physical Address (street, city, state, zip code)

**Business Impact:**

Potential for loss of revenue if the service account/locations (our client) addresses are not correct.

## Acceptance Criteria

- A Platform Event to update QGenda Data is fired any time the following fields are updated on an existing Facility Account

** Account.BillingStreet

** Account.BillingCity

** Account.BillingstateCode

** Account.Billingstate

** Account.BillingZipPostalCode

** Account.County__c ([Reg Agent County|https://corehc.my.salesforce-setup.com/lightning/setup/ObjectManager/Account/FieldsAndRelationships/00N37000005qtNc/view])

** Account.BillingcountryCode

** Account.Billingcountry

** Account.Name

- The platform event includes the all the field values when any one of the field(s) listed above changes
