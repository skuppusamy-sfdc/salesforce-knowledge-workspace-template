## Description

Adjust HPF Validation Rule to apply only  Insurance_Provided_By__c changes and not all the times.

!image20260320170244.png|width=281,alt="image20260320170244.png"!

## Acceptance Criteria

Given I am a user in the Corporate Support app

And I have Permission Set Group = 'Data Quality' OR ‘Risk Management Group’

When I am on a HPF record

And I edit the the “Insurance Provided By”

Then the system shall validate that:

- If Facility Title ≠ "External Locums"

** Then “Insurance Provided By” =  "SCP Insured Locums" or "Locum Company"

- If Facility Title = "External Locums"

** Then “Insurance Provided By” = "SCP", "Commercial", or "Hospital"

And Error Message if criteria is met: The selected value is invalid for this clinician’s HPF

----

NOTE: there is another value = ‘Self Insured’, but I do not currently have any rules in place for that value.

## Solution

[~accountid:61f8618e8d9e3c00688e380b] provided the following update criteria

--

AND(

  ISCHANGED(Insurance_Provided_By__c),

  OR(

    AND(

      Facility_Title__r.Name <> "External Locums",

      OR(

        ISPICKVAL(Insurance_Provided_By__c, "SCP Insured Locums"),

        ISPICKVAL(Insurance_Provided_By__c, "Locum Company")

      )

    ),

    AND(

      Facility_Title__r.Name = "External Locums",

      OR(

        ISPICKVAL(Insurance_Provided_By__c, "SCP"),

        ISPICKVAL(Insurance_Provided_By__c, "Commercial"),

        ISPICKVAL(Insurance_Provided_By__c, "Hospital")

      )

    )

  ),

  NOT(ISBLANK(Facility_Title__c))

)
