## Description

Given that Net new PO contract is created via Guided flow, with Coverage change = true. then User is not able to add any shift record on the contract as per the existing development.

### Test Steps

1. Launch the “**Request Contract Change”** Guided Experience, Select Yes to Coverage change, Add Start date & Description. 

1. Click on Next Button & Select Start New Contract & Click on Finish button. (New PO contract will be created.

1. On New **PO Contract**, Go to Related &  Try to create Shift Record.

**Expected :**  User should able to add shift rate on contract.

**RCA / Suggestion :** User should not be able to create first  contract with coverage change = True.
