## Description

**Tester:** Murphy Alkhatib

**Role:** Recruiter

**Description:** 

This situation arises when a clinician has previously worked with the organization and has an existing HPF, but due to data migration or the process of moving the clinician through the Kanban job application stages, a new HPF is created. 

If a clinician is moved through the Kanban process, it can inadvertently generate additional HPFs, leading to duplicates. This was particularly highlighted when a clinician had two HPFs with different effective dates, which caused confusion regarding which one to use for contract requests.

**Steps to Reproduce:**

Recruiter requested agreement,  two HPFs with different effective dates, which caused confusion regarding which one to use for contract requests.

**Expected vs. Actual:**

It caused confusion on what to choose and which rates etc.
