**Description:** 

The databridge view is not returning any data for this field, which is causing downstream issues when that field is being used in filters. 

SELECT	p.Global_Facility_ID__c, p.Global_Provider_ID__c, Agreement_Signed_Date__c

FROM	Salesforce_Live.dbo.Provider_Facility_Associations__c p

WHERE

	p.IsDeleted = 0

	AND p.Hospital_Priviledged_Date__c IS NULL

	AND p.Agreement_Signed_Date__c IS NOT NULL

	AND p.Status__c NOT IN ( 'Inactive' )

	AND p.Agreement_Signed_Date__c = '20260126' /** sample date for validation **/

SELECT rt.DeveloperName, rt.Name, c.CustomerSignedDate, c.*

FROM UnifiedPlatformData.COREHC.Contract c JOIN

	UnifiedPlatformData.COREHC.RecordType rt with(nolock) ON [rt.id|http://rt.id] = c.RecordTypeId

WHERE c.CustomerSignedDate IS NOT NULL

-WHERE CAST(CustomerSignedDate AS DATE)  = '202601-26'

AND rt.DeveloperName != 'Carrier_Contract'

ORDER BY c.CustomerSignedDate desc

Data migration issue - Customer Signed Date field value missing on contract
