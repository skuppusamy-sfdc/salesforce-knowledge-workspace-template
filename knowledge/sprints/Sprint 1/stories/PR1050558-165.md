## Description

As an IT user, I need to have Permission Sets configured, so that users have the correct access to objects and fields within Salesforce

## Acceptance Criteria

Given that I am an IT user

When I open Setup and view the Permission Sets page

Then I can see that Permission Sets were created to match the [Security Matrix sheet|https://schumachergroup.sharepoint.com/:x:/r/teams/UnifiedPlatform-ReGenesis/Shared%20Documents/Program%20Documentation/Program%20Planning/Discovery%20Deliverables%20%26%20Documentation/Salesforce%20team%20Deliverables/Architecture/05062025%20WIP%20Security%20Matrix.xlsx?d=w796b8306c5314622a033076ac7c727e6&csf=1&web=1&e=RqMd8d]

Given that I am an IT user and navigate to the Object Manager

Then I can see that the Collaborative Practice Agreement and Corporation and Tax Information objects have been created

_Note: orange objects are part of the Asymbl managed package and will be configured once they are onboarded_

## Solution

+Create Collaborative Practice Agreement object+

Labels: Collaborative Practice Agreement / Collaborative Practice Agreements (plural)

Does not start with vowel sound

Object Name: Collaborative_Practice_Agreement

Record Name: Collaborative Practice Agreement ID

Data Type: Auto Number

Display Format: CPA-{00000}

Starting Number: 0

Allow Reports = checked, Allow Activities = checked

Allow Sharing = checked, Allow Bulk API Access = checked, Allow Streaming API Access = checked

+Create Corporation and Tax Information object+

Labels: Corporation and Tax Information (same for plural)

Does not start with vowel sound

Object Name: Corporation_and_Tax_Information

Record Name: Corporation and Tax Information ID

Data Type: Auto Number

Display Format: CTI-{00000}

Starting Number: 0

Allow Reports = checked, Allow Activities = checked

Allow Sharing = checked, Allow Bulk API Access = checked, Allow Streaming API Access = checked

Create permission sets
