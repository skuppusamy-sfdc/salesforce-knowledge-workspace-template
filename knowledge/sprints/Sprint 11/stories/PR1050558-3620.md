## Solution

The IP CreatePreventiveCareAgreementRecords is used by two Omniscripts ( Clinician ConfigureRates and ServiceAccFacilityRateCreation). As part of the changes for **ServiceAccFacilityRateCreation**, a flag isShiftCoverage was passed over and checked at the step DM_CreatePCAForNotLocumTypeOfRate. The check was `%isShiftCoverageChange% != "true"` in QA and `%isShiftCoverageChange% == "false"` , in UAT

because this flag is not sent, it would be null and hence the == false did not work. so it did not create the PCA. The fix will be made to check for != true ( we can also check for == null)
