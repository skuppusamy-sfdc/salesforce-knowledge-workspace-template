## Description

As a system administrator, I want to ensure that when users are working with cloned Practice Operations contracts they are not able to hard delete shifts, rates, and rate pay codes, so that we will have traceability of what was removed from the contract.

## Acceptance Criteria

AC1: “Cloned From” **is NOT** populated, **hard delete** records

Given I am system administrator within the Corporate Support apps

When an SFO is working on a **new** contract (RT: **Practice Operations**)/**not cloned**

And contract status = Draft

Then the user shall have the option to select the Delete option on a **Shift** record, if Deleted & “Cloned From” is NOT populated:

- System shall NOT update the “Is Deleted” field 

- System shall allow hard deletes

- System shall hide/prevent users from accessing the delete option if the contract status is ≠ Draft

** System shall delete any related Preventive Care Agreement (**PCA**) records & **Rate Pay Code* records

Then the user shall have the option to select the Delete option on a **Rate Pay Code** record, if Deleted & “Cloned From” is NOT populated:

- System shall NOT update the “Is Deleted” field 

- System shall allow hard deletes

- System shall hide/prevent users from accessing the delete option if the contract status is ≠ Draft

Then on the Summary tab, the user shall have the option to select the Delete option on a **Rate and Type of Rate** record(s), if Deleted & “Cloned From” is NOT populated:

- System shall NOT update the “Is Deleted” field 

- System shall allow hard deletes

- System shall hide/prevent users from accessing the delete option if the contract status is ≠ Draft

** System shall delete any related Preventive Care Agreement (**PCA**), Contract Payment Agreement (**CPA*)record(s), & Rate Pay Code(s).

AC2: “Cloned From” **is** populated, **soft delete** records

Given I am system administrator within the Corporate Support apps

When a SFO is working on a **cloned** contract (RT: **Practice Operations**)

And contract status = Draft

Then the user shall have the option to select the Delete option on a **Shift** record, if Deleted & “Cloned From” is populated:

- System shall update the “Is Deleted” field = True/Checked

- System shall NOT allow hard deletes

- System shall hide/prevent users from accessing the delete option if the contract status is ≠ Draft

** System shall update any related Preventive Care Agreement (**PCA*) records field “Is Deleted” = True/Checked

And if a Shift “Is Deleted” field = True/Checked, the user has the ability to **Re-add**

- System User shall update the “Is Deleted” field = False/Unchecked

** System shall update any related Preventive Care Agreement (**PCA*) and Rate Pay Code(RPC)  records field via ‘Re-add’ on Summary tab (then system will update “Is Deleted” = False/Unchecked) 

Then the user shall have the option to select the Delete option on a **Rate Pay Code** record, if Deleted & “Cloned From” is populated:

- System shall update the “Is Deleted” field = True/Checked

- System shall NOT allow hard deletes

- System shall hide/prevent users from accessing the delete option if the contract status is ≠ Draft

And if a Rate Pay Code (**RPC**) “Is Deleted” field = True/Checked, the user has the ability to **“Readd”,** if Readded: 

- System shall update the “Is Deleted” field = False/Unchecked

Then on the Summary tab, the user shall have the option to select the Delete option on a **Rate and Type of Rate** record(s), if Deleted & “Cloned From” is populated: 

- System shall NOT allow hard deletes

- System shall hide/prevent users from accessing the delete option if the contract status is ≠ Draft

** If the user deletes a **Rate**, then System shall update the related Preventive Care Agreement (**PCA*) record field “Is Deleted” = True/Checked [Existing functionality  to delete and “Readd”]

** And displays record under “Removed” 

** If the user deletes a **Type of Rate**, then the system shall update all the related Preventive Care Agreement (**PCA**) record(s) field “Is Deleted” = True/Checked, within the selected Contract Payment Agreement (**CPA*) record  [Split to story [+PR10505583777+|https://salesforce.atlassian.net/browse/PR10505583777]]

** And displays record(s) under “Removed”
