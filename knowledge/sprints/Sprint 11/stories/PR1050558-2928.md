## Description

As a Recruiter, I need contract dates to be updated automatically, so dates follow specified business rules

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When a clinician has a Clinician Contract that is **active**: 

- ‘Start Date' less than or equal to today’s date

- ‘Status’ equals “Executed”

- ‘End Date’ is blank/No End Date OR End Date is greater (future) or equal to today’s date

And Clinician has a **new** Executed Contract:

- ‘Status’ equals “Executed”

- ‘Employment Type’ is the same other both active/old contract and new executed contract

Then on ‘Start Date’ of the Newly Executed Contract, the system will automatically update the active/old contract’s:

- ‘End Date’ = [New Executed Contract’s Start Date minus 1 day]

- ‘Status’ equals “Cancelled”

- ‘Cancelled Reason’ equals “Superseded”

And if the Newly Executed contract’s ‘Start Date’ is today or in the past, then the system shall **immediately** update the active/old contract: 

- ‘End Date’ = [New Executed Contract’s Start Date minus 1 day]

- ‘Status’ equals “Cancelled”

- ‘Cancelled Reason’ equals “Superseded”

NOTE: ‘End Date’ will not get updated to if the End Date is before the start date, End Date must be AFTER the Start date.

and If there is an existing contract (Old contract) of the same employment Type and Status = executed, EndDate is > TODAY or null and startdate < = today,

Then make the following update on existing executed contract

OldContract.EndDate = NewContract.StartDate - 1,

OldContract.Status - Cancelled

OldContract.CancellationReason = Superseded.
