## Description

As a user in the “BD Finance” Deal Team/Opportunity Team Role or “BD Operations” queue, when the Pro Forma Model has been confirmed, I want the system to automatically generate a task when the Contract(s) (Record Type: Practice Operations) have been created, so that the appropriate individual(s) will be notified to populate the Shift Map (Shift) and Rates template.

NOTE: the template will be used the the Admins to load the shifts and rates into the newly created contract(s)

## Acceptance Criteria

AC1: Auto create **task** to inform that Practice Ops are ready

Given I am a user in the “BD Finance” Deal Team/Opportunity Team Role or “BD Operations” queue within the Corporate Support Apps

And the Pro Forma Model Status = Approved for Contracting

And the system has created one Contract [Record Type: Practice Operations] per Opportunity Services in Status = Active

Then the system shall automatically create one task:  

1. Record Type = Default

1. Subject = Populate the Shift Map & Rates template

1. Assigned To =**BD Finance** OR if no Deal Team/Opportunity Team Role for **BD Finance**, then set to **Queue = “BD Operations”** (users will need to be individually assigned - PROD Zachary Kieffer)

1. Status = Open

1. Due Date = 1 business day

1. Priority = Normal

1. Name = BLANK

1. Related To = Opportunity 

AC2: Queue Creation

Given I am a user in the “BD Finance” Deal Team/Opportunity Team Role or “BD Operations” queue in Corporate Support Apps

When I navigate to the Tasks tab

Then I can see all the task assigned to the “BD Operations” queue (not assigned to an individual) 

And Filtered by: Task Record Type = Default, Assigned To/Filter by Owner = “BD Operations”, Status = Open  

Then I see the following columns: Subject, Related To, Status, Assigned To, Due Date, Priority, Last Modified Date/Time

And I can update the Assigned To field from the list view or from the record.

NOTE: if there is more than one “**BD Finance**“ Deal Team/Opportunity Team member, then assign to the most recently added (This includes if an existing team member’s role was modified)

## Solution

AC1: Create/Reuse a after save Record triggered flow on Contract, when Contract Record Type is Practice Operations, Pro Forma Model Status = Approved for Contracting and the Opportunity.Services__c has at least one in Status = Active then use the decision matrix that was created as part of story (PR1050558247) to create task.

Task Subject, RelatedTo (WhatId) , Assigned to and fill details according to this story.

AC2: Create queue and a list view according to the conditions.
