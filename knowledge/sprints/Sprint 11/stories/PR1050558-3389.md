## Description

As a System Administrator, when managing client contracts, I want to relate a Practice Operations contract, a Healthcare Practitioner Facility, and a specific Process Type, so that I have a tracking mechanism of the type of process needed to update the clinician’s contract based on the changes made at the client contract level.

## Acceptance Criteria

AC1: Update batch job for Clinician Contract updates

Given I am a System Administrator

When a Client Change Management record is created with Process Type = ‘Automation’ [record creation part of story [Client Change Management  Create Records|https://salesforce.atlassian.net/browse/PR10505583388]]

Then I want the system to automatically make the appropriate updates as specified in stories: [+PR10505582916+|https://salesforce.atlassian.net/browse/PR10505582916] & [+PR10505583187+|https://salesforce.atlassian.net/browse/PR10505583187]+.+

## Solution

Run the Job around 9 PM every day

Change the starting point of the Batch Job from Contract to Client Change Management record. 

1. Pick the records from Client_Change_Management__c.Contract.StartDate - 1 <= Today() AND Client_Change_Management__c.Process_Type__c = Automation AND Status__c = New 

1. Update the Clinician Contract related to the HPF (Code should already exist for [https://salesforce.atlassian.net/browse/PR10505582916|https://salesforce.atlassian.net/browse/PR10505582916|smartlink]  and [https://salesforce.atlassian.net/browse/PR10505583187|https://salesforce.atlassian.net/browse/PR10505583187|smartlink] )

1. In the Finally section update back the Status of Client_Change_Management__c.Status__c = Completed

1. In for any error log the error in Error Log object 

## Error_Message__c = Actual Error Message

## Error_Type__c = Apex

## Source_Component__c = Apex Name

## Record_Id__c = Client_Change_Management__c.id

## Error_Timestamp__c = Time

## Stack_Trace__c = Apex Stack Trace
