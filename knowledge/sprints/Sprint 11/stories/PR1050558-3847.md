## Description

**Steps to reproduce:**

1. Login to Core HC org as System Admin

1. Open a PO contract in Executed status and referenced in clinician contracts

1. Clone this contract using ‘Request Contract Change’ button on linked Service account

1. Now, update the values of Stipend and Target rates for 1 facility title alone

1. Add a new Leadership Incentives rate

1. Mark the status of cloned PO contract as Executed

1. Verify the POC and CCM records created

**Expected Result:**

POC and CCM records should be created for contract’s facility title only for the conditions changed

**Actual Result:**

1. POC records are created even if no conditions are satisfied **- retested and passed (aggregate query Shift Type check changed)**

1. POC records are created for ‘Shift Mixed Bag Shift Rates’ even when condition is not satisfied **- retested and passed (Rule 5)**

1. POC and CCM record not created for Rate Pay code change **- retested and passed (rule 11)**

1. Shift rate increased/decreased rule getting added when Shift Bonus rate is increased/decreased **- retested and passed (Rule  8, 9 updated, condition added** Shift__r.Shift_Name__r.Code != “Shift Bonus”**)** 

1. No POC record created for NPPA Tier Change Query  Need to confirm the logic as `PreventiveCareAgreement.Shift_r.CodeSetSubType = "Employee"` will never be true since there is no Shift record with as `CodeSetSubType` Employee ** Solution Change to PreventiveCareAgreement.Facility_Title_r.CodeSetSubType = “Employee”**
