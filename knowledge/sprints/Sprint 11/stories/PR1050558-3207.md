## Description

As a SFO user, on the Start Date of a newly Executed Practice Operations contract, I want the system to automatically update the current Executed Practice Operations contract’s ‘End Date’ to the new contract's Start Date minus 1 day and update the ‘Status’ to “Superseded”, so that I can ensure proper contract management.

## Acceptance Criteria

Given I am a SFO in the Corporate Support apps

When a Service Account has an active **Practice Operations** Contract, active is defined as:

- ‘Start Date' less than or equal to today’s date

- ‘Status’ equals “Executed”

- ‘End Date’ is blank/No End Date OR End Date is greater (future) OR equal to today’s date

And the Service Account gets a **new** Executed Practice Operations Contract (‘Status’ equals “Executed”)

Then **on the 'Start Date'** of the **new** Executed Practice Operations Contract, the system will automatically update the active contract’s:

- ‘End Date’ = [New Executed Contract’s Start Date minus 1 day]

- ‘Status’ = “Superseded” (new status added in [+PR10505582924+|https://salesforce.atlassian.net/browse/PR10505582924])

And if the ‘Start Date' of the **new** Executed Practice Operations Contract is **in the past** (Start Date is less than today’s date)

Then the system shall **immediately** update the active contract’s:

- ‘End Date’ = [New Executed Contract’s Start Date minus 1 day]

- ‘Status’ = “Superseded” (new status added in [+PR10505582924+|https://salesforce.atlassian.net/browse/PR10505582924])

## Solution

** Then **on the 'Start Date'** of the **new* Executed Practice Operations Contract, the system will automatically update the active contract’s:

** Use scheduled Flow that run every DAY that update the Contract Record → Record Type: Practice Operation

*** **Entry Criteria:* Contract Records that has Contract.Status = “Executed” and Contract.StartDate = batch run date i.e. Today() AND Contract.RecordType=“Practice Operation” (this will be the current Contract)

** For each Contract record get the previous contract by using the Contract.AcocuntId = current.Contract.AcocuntId and Contract.Id <> current.Contract.Id AND Contract.StartDate < current.Contract.StartDate and Contract.Status = “Executed” AND Contract.RecordType=“Practice Operation” Order BY Contract.StartDate LIMIT 1

** Set the Contract.EndDate = current.Contract.StartDate -1 and Contract.Status = Superseded and update each Contract related to that ServiceAccount

** Batch need to run after 12 AM on that day

** And if the ‘Start Date' of the **new** Executed Practice Operations Contract is **in the past* (Start Date is less than today’s date)

** Use are record triggered flow in Contract  Contract Record → Record Type: Practice Operation and Contract.Status = “Executed” and Contract.StartDate <= Today()

** For  Contract record get the previous contract by using the Contract.AcocuntId = current.Contract.AcocuntId and [Contract.Id|http://Contract.Id] <> [current.Contract.Id|http://current.Contract.Id] AND Contract.StartDate < current.Contract.StartDate and Contract.Status = “Executed” AND Contract.RecordType=“Practice Operation” Order BY Contract.StartDate LIMIT 1

** Set the Contract.EndDate = current.Contract.StartDate -1 and Contract.Status = Superseded and update each Contract related to that ServiceAccount
