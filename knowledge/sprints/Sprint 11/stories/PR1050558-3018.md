## Description

As a SFO, when a Practice Operations contract status gets updated to Executed and there is a future executed Practice Operations contract, then I want to be notified, so that I can make the necessary adjustments.

## Acceptance Criteria

AC1: Send Notification when there is a Future Practice Operations Contract

Given I am a an SFO within the Corporate Support apps

When a Practice Operations contract Status is updated to Executed

And there is a future dated Practice Operations contract in Status = Executed

Then the system shall automatically generate a task for each contract meeting the above criteria for the SFO so they can make the necessary adjustments

- Task Record Type = ‘Default’

- Subject = A Practice Operations contract has been executed where a future date contract exists

** Assigned To = <Practice Operations Contacting Queue **NEW* - SFO users will be assigned>

- Comments = 

”Practice Operations contract number <insert ‘Contract Number’> [link to the record] for Contract Start Date: <Insert ‘Contract Start Date’> has been updated to Status = ‘Executed’.  There is a future dated Practice Operations contract number <Insert ‘Contract Number'> [link to the record] for Contract Start Date: <Insert ‘Contract Start Date’> in Status = ‘Executed’.

Please review the newly executed Practice Operations contract and if changes are required to the future dated contract, then perform the following steps:

Step 1: Update the future dated Practice Operations contract status = 'Abandoned’.

Step 2: From the Service Account create a contract using the 'Request Contract Change” guided experience and make updates as needed.”

- Status = ‘Open’

- Priority = ‘Normal’

- Related To = <Practice Operations Contract that was just executed> 

AC2: Update Client Change Management records

Given I am a System Administrator

When a Practice Operations contract status is updated to “Abandoned”

Then the system shall locate Client Change Management records meeting the following criteria:

- Related to the Abandoned Practice Operations contract

- Status = New

And for all Client Change Management records meeting the above criteria, the system shall update the Status to “Canceled” 

AC3: Practice Operations Contact Queue

Given I am an SFO user within the Corporate Support Apps

When I navigate to Tasks tab

Then I have the ability to view tasks within List View = Practice Operations Contracting Queue

And filtered by: 

- Task Record Type = Default

- Filter by Owner = Practice Operations Contracting Queue

And the following columns are visible:

- Subject

- Related To

- Due Date

- Status

- Priority

- Type

- Assigned Alias

- Last Modified Date/Time

- Last Modified By Alias

## Solution

AC1: When Contract Record Type = Practice Operation is Status = Executed 

Check if there are any forward Contract for that Account → Contract.StartDate > current.Contract.Start Date and Status = Executed if there are contract records

Create a Task for SFO (queue) with comments as specified in AC

AC2: When Contract Record Type = Practice Operation is Status = Abandoned 

update all Client Change Management Records where Status = New to Cancelled
