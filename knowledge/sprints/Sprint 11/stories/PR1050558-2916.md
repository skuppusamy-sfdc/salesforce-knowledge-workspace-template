## Description

As a SFO, I want the system to automatically update all related Clinician executed Contracts based on specific type of updates made to the **Practice Operations** contract record, so that Clinician Contracts stay in sync with the Client contract and contract generation in Qgenda can be appropriately triggered. 

** **Coverage Change - Only Shift changes*: no new shift record created. New Shift is add with the same Type and same Rate. 

** **Pay Code Change*: Only paycode change, new paycode added to shift (no rate change)  [See story [+PR10505583187+|https://salesforce.atlassian.net/browse/PR10505583187]+]+

## Acceptance Criteria

AC1: Coverage Change - new shift with same shift type and same rate 

Given I am an SFO within the Corporate Support Apps

When I clone a **Practice Operations contract** in Status = Executed 

And I add a new shift with the **same Shift Type****** with the **same Rate* 

And user can only select from **existing rates for that Shift Type**.

And I add a **Rate Pay Code** for that new shift

Then the cloned Practice Operations contract Status gets updated to Status = “Executed” & Start Date equals current date, the system shall automatically Add the Shift & Rate Pay Code to all related **Clinician** contracts meeting the following criteria:

- Clinician Contract Status ≠ Canceled

- HPF Service Account matches Practice Operations Contract’s Service Account 

And if the Clinician rate (same Shift Type - Code Set is higher (overrate), then update the rate to match the Clinician’s rate

(example of adding Same Shift Type & Rate:

I have **Existing**: Shift Type = Day, Iteration = 1, Unit of Measure = per hour, Rate = $100

I **Add**:                Shift Type = Day, Iteration = **2**, Unit of Measure = per hour, Rate = $100)

(example of Different Rates for same shift type, user to select based on current rate values)

Day 1 = 100

Day 2 = 110

Day 3 = User to select from: ‘100’ or ‘110’

*Shift Type = Shift’s “Shift Name” (Shift Name is the Code Set record where Code Set Type = Shift)

## Solution

Create a batch job that is  Scheduled to run daily

on CPA of  (Practice Operations Contract ) where StartDate = Today & Status = Executed , Contract.RecordType = Client, and Type= ShiftType and 

Existing_Shift_Type_Added__c = true and Is_Newly_Added__c = true and rateIncreased = false and rateDecreased = false ( to get the newly added shift of same type and no rate change)

OR Rate_Pay_Code_Added__c!= null or Rate_Pay_Code_Removed__c != null ( to get only if rate pay codes are added or removed - part of story 3187)

Get the Client PCAs of this CPA (as the rates from this should be added to the clinician contract)

**Get Clinician Contracts**

Query for CPAs where Contract.RecordType = Clinician and Clinician Contract with Status != Cancelled, (as we are applying changes from PO contract to all contracts) and PNC>HPF>Account = POContract > ServiceAccount

Copy rates for Same shift type

For each of the clinician CPA, get the PCAs and check if they match with the PO.PCA’s ShiftType (PCA > Shift > Shift_Full_Name__c)  (eg., Day, Night) and check if the facilityTitle matches between Clinician PCA and PO PCA

Check if the PO.PCA.Rate > Clinician PCA.Rate, if so update the rate.

For the matched Clinician pCA, update the FacilityRate lookup to the new Po Contract.

Insert the PCAs
