## Description

**Steps to reproduce:**

1. Login as SFO user - Richard Caramel

1. Open a service account with Practice Operations contract with start date less than today’s date and status as ‘Executed’

1. Create a new Practice Operations contract with start date as current date and mark it as Executed

**Expected Result:**

When a Practice Operations contract with start date as today’s date is marked as Executed, the previous active contract with latest start date should be marked as ‘Superseded’ and End Date should be updated as Current Date - 1

**Actual Result:**

When a Practice Operations contract with start date as today’s date is marked as Executed, the  Status and End Date of previous active contract is not updated

**Note:** The start date condition in the flow **‘Contract After Save’** for getting current and previous active contract is not set as per solution

## Root Cause

The issue occurred in the flow because the condition was configured using only the “less than (<)” operator instead of “less than or equal to (<=)”. Due to this, records with equal values were not evaluated. The condition has now been corrected to use “<=” in the flow.
