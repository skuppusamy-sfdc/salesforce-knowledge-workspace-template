Description: Test Case: CONT002  Create net-new Contract

To verify that a user can successfully create and submit a new contract request

1. Navigate to the Person Account (Search the global search for your Clinician and ensure you choose the search result with Account)

2. Click on the down arrow in the Quick Action area of the page and choose "Contract Request"

3. Choose Employee Type of "Leased" (Time Type will default to "Not Applicable")

4. Click "Next"

5. Enter a Contract Start Date

6. Click on "Start New Request" and click "Next"

7. Select one of more Facilities and click "Next"

8. Skip the Contract Confirmation by clicking "Next"

9. From the Contract Detail screen, click on the "Summary" tab

10. Click "Submit Contract Request"

Expected Result

When initiating a new contract request, the "HPF" option should be visible and selectable in the relevant field (e.g., funding type, contract type).

Actual Result

The "HPF" option was not available in the user interface. The user was unable to select it and therefore could not create the HPF contract.

## Root Cause

The reported issue regarding the missing "HPF" option during contract creation for the "Leased" employee type was due to a **misunderstanding of the system's filtering logic and data dependencies**, not a functional defect.

The system is correctly designed to **filter available HPF options** based on both the **active status** and the **employee/contract type** being created.

The test case failed because the clinician used only possessed an active HPF of type **"Locum,"** while the user attempted to create a contract of type **"Leased."** The system correctly filtered out the irrelevant "Locum" HPF, resulting in the option being hidden.

**Resolution:** The functionality was validated to work as expected when the contract type matched the HPF type ("Locum" contract successfully displayed the "Locum" HPF). The issue was closed after confirming the system is functioning according to its intended filtering design.
