## Description

As a System Administrator, when a Practice Operations or Clinician Contract is cloned via our guided experiences, I want the system to reset the Change Management flags, so that we are accurately capturing the changes for the new cloned Contract.

## Acceptance Criteria

AC1: Reset **PCA** Change Management flags

Given I am a system administrator

When a user completes a guided experience to clone an executed **Practice Operations** or **Clinician Contract** record

Then the system shall rest/clear the following **Preventive Care Agreement (PCA)** Change Management flags (fields) on the newly created contract record:

- Existing Rate Increased?

- Existing Rate Decreased?

- New Shift Type Added?

- New Shift Bonus Type Added?

- Existing Shift Type Added?

- Is Newly Added?

- Systematically Copied from Previous Contract (not Cloned)?

- Rate Pay Code Added?

- Rate Pay Code Remove?

- Is Deleted (Records with Is Deleted Should not be cloned)

AC2: Reset **Rate Pay Code** Change Management flags

Given I am a system administrator

When a user completes a guided experience to clone an executed **Practice Operations** or **Clinician Contract** record

Then the system shall rest/clear the following **Rate Pay Code** Change Management flags (fields) on the newly created contract record:

- Is Newly Added?

- Is Deleted (Records with Is Deleted Should not be cloned)

AC3: Reset **Shift** Change Management flags

Given I am a system administrator

When a user completes a guided experience to clone an executed **Practice Operations** or **Clinician Contract** record

Then the system shall rest/clear the following **Shift** Change Management flags (fields) on the newly created contract record:

- Existing Shift Type Added?

- New Shift Type Added?

- Is Deleted (Records with Is Deleted Should not be cloned)

NOTE: Shifts are only created as a part of **Practice Operations** contract and used in **Clinician Contract**

## Solution

During the Contract Clone Process Set the Check Box Fields to as FALSE Contract  

- RecordType: Practice Operations

- RecordType: Clinician

Shift__c

1. Existing_Shift_Type_Added__c = FALSE

1. New_Shift_Type_Added__c = FALSE

Rate_Pay_Code__c

1. Is_Newly_Added__c = FALSE

Preventive_Care_Agreement__c

1. Existing_Rate_Increase__c

1. Existing_Rate_Decreased__c

1. New_Shift_Type_Added__c

1. New_Shift_Bonus_Type_Added__c

1. Existing_Shift_Type_Added__c

1. Is_Newly_Added__c

1. System_Copied_from_Previous_Contract__c

Any Is_Deleted = TRUE should not be cloned

Shift__c.Is_Deleted__c

Rate_Pay_Code__c.Is_Deleted__c

Preventive_Care_Agreement__c.Is_Deleted__c
