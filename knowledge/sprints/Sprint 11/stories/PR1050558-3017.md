## Description

As a SFO, I want the system to guide me through the Practice Operations contract change request process, so that a new contract is appropriate cloned from the correct executed contract and additional rules are in place to ensure that the new contract is created correctly.

## Acceptance Criteria

NOTE: guide process starts with story [1739|https://salesforce.atlassian.net/browse/PR10505581739], next is this rules story, than after this story is [2906|https://salesforce.atlassian.net/browse/PR10505582906]. 

AC1: Clone from current active contract - **no** contract with **future start date**

Given I am a SFO in the Corporate Support apps

When I navigate to a Service Account record

And there is ONLY one Executed Practice Operations Contract; NO Executed contracts with a future start date (Executed ‘Contract Start Date’ is greater than requested Contract’s start date)

And I select “Request Contract Change”, input required fields, and click Next

Then the system shall clone the Executed contract and new contract Status is set to “Draft”, 

AC2: Clone from current active contract - Allow Draft in between 2 contracts ONLY if **Future contract type is Coverage Change**

Given I am a SFO in the Corporate Support apps

When I navigate to a Service Account record

And there is a Practice Operations contract with future start date and Reason for Change = Shift Coverage Change ONLY

And I select “Request Contract Change”, input required fields, and click Next

Then the system shall clone the current Executed contract (requested contracts ‘Contract Start Date’ is within the executed contract’s start date) and new contract Status is set to “Draft” - clone all fields except EndDate 

**NEW** AC2: Clone from current active contract - Allow Draft in between 2 contracts ONLY if **Future contract type is NOT Coverage Change & New change is Coverage Change**

Given I am a SFO in the Corporate Support apps

When I navigate to a Service Account record

And there is a Practice Operations contract with future start date and Reason for Change = Shift Coverage Change ONLY

And I select “Request Contract Change”, input required fields, and click Next

Then the system shall clone the current Executed contract (requested contracts ‘Contract Start Date’ is within the executed contract’s start date) and new contract Status is set to “Draft”

AC3: Don’t clone - Only **one Draft** Contract 

Given I am a SFO in the Corporate Support apps

When I navigate to a Service Account record

And there is a Practice Operations contract in Status = Draft, Submitted, In Progress

And I select “Request Contract Change”, input required fields and click Next

Then display error message: There is a draft Practice Operation contract under this service account. The system cannot request a contract change at this time. Complete the pending draft Practice Operation contract or cancel the change to proceed further.

AC4: Check for **different rates within Shift Types** (across facility titles & Shift Type)

Given I am a SFO in the Corporate Support apps

When I navigate to a Service Account record

And I select “Request Contract Change”, and **selected Shift Coverage Change only** and click Next

Then before cloning the system shall check if the current Executed Contract (requested contracts ‘Contract Start Date’ is within the executed contract’s start date) if it has shift types that contains different rates per Facility Title and Shift Type combination

- If there are different rates within a single Shift Type & Facility Title combination, then do NOT allow the user to proceed

** Display error message: Unable to proceed with the selected type of change. Current executed contract contains different rates per Shift Type & Facility Title combination.  

- If there are same rates within a single Shift Type & Facility Title combination, then allow user to proceed. 

Example of Same Shift Type and Same Facility Title **w/ same rates**

[Facility Title]  [Shift Type] [Iteration]  [Rate] 

Ambassador NP/PA  Day 1  $100

Ambassador NP/PA  Day 2  $100

EE Physician  Day 1  $200

EE Physician  Day 2  $200

Example of Same Shift Type and Same Facility Title **w/ different rates**

[Facility Title]  [Shift Type] [Iteration]  [Rate] 

Ambassador NP/PA  Day 1  $100

Ambassador NP/PA  Day 2  $110

EE Physician  Day 1  $200

EE Physician  Day 2  $210

AC5: Remove “Clone” button from the Practice Operations contract

Given I am a SFO in the Corporate Support apps

When I navigate to a Practice Operations contract

Then the “Clone” button shall be removed.

AC6: No overlapping Contract Start Dates

Given I am a SFO in the Corporate Support apps

When I navigate to a Service Account record

And I select “Request Contract Change”

And the ‘Contract Start Date’ inputted matches the start date of an executed contract

Then the system shall display an error message if the inputted ‘Contract Start Date’ matches the start date of an executed contract (Design: display error on second step)

- Error Message: There is a contract for the same ‘Contract Start Date’, either select a different date for the change or first cancel the existing contract and try again.

## Solution

Based on Contract Start Date entered in Previous Screen [https://salesforce.atlassian.net/browse/PR10505581739|https://salesforce.atlassian.net/browse/PR10505581739|smart-link] 

Contract Start Date = user entered date from screen [https://salesforce.atlassian.net/browse/PR10505581739|https://salesforce.atlassian.net/browse/PR10505581739|smart-link] 

NOTE: As part of this story, the mapping of Radio (Coverage change - Yes or No) of 1739 should be to contract.Shift_Coverage_Change__c (practice operations)

1. For All Use Case First Step:

## Check if there is forward contract in the System Query: Contract.RecordType = Practice Operations AND Contract.Status in Draft, Submitted, In Progress AND AccountId = Account from where the button was clicked

### Display All the Contract forward Contract using the query output above

### Error Message: There is a draft Practice Operation contract under this service account. The system cannot request a contract change at this time. Complete the pending draft Practice Operation contract or cancel the change to proceed further. 

## Check if there is forward contract in the System Query: Contract.RecordType = Practice Operations AND Contract.Status in Executed AND StartDate = Contract Start Date AND AccountId = Account from where the button was clicked

### Display All the Contract forward Contract using the query output above

### Error Message: There is a contract for the Contract Start Date select either a different date for the change or first cancel the existing contract and try again.

## Check if there is contract to clone Query:  Contract.RecordType = Practice Operations AND Contract.Status = Executed AND AccountId = Account (from where the button was clicked) Order By Contract.StartDate DESC LIMIT 1

### when not found message “No previous contract not found to perform a change to clone from”

1. When **Are you requesting only Coverage Change?** = No

## Check if there is forward contract in the System Query: Contract.RecordType = Practice Operations AND StartDate > Contract Start Date AND Shift_Coverage_Change__c = false AND Contract.Status = Executed  AND AccountId = Account from where the button was clicked

### if found then display there error message

#### Display All the Contract forward Contract using the query output above

#### Error Message: “There is an executed Practice Operation contract for Rate Change under this service account that lies within the time window. The system cannot request a contract change at this time.“

### When **not** found 

#### Display Contract that can be cloned using the Query:  Contract.RecordType = Practice Operations AND Contract.Status = Executed AND  Contract.StartDate  < Contract Start Date  AND AccountId = Account (from where the button was clicked) Order By Contract.StartDate DESC LIMIT 1 (Clone the last Contract that executed that is before the requested start date (Contract Start Date) with and navigate to the cloned contract record) (e.g. Contract Start Date = 04/15/2025, C1: 01/01/2025, C2: 04/01/2025, C3: 06/01/2025 then you need to pick C2 to clone)

##### When User Clicks the Create Button

###### when found Clone the entire Contract records including Shift__c, ContractPaymentAgreement, PreventiveCareAgreement and Rate_Pay_Code__c

### When not found **(Insert new contract in future date)**

#### Display Contract that can be cloned using the Query:  Contract.RecordType = Practice Operations AND Contract.Status = Executed AND  Contract.StartDate  < Contract Start Date  AND AccountId = Account (from where the button was clicked) Order By Contract.StartDate DESC LIMIT 1
