## Description

As an Implementation Specialist, when the Opportunity (RT: Growth Opportunity) updated to “LOI Executed”, I want a task to be automatically created for our team, so we know to manually create the PMT Project & Clone the Phases/Tasks which will be used to track the onboarding process for each Service Account.

## Acceptance Criteria

AC1: Auto Task Creation to Create PMT Project

Given I am a Implementation Specialist in the Corporate Support Apps

When the Opportunity (RT: Growth Opportunity) field = “LOI Complete” is checked/true 

And “PMT Project” field is not populated

Then the system shall automatically create a **Task** to notify the users assigned to the “Implementation Queue” to create a new PMT Project:

- Record Type = Default

- Subject = Create a PMT Project for <Opportunity Name>

- Assigned To = Implementation Queue

- Comments = 1. Please create a new PMT Project for the Opportunity 2. “Clone Phases/Tasks” for each Service Account 3. When ready to send out the related tasks, update the PMT Task Status = “In Progress”

- Comments = 1. Create a new PMT Project for the Opportunity from PMT Template 2. Associate the Opportunity to the PMT Project record  3. “Clone Phases/Tasks” for each Service Account 4. For each PMT Phase associate the Service Account  5. When ready to send out the related tasks, update the PMT Task Status = “In Progress”

- Status = Open (default)

- Due Date =<+5 days from date of creation>

- Type = Client Onboarding

- Priority = Normal (default)

- Related To = <Opportunity (RT: Opportunity Growth)> 

NOTE: if the criteria is met we will create a task, user can manually close (canceled) if they have not populated the “PMT Project” and a duplicate task is created.

## Solution

Use a update record triggered flow on Opportunity (flow should only trigger when record is updated to meet the condition)

Opportunity.LOI_Complete__c = true and Opportunity.PMT_Project__c = null

Create a task record as specified in the AC
