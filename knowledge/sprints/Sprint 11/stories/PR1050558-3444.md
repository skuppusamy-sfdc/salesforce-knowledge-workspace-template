## Description

As a Recruiter, when an update is made to a Clinicians contract and as a Client Rate Speciality , I need the system to capture that the existing rates were increased, decreased, or rate was added (net new), SO that Addendum rules can be defined for Conga and Client Contract

## Acceptance Criteria

Given I am a system administrator within the Corporate Support apps

When I am in Setup on Object = **Contract Payment Agreement**

Then the following new fields are create (should be visible in Page Layout as Read Only - Check Data Dictionary for Details)

- Mixed Bag Shift Rates? (Logic provided in solution)

## Solution

- Apples for Both Practice Operation Contract and Clinician Contract

** For each **Contract Payment Agreement** records select query all related Shift__c records **Preventive Care Agreement** records and count based on logic below and update the count into the **Contract Payment Agreement.Mixed_Bag_Shift_Rates__c* records using PreventiveCareAgreement.ContractPaymentAgreementId

** Mixed Bag Shift Rates

***  For one of the FacilityTitle__c if Max(Rate__c) != MIN(Rate__c) then Set the Mixed Bag Shift Rates = 1

**** Query:SELECT FacilityTitle__c, Shift__r.Shift_Name__c, Max(Rate__c), MIN(Rate__c) FROM ContractPaymentAgreement WHERE Shift__r.Shift_Name__r.Code != “Shift Bonus” AND 

GROUP BY FacilityTitle__c, Shift__r.Shift_Name__c
