## Description

As a CET (Clinician Experience Team),  I need tasks to be automatically created on SCP Onboarding child cases, so that I can begin necessary onboarding or early onboarding tasks needed for a clinician.

## Acceptance Criteria

Given I am a CET user in the Corporate Support apps

When Facility Onboarding cases are created (either manually through Initiate Early Onboarding or automatically when Clinician Contract Status = Executed)

Then I am able to see that the following **tasks are created** within their appropriate cases:

- SCP Onboarding Parent Case (already built)

** Training & SCP University Child Case (already built)

*** Complete Training

*** Compliance Training

** HR Setup (already built)

*** W2 Completion

*** Workday Termination

*** I9 Update

Tasks are created with the Default record type

_additional criteria for creating tasks are captured in the clinician onboarding sheet_

## Solution

Create Decision Matrix that has the following columns

Name: Task Creation Decision

CaseSubject (Input, Text)

TaskSubject(Output, Text)

RelatedToType (Output, Text)

RelatedTo (output, Text)

AssignedToType (Output, Text)

AssignedTo (Output, Text)

Record triggered flow on Case, to create tasks (related to case) based on the Decision matrix created above, where the input is Case subject and output of the Decision matrix can be used to create task records.

Task record type: Default
