## Description

**Steps to reproduce:**

1. Login as SFO user - Richard Caramel

1. Open a service account and navigate to Related tab

1. Open a client contract record with rates specified for various type from the Contracts related list

1. Click Clone button

1. On the cloned contract, try to add/update/remove rates

**Expected Result:**

Rates should added/updated or removed when it is done by SFO user on the client contract

**Actual Result:**

Rates are not getting added/updated or removed when it is done by SFO user. However it is working for System Administrator but inconsistently

SFO is able to specify rates on a new client contract but not on a cloned client contract

Original client contract: [https://corehc-qa.sandbox.lightning.force.com/lightning/r/800gP000004MkRiQAK/view|https://corehc-qa.sandbox.lightning.force.com/lightning/r/800gP000004MkRiQAK/view]

Cloned client contract: [https://corehc-qa.sandbox.lightning.force.com/lightning/r/Contract/800gP000004SvsIQAS/view|https://corehc-qa.sandbox.lightning.force.com/lightning/r/Contract/800gP000004SvsIQAS/view]

## Root Cause

Contract and its child object sharing is not public read/write, to clone the contract record owner also needs to be updated for proper sharing of record with current SFO user.

As currently no sharing rule for Contract - to fix it changed record owner on cloned contract with current user to give read/write access on clone record
