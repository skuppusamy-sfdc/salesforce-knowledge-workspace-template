## Description

As a Recruiter, when I create Clinician Contract rates, I want the Contract Payment Agreement request records names to be consistent and unique, so that it's easy to view and there are no duplicates.

## Acceptance Criteria

Given that I am a Recruiter in the Corporate Support apps

When I navigate to a **Clinician** record

And I click on the **Contract Request** button 

And select the option to Start From Previous Active Contract

Then they system should clone the Contract Payment Agreement record(s)

And Name = ContractnumberGlobal_Account_Idratetype

Given that I am a Recruiter in the Corporate Support apps

When I navigate to a **Clinician** **Contract** record in Status = Draft

And I select the Summary tab

And I click on the **Configure Rates** button

Then upon clicking Finish, the system shall create new Contract Payment Agreement record(s)

And Name = ContractnumberGlobal_Account_Idratetype

Given that I am a Recruiter in the Corporate Support apps

When I navigate to a **Clinician** **Contract** record in Status = Draft

And I select the Summary tab

And I click on the **Configure Rates** button on an HPF/Facility with no rates

And I select a Type of Rate where the Client Contract does not have any rates for that type OR no rates for that Facility Title

Then the following message is displayed: **No available rates**

## Solution

Clinician Contract:

ContractPaymentAgreement. Name = ContractnumberGlobal_Account_Idratetype
