## Description

As a Medical Staff Liaison, I need a parent onboarding case to always be created during the onboarding process, so that I can easily view all child cases under a single umbrella case

## Acceptance Criteria

Given I am a MSL in the Corporate Support apps

When a clinician’s contract record changes so that **Status = Executed** or **Initiate Early Onboarding** was manually triggered

Then I am able to see that the following records are created:

- SCP Onboarding parent case [no change]

** Training & SCP University child case [no change]

** HR Setup child case [no change]

- Facility Onboarding parent case (one record for each HPF)

*** MSS Onboarding child case [**NEW  see details on the Onboarding sheet  Sprint 7 tab*]

*** Hospital Application Preparation child case **[UPDATE - former name was Hospital Verification & Application]*

** Hospital Privileging child case [no change]

** Payer Enrollment child case [no change]

** Hospital Orientation child case [no change]

No duplicates are created if a case already exists from the Initiate Early Onboarding experience (NOTE: for SCP, no **existing open** SCP Onboarding (per Clinician) case. Open = In Progress or Not Started)

Case creation matches what happens when Initiate Early Onboarding is completed with all possible facilities selected (Case Origin set to “Automatic”)

Priority for all newly created records is set to Medium

## Solution

Use a record triggered flow and reuse the existing Child Case Creation Flow

- When the Clinician Contract Status changes to Executed Search for Query HPF Record with Status = Prospect or Candidate with below criteria 

** Contract > ProviderNetworkContract > HealthcarePractitionerFacility, HealthcarePractitionerFacility.Status = Prospect, Candidate and 

***  Case record Case.Healthcare_Practitioner_Facility__c =  HealthcarePractitionerFacility.Id 

- When no Case record is found then Create a Case record as specified in the AC for each HealthcarePractitionerFacility

- When Case records are found then DO NOT Create a Case record for that HealthcarePractitionerFacility

- The creation of the Parent Case should auto create Child cases

** CET: Clinician Experience Team, supports Clinician once contract is complete. **(Update Security Matrix)*

- Please ensure that the solution is reusable, in a future story upon the Contract being Executed, we will need to automatically trigger the creation of the Facility onboarding parent cases and child cases.

Testing:
