## Description

As a SFO, I need to specify if a shift is for NP/PAs or Physicians, so that shift maps can accurately be created in QGenda

## Acceptance Criteria

Given I am an SFO in the Corporate Support apps

When I navigate to a Service Account record

And I open a Contract (Client) record where Status = Draft

Then I am able to create new Shift records with the following: 

- Type: NP/PA or MD/DO or RN/LPN

** Required to create a shift record

- All Fields are Required: Name, Iteration, Day of the Week, Start Time, End Time, Type

## Solution

Create a new Picklist field Type in **Shift** object as specified in Data Dictionary

make the Field as Required as marked in Data Dictionary.

1. Shift Name

1. Shift Iteration

1. Shift Day of the Week

1. Shift Start Time

1. Shift End Time

Error message = Contract must be in “Draft” status in order to add a new Shift.
