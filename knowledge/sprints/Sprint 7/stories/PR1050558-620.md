## Description

As a Medical Staff Liaison, I need case records to be closed automatically, so that I do not have to do duplicate work in multiple systems

## Acceptance Criteria

Given I am a MSL in the Corporate Support apps

**When all of the Task records in a Clinician Onboarding Case are closed**

And the Case record does not already have Status = Closed

Then the Case is automatically updated so that Status = Closed and Closed Reason = Complete 

Given I am a MSL in the Corporate Support apps

**When all of the child Case records in a Clinician Onboarding Case are closed**

And the Parent Case record does not already have Status = Closed

Then the Parent Case is automatically updated so that Status = Closed and Closed Reason = Complete

## Solution

Use a record triggered flow (reuse existing flow if possible) on Task

When task status is updated to ‘Completed’, and the related Case is of record type “**Clinician Onboarding**” ( and its a parent case)

Get all child tasks of the Case and check if the status is Completed.

Parent Id = CaseId of the triggering Task record

Status != Closed

Decision : Check if count of Get records is 0

Then update the Parent Case record’s status

Similar logic for checking all Child cases of Parent Case of record type “**Clinician Onboarding**”
