## Description

**Steps to reproduce:**

1. Login as SFO user - Richard Caramel

1. Open a service account and navigate to Related tab

1. Open a client contract record with rates specified for various type from the Contracts related list

1. Click Clone button

1. On the cloned contract, try to update existing values for the type of rate ‘Shift' using 'Specify Rates’ button and specify a different rate value for the existing **type, facility, unit of measure** **combination**

**Expected Result:**

Existing PCA record should get updated and the new values should be displayed under ‘Changed’ section in Summary tab of client contract

**Actual Result:**

A new PCA record is getting created and the new values are displayed under ‘New’ section in Summary tab of client contract

New client contract: [https://corehc-qa.sandbox.lightning.force.com/lightning/r/Contract/800gP000004YFgfQAG/view|https://corehc-qa.sandbox.lightning.force.com/lightning/r/Contract/800gP000004YFgfQAG/view]

Cloned client contract: [https://corehc-qa.sandbox.lightning.force.com/lightning/r/Contract/800gP000004YG6UQAW/view|https://corehc-qa.sandbox.lightning.force.com/lightning/r/Contract/800gP000004YG6UQAW/view]

## Root Cause

Cloned CPA was linked to old Shift of old contract due to which upsert was not working correctly.

Had to create new Cloned from field in Shift object.

when new cloned CPA was created then it gets linked to new cloned Shift (which is linked to old shift)
