## Description

As a Legal team member, I need to review Clinical Match requests, so that I can make the appropriate decision for a clinicians match group to be displayed to other SCP corporate support team members

## Acceptance Criteria

Given I am an MSL in the Corporate Support apps

When I navigate to a clinician person account record

Then I complete the “Initiate Clinical Match” process

Then the following happens: 

- Case records are created for the following:

** Assign to Queue - Clinical Match Queue 

** reviewer is able to indicate Clinical Match Group level on the Person Account (required, single select)

- Restricted Visibility Needed

** Only Compliance, Risk, Legal can view Clinical Match Requests

NOTE:  visibility is managed across teams and **specific to individuals** within a team - access is granted to specific users

## Solution

**Object, Group Setup**

Create new record type on Case - ‘Clinical Match’

Create Queue for the Case object with the name 'Clinical Match Queue'

Create public group ‘Clinical Match Request Group’ [~accountid:712020:b8b82845166549bbbcedce96d04329bd]  Should there be a different name for the group

The queue should be visible to public group ‘Clinical match group’ - membership added by Sys admin.

Create Case record of record type ‘Clinical Match' and assign to Queue ‘Clinical match queue’

**Record Sharing**

Add Case Sharing Rule (Record level access - based on Public group)

Case : Case Record Type = 'Clinical Match’ Share With Clinical Match Group - Read/Write

Add post deployment steps to add members to public group.

**Field level access**

Contact.Provider_Review_Level__c - Read only for all personas EXCEPT

New permission set : Edit Clinical Match Group - Edit permission
