## Description

As a Clinician Contracting Specialist, I need Salesforce to enforce validations around updating clinician contracts, so that duplicate record creation and mismatches are prevented

## Acceptance Criteria

Given I am a Recruiter in the Corporate Support apps

When I attempt the following scenarios while creating or updating Clinician Contract records

Then the following happens: 

- Duplicate Executed Contract with Same Employment Type

** Update original Contract record where Status = Executed to Status = Canceled and Cancel Reason = Terminated

- Drafts Expire After 30 Days

** When a contract record is created, set the Draft Expiry Date to 30 days from creation date

** When the Draft Expiry Date is today, update the Contract Status to Status = Canceled and Cancel Reason = Expired

_Note: multiple Canceled contracts can exist with the same Employment Type_

## Solution

**Duplicate Executed Contract with Same Employment Type**

  Record triggered flow on Contract, when Contract is created updated, query for other contracts (of record type = Contract) with the same employee Type, for the Clinician and check the status. 

If Contract.Status is execute, update Contract.Status = Canceled and Contract.Cancel_Reason__c = Terminated 

**Drafts Expire After 30 Days**

Use existing flow that is used to create contract (if available). Set the Draft Expiry Date to Today + 30

Need to check if Draft Expiry Date exists, if not create the field.

Create a schedule trigger flow , that runs daily and checks/filters for Contracts where Draft expiry date is TODAY.

For those contracts, update the Contract.Status = Canceled and Contract.Cancel_Reason__c = Expired
