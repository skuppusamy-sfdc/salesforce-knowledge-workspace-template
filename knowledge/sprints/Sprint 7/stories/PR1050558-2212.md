## Description

Steps to Reproduce:

1-Login as System Admin and navigate to Account object.

2-Create a Service Account record.

3-Now navigate to related tab and create a Client Contract record.

4-Now login as SFO and navigate on same Client contract record.

5-Navigate to the Shift list and create a shift record.

Issue - While creating a Shift record, getting insufficient access error.

Expected Result - User should not get the insufficient access error.

Actual Result - While creating a Shift record user is getting insufficient access error.

## Root Cause

As per Security Matrix, Contracts records were not shared with SFO User and hence the user was not able to Create the Shift Record. Later Samudhra gave the solution to add the SFO User to the Account team.
