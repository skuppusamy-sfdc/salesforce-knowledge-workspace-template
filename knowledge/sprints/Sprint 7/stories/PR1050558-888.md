## Description

As a system, I need to display “pizza tracker” values based on component location so that I can ensure that internal audiences (internal SCP Console users) are seeing internal values and external audiences (Experience Cloud users) are seeing different, limited values.

## Acceptance Criteria

Given I a SCP Corporate Support user using the Corporate Support App,

When I navigate to a clinician’s Person Account record,

Then I am able to

- View “Status / Label (Internal)” under the Section Headers from the “Status Label + Logic” tab when a Section Header is visible

** Values in “Status / Label (External)” IS NOT VISIBLE TO INTERNAL USERS

Given I am external Experience Cloud user,

When I navigate to my home page,

Then I am able to

- View “Status / Label (External)” under the Section Headers from the “Status Label + Logic” tab when a Section Header is visible

** Values in “Status / Label (Internal)” IS NOT VISIBLE TO EXTERNAL USERS

- If candidate is no longer in consideration for a given job but other applications are still active / header still available, remove that line from visibility

NOTE: 

- External Users will have a different UI/user story to view the details of their specific applications, interviews, etc.
